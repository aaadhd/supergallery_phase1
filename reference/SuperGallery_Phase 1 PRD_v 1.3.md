# 1. 제품 개요

## 1.1 제품 정의

**Artier(아티어-가칭)**는 그림을 그리는 누구나 작품을 올리고, 발견되고, 더 넓은 기회로 연결될 수 있는 디지털 아트 플랫폼이다. 

## 1.2 해결하는 문제

국내 디지털 드로잉 인구는 빠르게 성장하고 있으나, 중장년 창작자를 위한 **작품 발표·전시 공간**이 사실상 없다. 기존 플랫폼(노트폴리오, Behance 등)은 전문가 포트폴리오 중심이고, SNS(인스타그램 등)는 알고리즘 의존도가 높아 신규 작가의 발견이 어렵다.

| 문제 | 대상 | 현재 대안의 한계 |
| --- | --- | --- |
| 작품을 올릴 공간이 없다 | 비전공 창작자, 중장년 | 전문가용 플랫폼 진입 장벽 높음 |
| 올려도 발견되지 않는다 | 신규 작가 전반 | SNS 알고리즘 불리, 팔로워 없으면 노출 없음 |
| 그룹전시를 열 수 없다 | 드로잉 클래스 강사·수강생 | 별도 협업 도구 필요, 번거로움 |
| 창작이 기회로 연결되지 않는다 | 실력 있는 비전공 작가 | 포트폴리오·교육·커머스 연결 채널 없음 |

## 1.3 타겟 유저

| 세그먼트 | 정의 |
| --- | --- |
| 전공자 | 미술 전공·직업 화가 경험이 있는 창작자 |
| 비전공 실력자 | 전공자는 아니지만 자신만의 작업 세계를 갖춘 창작자 |
| 일반 창작자 | 취미·일상 속에서 꾸준히 그림을 그리는 사용자 |

> Phase 1 초기 큐레이션은 **중장년(50+) 창작자 및 재도전형 창작자** 중심으로 운영하여 플랫폼의 분위기와 정체성을 형성한다.
> 

## 1.4 핵심 가치 제안

| 가치 | 설명 |
| --- | --- |
| 내 작품을 공개 전시 | 오프라인 전시 없이도 온라인 갤러리에 작품을 올리고 공개적으로 선보일 수 있다 |
| 발견과 재도전의 기회 | 작품이 더 많은 사람에게 노출되고, 다시 활동을 이어갈 계기를 제공한다 |
| 감상과 교류 | 작품을 감상하고 의견을 나누며 작가와 사용자 간 연결을 만든다 |
| 확장 가능성 | 전시에 그치지 않고 교육·유통 등 더 넓은 기회로 연결된다 |

# 2. 범위 정의 (Scope)

- 회원가입 / 로그인 (소셜 + 이메일, 한국/글로벌 분기)

- 작업물 상세 모달 (좋아요, 저장, 공유)
- 작가 프로필 (내 프로필 / 타인 프로필)
- 팔로우 / 언팔로우
- 알림 (실시간 푸시 알림 x)
- 검색 (작품명 / 작가명 / 전시명)
- 이벤트 목록 및 상세
- 신고
- 법적 페이지 (이용약관, 개인정보처리방침, FAQ, 문의하기)

- 작품 비공개 전환 (수정/삭제와 동일 레벨 Context Menu. Behance·노트폴리오 동일 기능 제공)

# 3. 성공 지표 (Success Metrics)

## 3.1 Phase 1 목표 (~오픈 후 3개월)

| 지표 | 목표값 | 측정 방법 |
| --- | --- | --- |
| 누적 가입 회원 수 | 800명 이상 | DB 집계 |
| 누적 업로드 작가 수 | 200명 이상 | 업로드 유저 기준 |
| 누적 작품 업로드 수 | 3,000점 이상 | DB 집계 |
| MAU | 500명 | 월간 활성 유저 |
| 30일 재방문율 | 15% 이상 | 코호트 분석 |

## 3.2 핵심 가설

1. **콘텐츠 가설**: 파트너 작가 50명 확보 + 작품 500~600점 사전 업로드 시, 둘러보기 피드의 첫인상이 신규 유저의 가입 전환율을 높일 것이다.
2. **바이럴 가설**: 전시 초대장 공유 기능이 작가 본인의 SNS 홍보 수단이 되어 외부 유입을 만들 것이다.
3. **리텐션 가설**: 주간 테마 기획전 + Artier's Pick 운영이 기존 작가의 재방문을 유도할 것이다.
4. **그룹 유입 가설**: 그룹전시 기능이 드로잉 클래스 강사-수강생 단위의 집단 유입을 만들 것이다.

# 4. 사용자 스토리

## 4.1 일반 창작자

- 나는 **그림을 올릴 공간이 없어서**, Artier에 가입하고 내 첫 작품을 업로드하고 싶다.
- 나는 **내 작품이 얼마나 발견되는지** 좋아요와 팔로워 수로 확인하고 싶다.
- 나는 **내 작품을 지인에게 공유할 때**, "내 전시에 초대합니다" 형식의 초대장으로 보내고 싶다.

## 4.2 드로잉 클래스 강사

- 나는 **수강생들의 작품을 한 곳에 모아**, 그룹전시로 수업 결과를 보여주고 싶다.
- 나는 **내 프로필 페이지가 수업 홍보 도구**가 되길 원하며, 수강생 작품을 모아보는 탭이 필요하다.
- 나는 **비가입 수강생에게도 초대 문자를 보내**, 그들이 Artier에 가입하고 자신의 작품을 확인하도록 하고 싶다.

## 4.3 감상자 (비작가 유저)

- 나는 **인스타그램처럼 스크롤하며** 다양한 작가의 작품을 발견하고 싶다.
- 나는 **마음에 드는 작가를 팔로우**하고, 신작이 올라오면 알림을 받고 싶다.
- 나는 **좋아요를 누른 작품**을 나중에 다시 볼 수 있도록 저장해두고 싶다.

## 4.4 운영팀

- 나는 **매주 Artier's Pick을 선정**하여 우수 작품에 배지를 부여하고 싶다.
- 나는 **신고된 작품을 7일 이내에 검토**하여 삭제, 경고, 기각을 처리하고 싶다.
- 나는 **배너와 이벤트를 어드민에서 직접 등록**하여 별도 개발 요청 없이 운영하고 싶다.

# 6. 비기능 요구사항

## 6.1 성능

| 항목 | 기준 |
| --- | --- |
| 피드 초기 로딩 | 3초 이내 (LCP 기준) |
| 이미지 업로드 | 10MB 이미지 기준 5초 이내 처리 |
| 페이지 전환 | SPA 기준 1초 이내 |
| 서버 응답 | API 평균 응답 500ms 이내 |

## 6.2 보안

- 이미지 EXIF 메타데이터 분석으로 카메라 촬영 파일 업로드 차단
- 개인정보 처리 최소화 원칙 (개인정보보호법 준수)
- 비밀번호 최소 8자, 영문+숫자 필수
- 비밀번호 재설정 링크 만료: 1시간
- 이메일 인증 링크 만료: 30분
- 403 처리: 로그인 리다이렉트 (redirect 파라미터 포함)
- SSL 인증서 필수 적용
- 이미지 직접 저장 차단 (All Rights Reserved 정책 적용 작품 기준). 단, 스크린샷·화면 캡처는 기술적으로 차단 불가 (Behance 동일 정책)

## 6.3 접근성 및 법적 요건

- 법적 페이지 폰트 12px 이상 유지 (정보통신망법)
- 전자상거래법 제10조 기준 사업자 정보 전체 노출
- 이용약관·개인정보처리방침 변경 시 시행 7일 전 전체 이메일 발송 (법적 의무)

# 7.3 개발자 결정 항목

> PM이 방향을 제시하되, 최종 구현 방식은 개발자 판단에 따름.
> 

| 항목 | PM 방향 제시 | 개발자 결정 사항 |
| --- | --- | --- |
| 전시 상세 URL 구조 | `/exhibitions/:id` 단일 URL로 통일 (개인/그룹전시 구분 없음) | Work 테이블과 Exhibition 테이블을 통합할지, 각자 유지하되 라우터에서 type 기반 분기할지 |
| 실시간 알림 방식 | 앱 내 알림 (실시간 푸시 x) | WebSocket vs Polling 선택 |
| 검색 엔진 | 작품명·작가명·전시명 검색 지원 | Elasticsearch, 전문검색 DB 등 구현 방식 |
| 자동 로그인 구현 | 소셜 로그인 후 자동 로그인 지원 | 토큰 방식 등 구현 방식 |
| 이메일 발송 도구 | 10종 템플릿, 재시도 3회 | 발송 서비스 선택 (SES, SendGrid 등) |
| 타인 프로필 URL | `/exhibitions/:id` 통일과 동일 맥락 | `/:username` vs `/user/:id` |
| 어드민 접근 방식 | 운영팀 전용, 역할 기반 접근 제어 | URL 구조, 인증 방식, 역할별 권한 범위, 로그 보관 기간 |
| 렌더링 방식 | — | 정적 HTML vs 마크다운 파싱 |
