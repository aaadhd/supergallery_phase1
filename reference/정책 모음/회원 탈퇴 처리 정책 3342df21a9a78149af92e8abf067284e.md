# 회원 탈퇴 처리 정책

관련 화면: 계정 설정, 내 프로필
구분: 운영 정책

> 탈퇴 시 데이터 처리 방식 및 username 보존 정책.
> 

> PRD v1.2 및 정책 논의 확정 내용 기준.
> 

---

## 탈퇴 후 데이터 처리 원칙

| 항목 | 정책 |
| --- | --- |
| 작품 처리 | 익명 유지 ("삭제된 사용자"로 표기). 전시에서 제거하지 않음 |
| username 유지 | 작품이 남기 때문에 username 삭제 안 함 |
| 닉네임(displayName) | **탈퇴 후 30일간 닉네임 잠금** — 다른 사용자가 동일 닉네임 사용 불가 |
| 이메일 | 주소 유지 (Social 계정 연결 해제 제외) |
| 모든 데이터 삭제 | 즉시 삭제 후 복구 불가 |

## 탈퇴 프로세스

```
탈퇴 요청
  ↓
확인 모달 ("탈퇴하면 복구가 불가합니다")
  ↓
탈퇴 완료 이메일 발송
  ↓
로그아웃 처리
  ↓
세션 일괄 만료
```

## 닉네임 유닉 정책

> **설계 배경**: username과 displayName이 분리되어 있으나, 사용자가 displayName 기반으로 활동할 것으로 예상되는 서비스 특성상 displayName도 Unique 정책 유지.
> 
- displayName Unique 적용 → 다른 유저가 탈퇴한 닉네임과 동일한 닉네임으로 가입 불가
- **탈퇴 후 30일이 지나도 해당 닉네임 사용 불가**

## 에러 모달 문구

| 위치 | 문구 |
| --- | --- |
| 탈퇴 확인 모달 | "탈퇴하면 작성하신 데이터는 복구가 불가합니다." |
| 탈퇴 완료 | "탈퇴가 완료되었습니다." (+ 탈퇴 완료 이메일 발송) |
