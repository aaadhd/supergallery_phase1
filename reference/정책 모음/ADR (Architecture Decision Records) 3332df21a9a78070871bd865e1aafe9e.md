## 🧑‍💼 계정 / 인증

| Issue | 문제상황 | 결정 | 일자 |
| --- | --- | --- | --- |
| **탈퇴한 회원의 username 처리** | username을 삭제하면, 남겨진 작품에 "삭제한 사용자"로 표기. 삭제하지 않으면 해당 username을 다른 사용자가 사용할 수 없음. | 작품은 남기 때문에 username은 삭제하지 않는다. **영구 점유.** | 2026-04-01 |
| **displayName 중복 여부** | username과 displayName이 분리되어 있다. displayName도 unique 정책을 가져갈 것인가? | 사용자 입장에서 displayName 기반으로 활동할 것으로 예상. **displayName도 UNIQUE 유지.** 탈퇴 후에도 해당 닉네임을 가져갈 수 없다. | 2026-04-01 |
| **이메일/비밀번호 가입 제공 여부** | OAuth, OTP 방식으로 로그인 수단이 많은데, 이메일/비밀번호 가입이 필요한가? | **서브로 보이도록 제공한다.** 소셜 버튼 하단에 "이메일로 가입" 링크 배치. | 2026-04-01 |
| **동일 이메일 소셜 로그인 충돌** | DB 구조상 한 Account에 복수 AuthProvider 연결이 가능하지만, 기획서 에러 메시지("이미 해당 이메일로 가입된 계정이 있습니다" / "이전에 [OO] 로그인으로 가입하셨습니다")의 뉘앙스는 중복 가입을 거부하는 방향. **자동 연결 vs 거부** 결정 필요. | **Phase 1: 자동 연결 없이 기존 가입 방식으로 로그인 유도.**
이유: (1) 시니어 타깃이라 "내가 뭘로 가입했더라" 혼란 방지, (2) 자동 연결은 보안 리스크 — 이메일 소유 미검증 상태에서 계정 탈취 가능.
계정 연결 기능은 Phase 2+ 계정 설정에서 명시적 연결로 제공. | 2026-04-01 |
| 외부 로그인 계정 | 현재는 iProud 용 계정으로 연동 중인데, Apple, Naver SNS 서비스 등, 유료 서비스 계정 정리 필요 | 아이스크림미디어 타 부서에 현황 체크 후 결정 | 2026-04-01 |
