📋 유저스토리 기반 WBS v2: 나는 마음에 드는 작품에 좋아요를 누를 수 있어야 한다 (https://www.notion.so/3352df21a9a78196a25be62e1a23d640?pvs=21)

## 개요

작품에 좋아요를 누르는 기능. 내 프로필 > 좋아요 탭에서 모아보기 가능. 좋아요 수는 피드 카드와 상세 모달에 노출됨.

## 진입점

- 작업물 상세 모달 — 좌측 액션 영역 하트 아이콘
- 둘러보기 피드 카드 (좋아요 수 표시)

## 동작 정책

| 케이스 | 동작 |
| --- | --- |
| 비로그인 상태에서 클릭 | 로그인 유도 모달 |
| 로그인 상태에서 클릭 | 즉시 좋아요 적용 + 아이콘 filled 상태 전환 + 카운트 +1 |
| 다시 클릭 (취소) | 좋아요 취소 + 카운트 -1 |
| 자신의 작품 | 좋아요 가능 (자기 작품 좋아요 허용) |

## 알림 연동

| 조건 | 발송 여부 |
| --- | --- |
| 타인이 내 작품에 좋아요 | ✅ 인앱 알림 발송 |
| 자신이 자신 작품에 좋아요 | ❌ 알림 없음 |

> 알림 문구: "OO님이 내 작품을 좋아합니다"
> 

## 포인트 연동

> Phase 1에서는 좋아요 받기에 대한 AP 적립 없음. 단, 피드 큐레이션 가중치 신호로는 활용됨.
> 

## 개발 스펙

**Like 테이블**

```
Like (
  id,
  user_id,
  work_id,
  created_at
)
```

| 항목 | 내용 |
| --- | --- |
| 중복 방지 | (user_id + work_id) Unique 제약 |
| 카운트 | Like 테이블 COUNT 또는 [works.like](http://works.like)_count 캐싱 (개발자 결정) |
| 비로그인 조회 | 좋아요 수는 노출, 좋아요 여부(filled 상태)는 미표시 |

## 미결 사항

- [x]  자신의 작품 좋아요 가능 여부 → **가능** ✅ (제한하면 예외 처리 복잡도 증가 대비 실익 없음)
- [x]  비로그인 좋아요 클릭 → **로그인 유도 모달** ✅
- [x]  좋아요 알림 기본값 → **ON** ✅ (콘텐츠 운영 정책 알림 운영 정책 표 참조)
- [x]  좋아요 수 카운트 방식 → **PM 추천: [works.like](http://works.like)_count 캐싱** ✅ (실시간 COUNT는 트래픽 증가 시 부담. 캐싱 + 비동기 반영이 성능상 유리. ⚠️ 개발자 최종 결정 필요)
