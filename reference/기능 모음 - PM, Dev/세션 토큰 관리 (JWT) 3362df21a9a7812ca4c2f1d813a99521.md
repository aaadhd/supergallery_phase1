# 세션 / 토큰 관리 (JWT)

## 개요

로그인 성공 후 Access Token과 Refresh Token을 발급하여 로그인 상태를 유지하는 기능. 유저가 인지하지 못하는 상태에서 자동으로 토큰이 갱신된다.

## 진입점

- 로그인 완료 시점 (소셜 로그인 / 이메일 로그인 모두 해당)

## 동작 정책

| 케이스 | 동작 |
| --- | --- |
| Access Token 만료 | Refresh Token으로 자동 갱신 (유저 인지 없이 처리) |
| 403 응답 | ?redirect= 파라미터 포함하여 로그인 후 원래 페이지 복원 |
| 멀티 디바이스 로그인 | 동일 계정 중복 로그인 허용 |
