# 에러 페이지 (404/500)

## 개요

존재하지 않는 URL 접근(404) 또는 서버 오류(500) 발생 시 유저에게 친절한 안내 화면을 노출하고 홈으로 복귀할 수 있도록 하는 기능.

## 진입점

- 존재하지 않는 URL 직접 접근 시 (404)
- 서버 오류 발생 시 (500)
- 삭제된 전시·작품·프로필 URL 접근 시 (404)
- 정지된 계정 프로필 접근 시 (403)

## 동작 정책

| 에러 유형 | 노출 내용 |
| --- | --- |
| 500 Server Error | "일시적인 오류가 발생했어요" 메시지 + 홈으로 버튼 + 새로고침 버튼 |
| 공통 | GNB 유지, 로그인 상태 유지, 홈으로 버튼 필수 포함 |

## 관련 EPIC

N: 법적페이지/공통UI

## 개발 스펙

- 에러 페이지는 서버 사이드 렌더링 (SEO 대응)
- 에러 페이지 URL: /404, /500 별도 라우트 또는 Next.js 기본 에러 핸들링 (Ralph 결정)
- 브랜드 톤에 맞는 일러스트 또는 텍스트 UI (PD 박금지 디자인)
