# 업로드 파일 검증 (사진 차단)

관련 화면: 작업 업로드

> 작품 업로드 시 카메라로 촬영한 사진 파일을 자동으로 차단하는 기능.
> 

> PRD ART-006 기반. 디지털 드로잉·창작물 중심 플랫폼 특성 유지를 위한 품질 관리.
> 

---

## 개요

업로드된 이미지의 EXIF 메타데이터를 분석하여 카메라(스마트폰 포함)로 촬영된 사진을 감지하고 업로드를 차단.

## 진입점

- 작업 업로드 > 이미지 파일 선택 또는 드래그앤드롭 시

## 차단 로직

| 항목 | 내용 |
| --- | --- |
| 분석 대상 | 업로드된 이미지의 EXIF 메타데이터 |
| 차단 기준 | `Make`, `Model` (카메라 제조사/기종) 필드 존재 시 차단 |
| 차단 대상 예시 | iPhone, Samsung Galaxy, SONY, Canon, Nikon 등으로 촬영된 파일 |
| 허용 대상 | Procreate, Photoshop, Illustrator 등 디지털 드로잉 툴 생성 파일 |
| 처리 시점 | 파일 선택/드롭 즉시 (서버 업로드 전 클라이언트 사이드 처리 권장) |

## 에러 처리

| 케이스 | 안내 문구 |
| --- | --- |
| 카메라 촬영 사진 감지 | "카메라로 촬영한 사진은 업로드할 수 없어요. 디지털 드로잉 작품을 올려주세요." |
| EXIF 분석 불가 파일 | 업로드 허용 (EXIF 없는 경우 차단하지 않음) |

## 예외 처리

- EXIF 데이터가 없는 파일 → 허용 (스크린샷, 편집된 파일 등 포함)
- EXIF는 있으나 카메라 필드 없는 파일 → 허용
- 운영팀 판단 하에 예외 허용 가능 (어드민에서 수동 처리)
