# 그룹명 처리 정책

관련 화면: 세부 정보 모달 (업로드), 작업 업로드

> 업로드 시 그룹명 입력에서 오타·띄어쓰기 차이로 그룹이 분리되는 문제를 방지하는 기능 로직 정리.
> 

> 화면 스펙은 **화면 모음 > 세부 정보 모달 (업로드)** 참조.
> 

---

## 배경

그룹명은 별도 팀 기능 없이 사용자가 직접 입력하는 텍스트 필드이므로, 동일한 그룹이라도 입력 방식에 따라 다른 그룹으로 분리될 수 있음.

> 예: "성동복지관 14기" vs "성동복지관14기" → 그대로 두면 2개의 다른 그룹으로 집계됨
> 

---

## 그룹명 처리 3원칙

### 1. 최근 사용 그룹명 자동 입력

- 사용자가 마지막으로 사용한 그룹명이 필드에 기본값으로 노출
- 매번 다시 입력할 필요 없이 바로 전시 올리기 가능

### 2. 자동완성 드롭다운

- 타이핑 시작 시 내가 이전에 사용한 그룹명 + 서비스 내 현재 존재하는 전체 그룹명 목록 드롭다운 제공
- 기존 그룹명 선택 유도해 오타리·띄어쓰기 차이로 인한 그룹 분리 방지

### 3. 그룹명 정규화 (내부 로직)

- 저장 시 띄어쓰기·특수문자 제거 + 대소문자 통일 후 동일 그룹 여부 판단
- 표시는 최초 입력자의 원본 표기 방식 유지

| 케이스 | 처리 |
| --- | --- |
| "성동복지관 14기" (최초 입력) | 그룹 생성. 표시명: "성동복지관 14기" |
| "성동복지관14기" (이후 입력) | 동일 그룹으로 집계. 표시명은 최초 입력값 유지 |
| 자동완성에서 선택 | 동일 그룹으로 확실히 연결 |

---

## 미결 사항

- [x]  자동완성 드롭다운 기준 → **내 이전 그룹명 + 서비스 전체 그룹명 포함** ✅
- [x]  정규화 처리 범위 → **띄어쓰기 + 특수문자 제거 + 대소문자 통일** ✅ (서비스 내 표시는 최초 입력자의 원본 표기 유지)
