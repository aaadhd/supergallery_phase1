## 포인트

- 로그인 유저만 임시저장 가능
- 비로그인 상태에서 클릭 시 로그인 유도
- 사전 조건으로 `work_id` 무관하게 draft 상태로 저장

## 동작 정의

| 케이스 | 동작 |
| --- | --- |
| 업로드 중 임시저장 버튼 클릭 | 현재까지 입력한 정보 draft로 저장 + "임시저장되었습니다" 토스트 |
| 페이지 이탈 or 앱 종료 | **자동 임시저장 없음** — 화면 닫기 시만 저장 |
| 초안 탭에서 이어쓰기 클릭 | 업로드 에디터 진입 + 저장된 데이터 복원 |
| 잔여 유효기간 | 저장 후 **30일** (이후 자동 삭제) |
| 초안 최대 보관 수 | 1인당 **10개** 제한 |

## 초안 탭 구성 (내 프로필)

| 항목 | 내용 |
| --- | --- |
| 노출 조건 | 로그인한 본인에게만 노출 |
| 카드 내용 | 전시명 (unnamed 지정 시 "제목 없음") + 이미지 썸네일 + 저장 일시 |
| 액션 | 이어쓰기 / 삭제 |
| 빈 상태 | "작성 중인 초안이 없어요. 작품을 올리면 여기에 보관됩니다." |

## 개발 스펙

```
Draft (
  id,
  user_id,
  title?,
  cover_image_url?,
  images: JSON[],
  metadata: JSON,
  created_at,
  updated_at,
  expires_at   -- 저장 후 30일
)
```

## 미결 사항

- [x]  임시저장 지원 여부 → **지원** ✅ (PRD In Scope)
- [x]  임시저장 유효기간 → **30일** ✅ (어뷰징 방지 위한 보관만 허용. 더 길면 DB 리소스 낭비)
- [x]  초안 최대 보관 수 → **10개** ✅
- [x]  자동 저장 여부 → **없음** ✅ (명시적 저장만. 시니어 유저에게 자동 저장은 백그라운드 동작이라 혹시라도 편집 내용이 사라지는 상황 발생 방지)
