## 검토 프로세스

| 단계 | 내용 |
| --- | --- |
| 1. 대기 | 작품 업로드 완료 → 검토 대기 상태로 어드민 목록에 노출 |
| 2. 검토 | 운영팀이 전시명, 이미지, 작가 정보, 스팸/부적절 여부 확인 |
| 3. 통과 | 둘러보기 피드 풀 편입 → 랜덤 노출 |
| 4. 거절 | 피드 미노출 + 작가에게 안내 알림 발송 |

## 검토 SLA

| 유형 | 기한 |
| --- | --- |
| 일반 업로드 | 업로드 후 **24시간 이내** |
| 불법/성인 콘텐츠 | **즉시** 처리 |

## 거절 사유 분류

| 사유 | 설명 |
| --- | --- |
| 저품질 | 해상도 극히 낮음, 작품으로 보기 어려움 |
| 스팸/광고 | 상업적 홍보 목적 콘텐츠 |
| 부적절 | 폭력, 성인, 혐오 콘텐츠 |
| 저작권 침해 의심 | 명백히 타인 작품 무단 도용 |

## 피드 노출 로직

| 구분 | 노출 방식 |
| --- | --- |
| Artier's Pick | 둘러보기 피드 상단 고정 |
| 일반 통과작 | 랜덤 배정 (추천 알고리즘 가중치 적용) |
| 본인 프로필 | 검토 전이라도 업로드 즉시 노출 |

## 개발 스펙

| 항목 | 내용 |
| --- | --- |
| 전시 상태값 | `pending` / `approved` / `rejected` |
| 기본값 | 업로드 직후 `pending` |
| 피드 쿼리 조건 | `status = approved` 인 전시만 둘러보기에 노출 |
| 거절 알림 | 작가에게 인앱 알림 + 이메일 알림 발송 |

## 유저 대면 상태 표시 (Phase 1)

> 업로드 후 둘러보기에 안 보이는 걸 유저가 "고장"으로 오해하지 않도록, 검수 상태를 명확하게 안내.
> 

> 특히 시니어 유저의 첫 업로드 경험이 가장 불안한 순간이므로, 이때 한 번 명확히 알려주면 CS를 많이 줄일 수 있음.
> 

### 업로드 완료 시 안내 메시지

| 항목 | 내용 |
| --- | --- |
| 노출 시점 | 업로드 완료 시 ("게시" 버튼 클릭 후) |
| 형식 | 토스트 또는 완료 화면 안내문 |
| 한국어 | "작품이 등록되었습니다. 운영팀 검토 후 둘러보기에 노출됩니다 (최대 24시간)" |
| 영어 | "Your work has been submitted. It will appear in Browse after review (up to 24 hours)" |

### 내 프로필 작품 카드 상태 라벨

| 상태값 | 라벨 (한) | 라벨 (EN) | 스타일 |
| --- | --- | --- | --- |
| `pending` | 검수 중 | Under Review | 회색 배경, 작은 텍스트 |
| `approved` | (라벨 없음) | (라벨 없음) | 정상 카드 |
| `rejected` | 게시 불가 | Not Approved | 빨간 배경, 작은 텍스트 |

> `pending` 라벨은 본인 프로필에서만 보임. 다른 유저에게는 노출되지 않음 (approved 전이므로 둘러보기에도 안 보임).
> 

> `rejected` 카드 클릭 시 거절 사유 안내 모달 노출. 사유 공개 여부는 미결 사항.
>
