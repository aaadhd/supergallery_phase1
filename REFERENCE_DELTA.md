3. 이 파일 **§2 파일별 대조표**에서 해당 파일 번호를 찾는다.

### 공통 교정 — 원문에 넣을 수 있는 표준 문장 (필요 시 발췌)

- **인프라 한계(Phase 1 데모)**: 「소셜 OAuth 실연동, 이메일·SMS·카카오 실발송, 서버 DB·OG 동적 생성, 실시간 푸시는 구현 범위外. 클라이언트는 UI·모의 로그·localStorage 영속으로 동작한다.」 (근거: IMPLEMENTATION_DELTA §7)
- **검색어 저장**: 「로그인 사용자 최근 검색어는 명세상 서버 저장이나, Phase 1 구현은 브라우저 localStorage(계정별 키)이다.」 (§8.1)
- **배너**: 「배너 노출 정책·기간 필드는 구현하나 영속 저장은 localStorage 기반이다.」 (§8.4)
- **강사 표시**: 「강사 여부는 프로필 토글이 아니라, ‘강사로 올리기’로 표시된 작품 업로드 이력이 있을 때 자동 표시된다.」 (§3.3, §8.7-3)
- **확인 창**: 「삭제·파괴적 작업 확인은 브라우저 기본 confirm이 아니라 앱 내 ConfirmDialog를 쓴다.」 (§8.7-4)
- **작품 카드**: 「피드·프로필 카드에서 좋아요·저장은 숫자가 아니라 상태 아이콘만 표시한다(댓글 수 표시 없음).」 (§8.7-5, PRD §2.2)
- **전시 초대 링크**: 「공유 링크는 `?from=invite` 쿼리를 포함할 수 있으며, 수신자는 전시 초대 랜딩(그리드·비로그인 CTA)으로 진입한다.」 (§8.7-6)
- **이벤트·피드**: 「이벤트 목록·상세·어드민은 단일 eventStore를 따른다. 둘러보기 피드는 Artier's Pick·테마전·추천 작가·신규·일반 버킷과 팔로우 가중치를 사용한다.」 (§8.5·§8.6)
