## 1. 알림 채널 정책

### 1.1 한국 사용자 (전화번호 +82)

| 용도 | 1순위 | 폴백 | 비고 |
|------|-------|------|------|
| 일반 알림 (좋아요, 팔로우, 초대, 검수 결과 등) | 카카오 알림톡 | SMS | |

## 2. 가입/본인인증 정책

### 2.1 필수 수집 정보

| 항목 | 필수 여부 | 검증 방법 |
|------|----------|----------|
| 전화번호 | 필수 | 본인인증 API (PASS 등) |

## 3. 비가입자 초대 정책

### 3.1 발송 시점

- 업로드 시 즉시 발송하지 **않는다**
- 전화번호를 Work의 `imageArtists`에 보관
- **검수 승인(approved) 시점에 발송**
- 검수 반려 시 보류된 초대 폐기

### 3.2 삭제 시 처리

- 초대가 보류된 작품 삭제 시: "초대된 비회원이 있습니다. 삭제하면 초대가 무효화됩니다." 경고
- 삭제 자체는 차단하지 않음 (게시자 자유)
- SMS 발송 후 작품 삭제 시: 랜딩 페이지에서 "이 전시는 삭제되었습니다. Artier에서 다른 작품을 둘러보세요." 안내 + 가입 CTA 유지

### 3.3 채널

- 한국 번호: 카카오 알림톡 → SMS 폴백
- 해외 번호: 이메일

## 8. 초안 저장 정책

- **자동 저장 없음.** 수동 "초안 저장" 버튼 클릭 시에만 저장.
- Upload 진입 시 기존 초안이 있으면 "저장된 초안이 있습니다" 토스트 + "이어서 작업" 액션.
- 초안은 프로필 → "초안" 탭에서 관리.
- 작품 올리기 화면에서 나가려 할 때 "초안으로 저장할 수 있습니다" 안내 표시.

## 10. 공유 정책

- 공유 버튼은 1개. "공유하기"를 누르면:
  - 모바일: 기기에 설치된 앱(카톡, 문자, 라인 등) 선택 화면이 뜸
  - PC: 초대 멘트 + 링크가 클립보드에 복사됨
- 공유 시 전달되는 내용: "[작가명] 님이 [전시명] 전시에 초대합니다. Artier에서 작품을 감상해 보세요. [링크]"
- 링크만 단독으로 보내지 않음. 항상 멘트와 함께 전달.

## 11. 카메라 사진 차단 정책

- 순수미술 플랫폼이므로 카메라로 촬영한 사진(스마트폰/DSLR)은 업로드 차단.
- 판별 방법: 이미지 파일의 실제 데이터(magic bytes)를 읽어 JPEG 여부를 확인한 뒤, EXIF 메타데이터에서 카메라 제조사/모델 정보가 있으면 차단.
- 파일 이름이나 확장자가 아닌 실제 파일 내용으로 판별하므로 확장자 변경으로 우회 불가.
- PNG, WebP, GIF 등 비-JPEG 포맷은 EXIF 구조가 없으므로 차단 대상 아님.
- 차단 시에는 업로드 화면에 인라인 안내("Artier는 디지털 드로잉 작품 전용 플랫폼입니다")가 표시되며, 파일은 추가되지 않는다.

## 12. 신고·제재 정책

### 12.1 신고 처리 액션

운영팀은 접수된 신고에 대해 다음 4가지 액션 중 하나를 선택한다.

| 액션 | 효과 |
|------|------|
| 삭제 | 전시 신고 한정. 해당 전시를 영구 삭제한다. |
| 경고 | 신고 대상 작가의 경고 카운트에 1회 누적. |
| 기각 | 신고를 부당하다고 판정. 신고자의 허위 신고 카운트에 1회 누적. |
| 비공개 | 전시를 둘러보기·검색에서 제외. 작가 본인의 프로필에서는 계속 보인다. |

> 좋아요·저장·픽·신고·배지는 모두 **전시(Work 컨테이너) 단위**에 붙는다. 개별 이미지(작품 1장) 단위의 인터랙션·배지는 없다.

### 12.2 자동 정지 승격 (전체 사용자 적용)

- **경고 3회 누적** → 대상 작가 **7일 정지** 자동 적용.
- **허위 신고 3회 누적** → 신고자 **7일 차단** 자동 적용.
- 자동 정지는 특정 시범 계정이 아니라 **가입한 모든 사용자에게 동일하게 발동**한다.
- 정지된 사용자는 다음 로그인 시점에 사유와 이의제기 안내를 받으며, 해제 시점까지 서비스 이용이 차단된다.

### 12.3 정지 단계

주의 / 7일 정지 / 30일 정지 / 영구 정지.
"주의"는 실제 서비스 차단은 없으며 안내 목적의 표시 단계이고, 누적 시 시한부 정지로 승격될 수 있다.

### 12.4 신고자 보호

- 동일 신고자가 동일 대상을 반복 신고하는 것은 차단된다 (신고자 + 대상유형 + 대상 ID 조합당 1회).
- 신고한 전시/작가는 해당 신고자에게만 자동으로 숨김 처리된다.

## 13. 업로드 유형·역할 정책

### 13.1 업로드 유형

업로드 화면의 첫 단계에서 아래 둘 중 하나를 선택한다.

| 유형 | 설명 |
|------|------|
| 혼자 올리기 | 내 작품만으로 구성된 전시 |
| 함께 올리기 | 동호회·클래스·지인 등 여러 작가의 작품을 한 전시에 모음 |

### 13.2 함께 올리기의 역할 선택

"함께 올리기"를 선택하면, 이미지를 추가하기 전에 **본인 포함 여부**를 먼저 결정한다.

| 역할 | 본인 작품 포함 | 대표 사례 |
|------|---------------|----------|
| 참가자 | 최소 1점 필수 | 동호회·클래스·지인 등 본인도 참여하는 그룹 전시 |
| 강사 | 포함 금지 | 수강생 작품만 대신 올리는 경우 |

- 참가자로 진행하면서 본인 작품 없이 전시를 등록하는 것은 차단된다.
- 강사로 진행하는 경우 작가 선택 목록에서 본인은 자동으로 제외되며, 이미 본인으로 지정된 이미지 슬롯은 역할이 바뀔 때 자동 초기화된다.
- 업로드 화면 상단에 현재 역할(그룹 전시 / 그룹 전시(강사))이 뱃지로 표시된다.

### 13.3 강사 표시 파생

- 강사 역할로 업로드한 이력이 한 건이라도 있으면, 프로필에 "수강생 작품" 탭이 자동으로 노출된다.
- 강사 여부를 프로필에서 수동으로 켜고 끄는 토글은 제공하지 않는다 (상태 불일치를 방지하기 위함).
- 해당 작품을 모두 삭제하면 탭은 자동으로 사라진다.

## 14. 커버 이미지 정책

- 전시 업로드 시 업로드한 커버 이미지는 **전시의 첫 장 표지**로 사용된다.
  - 둘러보기 피드의 썸네일로도 동일한 이미지가 사용된다.
- 커버 이미지는 전시 이미지 배열에 포함되지 않으므로, 작품 이미지 번호(예: 1/10)에는 영향을 주지 않는다.
- 커버 이미지를 별도로 지정하지 않으면 전시 이미지 중 선택된 대표 이미지(없으면 첫 이미지)가 썸네일·표지로 사용된다.
- 작품 상세 화면에서 커버 슬라이드는 블랙 배경 + 전시명 라벨로 표시되며, 작가 정보 오버레이는 커버에는 표시되지 않는다.

## 15. 큐레이션·배지 정책

### 15.1 큐레이션 개념 4종

Artier의 큐레이션 체계는 아래 4가지 개념으로 구분된다. 각각 목적·수명·공개 여부가 다르다.

| # | 개념 | 설명 | 공개 여부 | 배지 |
|---|------|------|----------|------|
| 1 | **Artier's Pick** | 운영팀이 매주 선정하는 추천 전시 (최대 10개, 교체) | 공개 | 영구 |
| 2 | **기획전** | 운영팀이 이미 존재하는 여러 전시를 주제·맥락으로 엮어 내보내는 컬렉션 (그룹전·작가전·특집전 포괄) | 공개 | 없음 |
| 3 | **이벤트** | 운영팀이 주제를 제시하고 사용자가 응모 → 운영팀이 선정하는 공모전. 주간 공모형 테마전도 이벤트의 한 종류 | 공개 | 선정작에만 영구 부여 |
| 4 | **운영 내부 컬렉션** | 좋은 전시·작가·그룹을 운영팀이 내부 북마크 (발굴·추적 용도) | **비공개** | 없음 |

### 15.2 배지 체계

공개되는 전시 배지는 아래 3종뿐이다. (응모작에는 배지 없음.)

| 배지 | 발생 | 수명 |
|------|------|------|
| ★ Artier's Pick | Pick 선정 이력 (한 번이라도 받았으면) | 영구 |
| 이벤트 선정작 | 이벤트·공모형 테마전에서 선정된 이력 | 영구 |
| 강사 | 강사 역할로 업로드된 전시 | 발행 정책으로 결정 (§13.3) |

본인에게만 보이는 상태 배지(검수 "확인 중"·"게시 불가")는 위의 공개 배지와 별개로 존재한다.

**배너 라벨은 제공하지 않는다.** 배너는 내용·기간·활성화만으로 역할이 충분하므로 NEW/HOT/EVENT/PICK 같은 부속 라벨을 두지 않는다.

### 15.3 운영 기준

**Pick과 기획전의 경계**

Pick은 "1개 전시씩 독립 선정", 기획전은 "3개 이상을 주제로 묶어 소개"를 원칙으로 한다. 이 경계를 넘어서는 기획(예: "이번 주 봄 풍경 10선")은 기획전으로 운영한다. 운영자 간 일관성 유지가 핵심.

**Pick 영구 배지 인플레이션 모니터링**

주간 10개 기준 연간 최대 500개가 영구 Pick 배지를 받는다. 2~3년 운영 시 배지 의미 희석 우려가 있으므로 **분기 단위로 Pick 배지 누적 수치를 점검**하고, 필요 시 "연도별 Pick" 구분자 도입 또는 선정 기준 재조정을 검토한다.

**이벤트 선정작 배지 세분화 여지**

Phase 1에서 이벤트 선정작 배지는 단일 라벨로 통합한다. Phase 2 이후 이벤트 브랜드가 정착하면(예: "Artier Awards"), 구현 단계에서 배지에 `eventId` 같은 연결 정보를 포함해 두어 나중에 이벤트별 라벨 세분화가 가능하도록 한다.

### 15.4 이벤트 선정 시스템 (Phase 1 후순위 TODO)

이벤트 선정작 배지 원칙은 확정되어 있으나 구현은 Phase 1 내 후순위. 구현 시 포함되어야 할 것:

- 어드민 이벤트 상세에 "선정 작품 관리" 섹션 추가 (응모작 → 선정 체크)
