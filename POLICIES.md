## 1. 알림 채널 정책

### 1.1 한국 사용자 (전화번호 +82)

| 용도 | 1순위 | 폴백 | 비고 |
|------|-------|------|------|
| 일반 알림 (좋아요, 팔로우, 초대, 검수 결과 등) | 카카오 알림톡 | SMS | |
| 비밀번호 재설정 | 카카오 알림톡 | SMS | |
| 법적 의무 알림 (약관 변경, 개인정보처리방침 변경 등) | 이메일 | — | 법적 기록 보존 목적 |

**한국 사용자에게 이메일은 법적 의무 알림 외에 사용하지 않는다.**

### 1.2 해외 사용자 (전화번호 +82 외)

| 용도 | 채널 | 비고 |
|------|------|------|
| 모든 알림 | 이메일 | 알림톡/SMS 미지원 |

### 1.3 구현 위치

- 채널 라우팅: `src/app/utils/inviteMessaging.ts` — `pickChannel()`
- 채널 타입: `InviteChannel = 'kakao_alimtalk' | 'sms' | 'email'`

## 2. 가입/본인인증 정책

### 2.1 필수 수집 정보

| 항목 | 필수 여부 | 검증 방법 |
|------|----------|----------|
| 전화번호 | 필수 | 본인인증 API (PASS 등) |
| 실명 | 필수 | 본인인증 시 자동 수집 |
| 생년월일 | 필수 | 본인인증 시 자동 수집 → 만 14세 검증 |
| 이메일 | 선택 (한국) / 필수 (해외) | 해외 사용자는 이메일이 주 알림 채널 |
| 닉네임 | 필수 | 2~20자, 비속어 불가 |

### 2.2 만 14세 검증

- 한국: 본인인증 결과에서 자동 검증
- 해외: 가입 시 생년월일 입력 + `meetsMinAge()` 검증
- 소셜 가입: 본인인증 API 연동 후 자동 처리 (현재는 미구현 상태)

## 3. 비가입자 초대 정책

### 3.1 발송 시점

- 업로드 시 즉시 발송하지 **않는다**
- 전화번호를 Work의 `imageArtists`에 보관
- **검수 승인(approved) 시점에 발송**
- 검수 반려 시 보류된 초대 폐기

### 3.2 삭제 시 처리

- 초대가 보류된 작품 삭제 시: "초대된 비회원이 있습니다. 삭제하면 초대가 무효화됩니다." 경고
- 삭제 자체는 차단하지 않음 (게시자 자유)
- SMS 발송 후 작품 삭제 시: 랜딩 페이지에서 "이 전시는 삭제되었습니다. Artier에서 다른 작품을 둘러보세요." 안내 + 가입 CTA 유지

### 3.3 채널

- 한국 번호: 카카오 알림톡 → SMS 폴백
- 해외 번호: 이메일

## 6. Phase 정의

| Phase | 범위 |
|-------|------|
| Phase 1 | 현재 서비스 전체. 프론트 + 백엔드 + 인프라 포함하여 **시장 출시까지** |
| Phase 2 | 2차 오픈 범위 (마켓, 아카데미 등 신규 기능) |

> "Phase 2에서 처리"는 출시 후 2차 오픈을 의미한다. 출시 전 마무리 작업은 Phase 1에 포함된다.

---

## 7. 포인트 노출 정책

- Phase 1: **프론트에 포인트 UI 미노출**. 백그라운드 적립만. 어드민에서만 확인 가능.
- `/points` 라우트는 `/`로 리다이렉트 유지.
- 포인트 적립 토스트도 사용자에게 미노출 (백그라운드 로그만).

---

## 8. 초안 저장 정책

- **자동 저장 없음.** 수동 "초안 저장" 버튼 클릭 시에만 저장.
- Upload 진입 시 기존 초안이 있으면 "저장된 초안이 있습니다" 토스트 + "이어서 작업" 액션.
- 초안은 프로필 → "초안" 탭에서 관리.
