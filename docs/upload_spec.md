## 1. 핵심 설계 원칙

- 업로드 시작 시 **유형(혼자/함께)을 먼저 선택**해 불필요한 필드를 처음부터 숨긴다.
- **혼자 올리기**: 전시명 하나만 필수 — 최대한 단순하게.
- **함께 올리기**: 우측 고정 패널로 연속 입력 — 반복 동작 최소화.
- 시니어 친화: 스프레드시트식 UI, 다중선택(Shift+Click), Tab 이동 방식 사용 금지.
- 확인·경고는 앱 내 확인 다이얼로그만 사용 (브라우저 네이티브 confirm/alert 금지).

## 2. Step 0 — 유형 선택

업로드 화면 진입 시 가장 먼저 표시. 이미지 업로드 전에 유형을 선택하게 한다.

```
┌─────────────────────────────────┐
│  [ 🖼 혼자 올리기 ]              │
│  내 작품을 올려요                │
│  개인 작품 업로드                │
├─────────────────────────────────┤
│  [ 👥 함께 올리기 ]              │
│  여러 명의 작품을 함께 올려요    │
│  동호회·클래스·친구들과 함께     │
└─────────────────────────────────┘
```

| 항목 | 혼자 올리기 | 함께 올리기 |
|---|---|---|
| 메인 카피 | 내 작품을 올려요 | 여러 명의 작품을 함께 올려요 |
| 서브 카피 | 개인 작품 업로드 | 동호회·클래스·친구들과 함께 |
| 대상 | 개인 작가 | 강사, 동호회, 친구 그룹 등 |

### 정책

- 유형 미선택 시 이미지 업로드 불가.
- 유형 변경 시 → 선택 화면으로 돌아가기 + 추가된 이미지 초기화.
  - 이미 추가된 이미지가 있으면 앱 확인 다이얼로그로 **손실 경고**.
- "함께 올리기"는 강사 계정 전용이 아니며 **누구나 선택** 가능.
- 강사 표시 여부는 함께 올리기 세부정보 모달의 체크박스에서 결정 (§5 참조).

## 3. Step 1 — 이미지 업로드 + 상단 고정 바

### 상단 고정 바

**혼자 올리기:**
```
[ 전시 제목을 입력하세요. * ]
```

**함께 올리기:**
```
[ 전시 제목을 입력하세요. * ]   [ 그룹명 * ]
```

- **전시명**: 필수. 최대 20자. 자동완성 드롭다운 **없음** (직접 입력만).
- **그룹명**: 함께 올리기에서만 노출, **필수**, 최대 20자. 과거 사용한 그룹명 기반 자동완성 제안.
- 이미지 업로드 중에도 입력 가능 (상단 고정).
- **비속어 검사**: 발행 시 전시명·그룹명에 대해 사전 기반 필터 적용 → 걸리면 토스트 안내 후 발행 중단.

### 이미지 업로드 캔버스

| 규칙 | 내용 |
|---|---|
| 업로드 방식 | 드래그&드롭, 파일 선택 |
| 최대 장수 | 전시당 10장 |
| 포맷 | JPG · JPEG · PNG · WEBP · GIF |
| 용량 | 10MB/장 |
| 최소 해상도 | 단변 **800px 이상** (갤러리 품질 보호) |
| GIF | 업로드 시 WEBP로 자동 변환 |
| 카메라 사진 | **업로드 차단** (EXIF 분석으로 판정) |

### 작품 톤 배경 "묻어나는" 효과

- 원본 이미지를 **배경 레이어로 깔고 blur·scale·opacity**만 적용해 작품 톤이 주변에 번져 보이는 효과를 낸다.
- 별도 색상 추출 알고리즘 불필요. 순수 CSS로 구현.

## 4. Step 1 우측 패널

### 혼자 올리기

- 이미지 클릭 시 우측 패널 선택적 오픈.
- **작품명** 입력만 표시 (선택. 미입력 시 "무제").
- 작가명 섹션 없음 (본인이므로 불필요).

### 함께 올리기 (고정 오픈)

- 이미지 업로드 완료 시 자동으로 첫 번째 이미지 선택 + 패널 오픈.
- 패널 항상 열려 있음 (닫기 버튼 없음).
- 좌측 이미지 클릭 → 해당 이미지로 패널 전환.
- 패널 내부에 **이전/다음** 버튼으로 이미지 간 순회.

**패널 구성:**
```
┌──────────────────────────┐
│ [이미지 썸네일]           │
│ 3 / 10                   │
├──────────────────────────┤
│ 작품명 (선택)             │
│ [                      ] │
│ 미입력 시 "[작가명]의 작품 N" 자동 생성 │
├──────────────────────────┤
│ 작가명 * (필수)           │
│ [작가명 또는 닉네임 검색 🔍] │
│  ↓ 결과 있으면 → 선택     │
│  ↓ 결과 없으면 → [직접 입력] 버튼 노출 │
│       → 이름 + 연락처 입력│
├──────────────────────────┤
│ [← 이전]        [다음 →] │
└──────────────────────────┘
```

### 작가명 입력 UX

- **검색창 하나로 통합** — 회원/비회원 구분 탭 없음.
- 검색 결과 있으면 → 선택 (회원 지정).
- 검색 결과 없을 때만 → "직접 입력" 버튼 노출.
  - 직접 입력 시 **실명 + 연락처 둘 다 필수**.
- 연락처 미입력 시 발행 불가.
- 본인도 검색 결과에 포함 (강사가 본인 작품을 함께 올리는 경우 대비).

## 5. Step 2 — 세부 정보 모달

### 포함 항목

- **커버 이미지**: 첫 번째 이미지 자동 적용, 모달에서 수동 변경 가능.
- **원작 확인 체크박스** (필수, 미체크 시 발행 불가. 문구 분기):
  - 혼자 올리기: "본인이 직접 창작한 원작임을 확인합니다"
  - 함께 올리기: "업로드하는 작품들에 대한 작가의 동의를 받았습니다"
- **이벤트 연결 표시**: 이벤트 업로드로 진입한 경우 자동 연결 상태 표시.
- **강사 체크박스** (함께 올리기에서만 노출) — §강사 표시 정책 참조.

### 제외 항목 (Phase 2로 보류)

- 카테고리 선택
- 태그 입력
- 수강생 이메일 태그 섹션

### 강사 표시 정책 (단일 진입점)

- 함께 올리기 세부정보 모달의 **"저는 강사예요" 체크박스**가 시스템 전체에서 강사 여부를 결정하는 **유일한 진입점**.
- 기본값 OFF.
- 체크 시 해당 작품에 강사 플래그가 영구 저장됨.
- 프로필 화면은 업로드 이력에서 자동 파생:
  - 한 작품이라도 강사 플래그로 발행 → 프로필에 "수강생 작품" 탭 자동 노출.
  - 모든 해당 작품 삭제 → 탭 자동 비노출.
- **프로필 편집 화면에는 강사 토글을 두지 않는다** (상태 불일치 방지).

## 6. 콘텐츠 편집기 — 이미지 블록 툴바

### 단일 이미지 블록

**업로드 전 툴바:**

| 아이콘 | 툴팁 | 동작 |
|---|---|---|
| ↕ | 콘텐츠 재정렬 | 블록 순서 변경 |
| 🗑 | 콘텐츠 삭제 | 블록 삭제 |

**업로드 후 툴바:**

| 아이콘 | 툴팁 | 동작 | 조건 |
|---|---|---|---|
| →← | 패딩 제거 | 좌우 여백 → 전폭 토글 | 콘텐츠 가로 1100px 이상에서만 활성 |
| 🖼 | 이미지 변경 | 새 이미지로 교체 | — |
| ↕ | 콘텐츠 재정렬 | 블록 순서 변경 | — |
| 🗑 | 콘텐츠 삭제 | 블록 삭제 | — |

> 패딩 제거 비활성 안내: "가로 폭 1100px 이상의 콘텐츠만 사용 가능한 기능입니다"

### 이미지 그리드 블록

**업로드 후 툴바:**

| 아이콘 | 툴팁 | 동작 |
|---|---|---|
| ↔ | 패딩 추가/제거 | 기본 없음 ↔ 있음 (단일 블록과 반대) |
| 🖼 | 그리드 편집 | "이미지 그리드 편집" 모달 오픈 |
| ↕ | 콘텐츠 재정렬 | 블록 순서 변경 |
| 🗑 | 콘텐츠 삭제 | 그리드 블록 전체 삭제 |

### 이미지 그리드 편집 모달

| 항목 | 내용 |
|---|---|
| 모달 제목 | 이미지 그리드 편집 |
| 각 셀 | 우상단 ✕ 버튼 → 셀 개별 삭제 |
| 이미지 추가 | 우상단 "+ 이미지 파일 추가" 버튼 |
| 하단 버튼 | 취소 / 완료 |

### 전역 설정

| 설정 | Phase 1 | 기본값 |
|---|---|---|
| 배경색상 전역 슬라이더 | 없음 (톤 묻어남은 blur 효과로 대체) | — |
| 콘텐츠 간격 | 10px 고정 | 10px |
| 콘텐츠 재정렬 | 지원 (드래그 앤 드롭) | — |

### 콘텐츠 재정렬 모드

- 2~4열 그리드 뷰로 전환.
- 드래그 앤 드롭으로 순서 변경.
- 완료 시 기존 편집 뷰로 복귀.

## 7. 파일 업로드 정책

| 대상 | 허용 포맷 | 최대 용량 | 최대 수량 | 최소 해상도 | 비고 |
|---|---|---|---|---|---|
| 이미지 블록 (단일) | JPG·JPEG·PNG·WEBP·GIF | 10MB/장 | 전시당 10장 합산 | 단변 800px 이상 | GIF → WEBP 자동 변환 |
| 이미지 그리드 | JPG·JPEG·PNG·WEBP·GIF | 10MB/장 | 그리드당 10장 (합산 이내) | 단변 800px 이상 | 동상 |
| 커버 이미지 | JPG·JPEG·PNG·WEBP | 5MB | 1장 | 760×760px 권장 | GIF 불가 |

- 단변 800px 미만: 업로드 차단.
- 카메라 사진: 업로드 차단 (EXIF 분석).
- 발행 진행 상태를 보여주는 별도 프로그레스 바 UI는 없음.

## 8. 발행 검증 로직

### 혼자 올리기

- 이미지 0장 → 발행 불가
- 원작 확인 미체크 → 발행 불가
- 전시명 미입력 → 발행 불가
- 작가명 검증 없음

### 함께 올리기

- 이미지 0장 → 발행 불가
- 원작 확인 미체크 → 발행 불가
- 전시명 미입력 → 발행 불가
- 그룹명 미입력 → 발행 불가
- **모든 이미지**에 대해 작가 지정 필수 (회원 선택 또는 비회원 직접 입력 둘 중 하나)
  - 작가 미지정 이미지가 있으면 토스트: "모든 작품에 작가를 입력해주세요."

### 검수 처리

- 발행 직후 작품 상태는 **`pending`** (운영팀 검수 대기).
- 운영팀 검수 승인 시 `approved` → 피드 노출.
- 반려 시 `rejected` → 작가 본인 프로필에만 보이고 반려 사유와 함께 재업로드 안내.

## 9. 작품명 자동생성 규칙

발행 시점에 작품명이 비어 있으면 아래 규칙으로 자동 생성한다.

| 유형 | 이미지 수 | 자동 작품명 |
|---|---|---|
| 혼자 올리기 | 1장 | 전시명 그대로 사용 (전시명도 없으면 "무제") |
| 혼자 올리기 | 2장 이상 | "무제" |
| 함께 올리기 | — | `[작가명]의 작품 N` (해당 장의 작가명 사용, 1부터 증가) |

전시명·작품명·그룹명은 모두 최대 **20자** (코드: `TITLE_FIELD_MAX_LEN` in `src/app/utils/workDisplay.ts`).

---

## 10. 그룹전시 자동 분류 규칙

발행 시 참여 고유 작가 수로 전시 유형을 결정한다.

- 고유 작가 **2명 이상** → 그룹전시
- 그 외 → 개인전시

> 강사가 자기 이름 없이 학생 작품만 올려도 고유 작가가 2명 이상이면 그룹전시로 분류된다.

## 11. 비회원 작가 처리

| 단계 | 내용 |
|---|---|
| 입력 | 실명(필수) + 연락처(필수) |
| 발행 직전 | **확인 모달**로 수신자 목록 + "실명 정확성 경고" 표시 (가입 후 자동 매칭 실패 방지) |
| 발행 시 | 카카오 알림톡 우선 / SMS 보조 발송 |
| 알림 내용 | "내 작품이 Artier에 게시됐어요. 확인해볼래요?" |
| 가입 후 | 연락처 + 실명 기반으로 작품 자동 연결 |
| 이름 불일치 시 | 가입 직후 **본인 확인 게이트** 모달 (수락/거부/보류) |
| 미가입 상태 | 이름 텍스트만 표시, 팔로우 버튼 없음 |

비회원 작가 처리 상세는 `docs/screens/00-flows.md` F-03 SMS 초대 매칭 흐름 참조.

## 12. 발행 후 처리

### 포인트

- 업로드 완료 시 작가 포인트(AP) +20 자동 적립.
- **24시간 이내 삭제 시 AP -20 회수** (어뷰징 방지).

### 상태·이동

- 발행 직후 상태는 `pending` (검수 대기).
- 발행 후 이동: **내 프로필의 전시 탭** (검수 대기 배너와 함께 노출).

### 발행 직후 공유 시 주의

- `pending` 상태 작품은 공개 피드에서 제외된다.
- 작가가 발행 직후 공유 링크를 전달한 경우 수신자가 **빈 전시로 오인**할 수 있음.
- UX 권장: 공유 버튼 근처에 "검수 통과 후 공개됩니다" 안내 표기 + 공유 랜딩에 "검수 중 — 공개 전" 상태 배지 노출.

### 임시저장

- 임시저장 후 복귀 시 직전 상태 그대로 복원.
- **자동 저장**: 편집 중 일정 간격(기본 30초)으로 임시저장 실행.
- 페이지 이탈 시 저장되지 않은 변경이 있으면 브라우저 기본 `beforeunload` 경고 노출.

## 13. 확인 다이얼로그 규칙

업로드 플로우 내 모든 확인·경고는 **앱 내 확인 다이얼로그**만 사용. 브라우저 네이티브 `confirm`/`alert`는 금지.

- 파괴적 작업(삭제·초안 폐기 등)은 빨간 확인 버튼.
- 일반 작업은 기본 버튼.

적용 대상:
- 업로드 유형 변경 (이미지가 이미 있을 때)
- 발행 취소 / 페이지 이탈
- 개별 이미지 삭제
- 초안 폐기
- 비회원 작가 초대 발송 직전 (실명 정확성 경고)
