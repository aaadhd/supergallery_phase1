# 03. 프로필 · 전시 상세

> 이 문서 단독으로 이해 가능하도록 작성. 레퍼런스 명세를 이 문서 기준으로 업데이트해주세요.

## 사전 지식

### 프로필이 가지는 개념 필드

| 필드 | 설명 |
|---|---|
| 닉네임 | 공개 표시명 (2~20자) |
| 실명 | 초대 매칭·본인 확인용 (비공개) |
| 전화번호 | 초대 매칭·중복 방지용 (비공개) |
| 이메일 | 로그인·연락용 |
| 프로필 이미지 | |
| 헤드라인(Prominent) | 눈에 띄는 한 줄 소개 (최대 20자) |
| 자기소개 | 긴 소개 (글자 제한 없음, 비속어 검사) |
| 위치 | 국가 (대한민국·미국·일본 등) |
| 외부 링크 | 인스타그램·유튜브·페이스북·트위터·핀터레스트·비메오 6종 **고정 행**에 핸들/ID만 입력 (URL 자동 조립). 각 플랫폼 고유 아이콘으로 표시 |

**강사 표시**: 별도 토글 UI 없음. "강사 업로드로 표시된 작품이 1건이라도 있는가?"로 자동 판정. 한 작품이라도 있으면 "수강생 작품" 탭이 자동 노출, 모두 삭제되면 자동 숨김.

### 전시 라우트 분기

하나의 전시 URL(`/exhibitions/:id`)이 쿼리 파라미터로 여러 랜딩에 분기:

| 쿼리 | 렌더 | 시나리오 |
|---|---|---|
| 없음 / `?from=credited` | 전시 상세 | 일반 진입 |
| `?from=invite` | 전시 초대 랜딩 | 비회원이 SMS 초대 링크 클릭 |
| `?from=work` | 작품 공유 랜딩 | 작품 단위 공유 링크 |

### 이 카테고리 공통 정책

- **시니어 친화 터치 영역**: 최소 44×44 픽셀
- **작품 이미지 비율 유지**: 잘라내기 금지
- **확인 다이얼로그**: 앱 자체 디자인. 브라우저 네이티브 금지
- **외부 링크 방식**: 사용자가 자유 URL 입력 X. 6개 플랫폼 고정 행에 핸들/ID만 입력 → 시스템이 URL 조립

## SCR-PROF-01 · 프로필

| 항목 | 값 |
|---|---|
| URL | `/profile` (본인) · `/profile/:id` (특정 작가) · `/me` (본인 별칭) |
| 쿼리 파라미터 | `?tab=work\|exhibition\|student` · `?published=pending&workId={id}` (검수 대기 안내 배너) · `?edit=1` (편집 모달 자동 오픈) |
| 접근 권한 | 비로그인 가능. 편집·작품 보기·팔로우는 로그인 필수 |
| 진입 경로 | 헤더 아바타 · 작가 카드 · 작품 업로드 완료 후 자동 이동 · 검색 결과 |
| 레이아웃 | 공통 Layout |
| 우선순위 | **P0** · 런칭 필수 |

### 화면의 목적
작가 정체성과 작품 포트폴리오를 보여주는 메인 페이지. 본인 진입과 타인 진입을 같은 페이지로 처리하되, 본인일 때만 편집·관리 UI가 추가로 노출.

### 사용자 시나리오

| 유형 | 기대 |
|---|---|
| 본인 진입 | 프로필 편집, 새 작품 올리기, 반려 작품 사유 확인, 수강생 작품 관리, 비공개/삭제 |
| 타인 진입 | 작품 감상, 팔로우, 외부 링크로 추가 연락 |
| 강사 프로필 진입 | "수강생 작품" 탭에서 클래스 결과물 감상 |

### 화면 구성

| 영역 | 요소 |
|---|---|
| 헤더 | 아바타(탭 시 원본 미리보기 모달) · 닉네임(실명 같이 표시 가능) · Prominent 헤드라인 · 위치 · 자기소개 |
| 팔로우/팔로워 카운트 | 숫자 + [팔로우] 버튼(타인 진입 시만) |
| 외부 링크 | 입력된 플랫폼의 아이콘만 그리드로 표시 (최대 6종). 탭 시 새 탭 |
| 공유 | 공유 아이콘 — 기기 공유 시트 or 링크 복사 |
| **탭** | 전시 / 작품 관리 / 수강생 작품(강사만) / 좋아요 / 저장 / 초안 |
| **탭 안내 문구** | 각 탭 상단에 현재 단위를 안내: 전시=전시 단위·관리, 작품 관리=내 그림 모아보기+작품명 수정, 수강생 작품=강사 업로드 작품, 좋아요/저장/초안=사용자 개인 큐레이션 목록 |
| **작품 관리 탭** | 전시에 포함된 내 그림을 이미지 단위로 모아보는 탭. 카드 `...` 메뉴에서 작품명 수정 모달 호출. 변경된 작품명은 원본 전시에도 동기화됨 (저작권자 = 작가 본인). 전시 구조 수정·삭제는 전시 탭에서만 가능 |
| **전시 탭** | 전시 카드 (그룹 전시는 참여 작가 묶음). 카드 하단에 전시명 + 그룹명(또는 작가명) 2줄 표시. 본인 진입 시 "내가 올린 전시만" 필터 |
| **수강생 작품 탭** | 강사 자동 파생 조건 충족 시만 노출. 강사가 그룹 전시로 발행한 작품 중 비-강사 슬롯들 모음 |
| 검수 대기 배너 | 발행 직후 진입 시 상단에 "발행 완료! 검수 후 공개돼요. 보통 24시간 내" 닫기 가능한 안내 |

### 본인 편집 기능

| 항목 | UI |
|---|---|
| 닉네임/실명/헤드라인/자기소개/위치 | 편집 모달 — 비속어 검사 |
| 외부 링크 | 6개 플랫폼 고정 행, 핸들/ID만 입력 (URL은 시스템 조립) |
| 프로필 이미지 | 원본 미리보기 모달에서 변경 |
| 작품명 수정 | 작품 관리 탭 `...` 메뉴 → 작품명 수정 모달 (최대 20자). 원본 전시에도 동기화 |
| 반려 배지 탭 | 모달 오픈: 반려 사유(4분류) + 운영팀 메시지 + [수정해서 다시 올리기] → 업로드 편집 모드로 이동 |
| 작품 삭제 | "…" 메뉴 → 파괴적 확인 다이얼로그. **24시간 내 발행한 작품 삭제 시 -20 AP 회수** |
| 전시 편집 | "내가 올린" 전시만 편집 가능 — 업로드 편집 모드로 이동 |

### 타인 진입 시 특수 처리

- 반려 배지·편집 메뉴·비공개 토글 **숨김**
- 참여한 전시에서도 본인이 올리지 않은 건 편집 메뉴 숨김
- 편집 정책: **수정 진입은 전시 탭으로 단일화**. 작품 탭(작품 보기)은 조회/정리 전용
- **탈퇴한 작가 진입**: 빈 프로필 + "이 작가는 더 이상 활동하지 않습니다" 안내

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 아바타 | 같은 화면에서 **프로필 이미지 모달** 열림 (원본 미리보기 · 본인은 변경 가능) |
| 팔로워/팔로잉 숫자 | (Phase 1 구현 여부 확인 필요) — Phase 2에서 해당 목록 화면으로 이동 예정 |
| [팔로우] 버튼 (타인) | 화면 이동 없음 — 상태 토글. 비로그인이면 로그인 유도 모달 |
| 외부 링크 아이콘 | **새 탭에서 해당 플랫폼 URL 열림** |
| 공유 아이콘 | 기기 공유 시트 or 링크 복사 — 화면 이동 없음 |
| 탭 (작품/전시/수강생) | **같은 화면에서 탭 전환** — URL 쿼리 `?tab=...` 갱신 (뒤로가기로 이전 탭 복귀) |
| 작품 카드 탭 | **작품 상세 모달** 열림 (08 문서 MOD-03) — 화면 이동 없음 |
| 전시 카드 탭 | **전시 상세(`/exhibitions/:id`)** 로 이동 |
| 수강생 작품 카드 탭 | 작품 상세 모달 열림 |
| 본인: 카드 "…" → 편집 | **업로드 화면(`/upload?editWorkId=...`)** 편집 모드로 이동 |
| 본인: 카드 "…" → 삭제 | 파괴적 확인 다이얼로그 → 확인 시 삭제 (화면 이동 없음) |
| 본인: 반려 배지 탭 | **반려 사유 모달** 열림 → 모달 안 [수정해서 다시 올리기] → 업로드 편집 모드로 이동 |
| 본인: "프로필 편집" 버튼 | 같은 화면 위에 **편집 다이얼로그** 열림 — URL에 `?edit=1` 추가 (이 URL로 직진입해도 자동 오픈) |
| 검수 대기 배너 [닫기] | 배너 숨김 (URL 쿼리 정리) |
| 상단 "뒤로가기" | 이전 화면으로 복귀 |

### 수용기준 (Acceptance Criteria)

- AC-01: Given 본인 프로필 진입 / When 페이지 로드 / Then 전시·작품 관리·수강생 작품·좋아요·저장·초안 6개 탭이 모두 표시된다
- AC-02: Given 타인 프로필 진입 / When 페이지 로드 / Then 전시 탭만 표시되고, 편집·삭제 메뉴는 노출되지 않는다
- AC-03: Given 전시 탭에서 본인 진입 / When "내가 올린 전시만" 체크박스 토글 / Then 본인이 올린 전시만 필터링되고, 전체/개인/그룹 필터와 건수가 정확히 반영된다
- AC-04: Given 전시 탭의 전시 카드 / When 카드 하단 확인 / Then 전시명 + 그룹명(또는 작가명) 2줄이 표시된다
- AC-05: Given 본인 프로필의 전시 탭 / When 본인이 올린 전시 카드의 `...` 메뉴 탭 / Then 편집·삭제 옵션이 표시된다
- AC-06: Given 검수 대기(pending) 상태 작품 / When 본인 프로필에서 확인 / Then 검수 대기 배지가 작품 카드에 표시된다
- AC-07: Given 반려(rejected) 상태 작품 / When 반려 배지 탭 / Then 반려 사유(4분류) + 운영팀 메시지가 모달로 표시되고, [수정해서 다시 올리기] 버튼이 업로드 편집 모드로 이동시킨다
- AC-08: Given 작품 관리 탭 / When 카드 `...` 메뉴에서 작품명 수정 / Then 수정된 작품명(최대 20자)이 원본 전시에도 동기화된다
- AC-09: Given 강사 업로드(`isInstructorUpload === true`) 작품이 1건 이상 존재 / When 프로필 진입 / Then "수강생 작품" 탭이 자동으로 노출되고 프로필에 강사 배지가 표시된다
- AC-10: Given 강사 업로드 작품을 모두 삭제 / When 프로필 재진입 / Then "수강생 작품" 탭과 강사 배지가 자동으로 숨겨진다
- AC-11: Given 프로필 편집 모달 / When 닉네임·헤드라인·자기소개에 비속어 포함 후 저장 시도 / Then 비속어 검사에 의해 저장이 차단되고 안내 메시지가 표시된다
- AC-12: Given 프로필 편집 모달에서 변경 사항이 있는 상태 / When 저장하지 않고 닫기 시도 / Then 확인 다이얼로그가 표시된다
- AC-13: Given 좋아요 또는 저장 탭에 포함된 작품 / When 해당 작품이 삭제됨 / Then 좋아요·저장 탭에서 해당 작품이 제거된다
- AC-14: Given 팔로워/팔로잉 숫자 / When 숫자 탭 / Then 팔로워/팔로잉 목록 모달이 열린다

### 엣지케이스

- EC-01: 존재하지 않는 작가 ID로 프로필 진입 → 404 안내 화면 표시
- EC-02: 탈퇴한 작가 프로필 진입 → 빈 프로필 + "이 작가는 더 이상 활동하지 않습니다" 안내 표시
- EC-03: 작품이 0건인 본인 프로필 → 업로드 CTA 표시 (빈 상태)
- EC-04: 작품이 0건인 타인 프로필 → "아직 작품이 없습니다" 안내만 표시
- EC-05: 타인 진입 시 검수 대기(pending)·반려(rejected) 작품 → 해당 작품 숨김 처리

## SCR-EX-01 · 전시 상세

| 항목 | 값 |
|---|---|
| URL | `/exhibitions/:id` (쿼리 없음 또는 `?from=credited`) |
| 접근 권한 | 비로그인 가능. 상호작용은 로그인 |
| 진입 경로 | 피드 카드 · 작가 프로필 전시 탭 · 검색 · 공유 링크 |
| 레이아웃 | 공통 Layout |

### 화면의 목적
개인·그룹 전시 본문. 여러 작품을 한 캔버스 경험으로 감상.

### 사용자 시나리오
- 작품 감상 목적으로 들어온 회원·게스트
- 기대: 큰 이미지로 작품들을 차례로 감상, 작가 정보 확인, 좋아요·공유

### 화면 구성

| 영역 | 요소 |
|---|---|
| 헤로 | 커버 이미지(흐림 처리된 배경 + 큰 메인) + 전시명 + 작가(개인) / 참여 작가 칩 리스트(그룹) |
| 기간·그룹명 | 개최 기간 + 그룹명(그룹 전시) |
| 설명 | 에디토리얼 텍스트 (있을 때만) |
| 작품 리스트 | 세로 스크롤로 차례대로. 각 작품: 큰 이미지 + 제목 + 작가(그룹) |
| 작품 탭 | 작품 상세 모달 오픈 (감상 + 좋아요/저장/공유) |
| 공유 버튼 | 공유 URL에 **초대 진입 파라미터가 자동으로 붙음** — 받은 사람은 초대 랜딩으로 진입 |
| 좋아요 | 작품 단위 토글 |

### 배경 톤 처리
커버 이미지를 화면 배경에 깔되 **흐림 + 확대 + 투명도 조정**으로 작품 색감이 자연스럽게 묻어남. 별도 색상 추출 알고리즘 없이 단순 시각 효과로 구현.

| 로딩 | 스켈레톤 |
| 존재하지 않음 | 404 안내로 전환 |
| 비공개 / 검수 대기 / 반려 | 본인 아니면 404 안내 |

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 상단 뒤로가기 | 이전 화면으로 복귀 (피드면 피드, 프로필 전시 탭이면 그 탭) |
| 참여 작가 칩 (그룹 전시) | 해당 작가 프로필(`/profile/:id`)로 이동 |
| 개인 전시 작가 아바타/이름 | 해당 작가 프로필로 이동 |
| 작품 이미지 탭 | **작품 상세 모달** 열림 (화면 이동 없음) — 좌우 화살표로 같은 전시 내 다음/이전 작품 |
| 작품 카드 좋아요 | 화면 이동 없음 — 상태 토글. 비로그인이면 로그인 유도 모달 |
| 공유 버튼 | 기기 공유 시트 or 링크 복사. **공유 URL에 초대 진입 파라미터 자동 부여** → 받는 사람이 탭하면 전시 초대 랜딩 또는 작품 공유 랜딩으로 진입 |
| 전시 설명 안 이벤트 링크 (있으면) | 해당 이벤트 상세(`/events/:id`)로 이동 |

### 수용기준 (Acceptance Criteria)

- AC-01: Given 전시 상세 페이지 진입 / When 페이지 로드 / Then 커버 이미지가 흐림+확대+투명도 배경 위에 큰 메인 이미지로 표시된다 (헤로 영역)
- AC-02: Given 전시 상세 페이지 / When 공유 버튼 탭 / Then 생성된 공유 URL에 초대 진입 파라미터(`?from=invite`)가 자동으로 포함된다
- AC-03: Given 작품 리스트의 작품 이미지 / When 탭 / Then 작품 상세 모달이 열리고 좌우 화살표로 같은 전시 내 이전/다음 작품 탐색이 가능하다
- AC-04: Given 작품 상세 모달 / When 좋아요 또는 저장 버튼 탭 / Then 해당 상태가 토글된다 (비로그인 시 로그인 유도 모달)
- AC-05: Given 그룹 전시 상세 / When 페이지 로드 / Then 참여 작가 칩 리스트가 표시되고, 각 칩 탭 시 해당 작가 프로필로 이동한다
- AC-06: Given 존재하지 않는 전시 ID / When 페이지 진입 / Then 404 안내 화면으로 전환된다

### 엣지케이스

- EC-01: 비공개(hidden)·검수 대기(pending)·반려(rejected) 전시에 비소유자가 접근 → 404 안내 표시
- EC-02: 전시 로딩 중 → 스켈레톤 UI 표시

### Phase 2 보강
- 작가가 방문객에게 남기는 인사글 (전시 코멘트)
- 평면 뷰 vs 격자 뷰 전환
- PDF·프린트 내보내기

## SCR-EX-02 · 전시 초대 랜딩

| 항목 | 값 |
|---|---|
| URL | `/exhibitions/:id?from=invite&invited_phone=...&invited_name=...` |
| 접근 권한 | 비로그인 전용 (이미 로그인이면 전시 상세로 자동 전환) |
| 진입 경로 | SMS·알림톡 메시지의 링크 클릭 |
| 레이아웃 | 공통 Layout |

### 화면의 목적
**비회원 작가**(작가가 비회원을 전화번호로 그룹 전시에 초대한 경우)에게 자기 그림이 이미 전시된 장면을 보여주고 가입으로 유도.

### 사용자 시나리오
- 주 사용자: SMS를 받은 비회원 (대부분 초대한 작가의 동호회/클래스 멤버)
- 기대: "오, 내 그림이 진짜 전시되네!" 확인 → 가입하면 내 계정에 작품이 자동 연결될 거라는 안내

### 화면 구성

| 영역 | 요소 |
|---|---|
| 환영 카피 | "○○님, 회원님의 작품이 Artier에 전시되었습니다" — 초대 시 작가가 입력한 이름으로 인사 |
| 전시 미리보기 | 커버 + 전시명 + 기간 |
| 본인 작품 하이라이트 | 초대받은 사람이 기여한 작품만 강조 (다른 작품은 작게) |
| **[가입하기]** 큰 버튼 | 주 CTA |
| 이미 회원? | "이미 계정이 있으신가요? 로그인" |

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| **[가입하기]** 주 CTA | URL 파라미터의 전화·이름을 prefill 저장소에 보관 + 초대 가입 플래그 세팅 → **가입 화면(`/signup?invited=1`)**으로 이동 |
| "이미 계정이 있으신가요? 로그인" | **로그인(`/login`)**으로 이동. 로그인하면 자동으로 전시 상세로 전환됨 |
| 작품 하이라이트 이미지 | 탭하면 확대 보기(모달). 가입 유도가 목적이라 상세 조작은 제한 — 비로그인 상태라 좋아요/저장은 안 됨 |

### 가입 핸드오프 동작

URL의 `invited_phone`·`invited_name`을 해석해 가입 흐름 전용 prefill 저장소에 보관. [가입하기] 탭 시 가입 화면으로 이동. 가입 → 온보딩 진행 → 온보딩에서 전화·실명이 prefill됨 (사용자가 수정 가능). 온보딩 완료 시 자동 매칭 시도.

### 매칭 메커니즘 (온보딩 완료 시점)

시스템이 저장된 비회원 초대 발송 기록을 검색:

| 조건 | 결과 |
|---|---|
| 전화 일치 + 이름 일치 | 작품의 비회원 슬롯을 회원 슬롯으로 자동 승격 → "초대받은 작품 N개가 내 계정과 연결되었어요" 토스트 |
| 전화 일치 + 이름 불일치 | 자동 매칭은 차단 → 홈 도착 시 본인 확인 모달이 자동으로 뜸. 수락 시 수동 매칭, 거부 시 발신 작가에게 허위 초대 경고 +1 |

### 수용기준 (Acceptance Criteria)

- AC-01: Given 비회원이 SMS 초대 링크로 진입 / When 페이지 로드 / Then URL의 `invited_name`을 사용해 "○○님, 회원님의 작품이 Artier에 전시되었습니다" 환영 카피가 표시된다
- AC-02: Given 초대 랜딩 페이지 / When [가입하기] CTA 탭 / Then URL의 `invited_phone`·`invited_name`이 prefill 저장소에 보관되고 가입 화면(`/signup?invited=1`)으로 이동한다
- AC-03: Given 이미 로그인된 상태 / When 초대 랜딩 URL 진입 / Then 자동으로 전시 상세(`/exhibitions/:id`)로 리다이렉트된다
- AC-04: Given 가입 후 온보딩 완료 시 / When 전화번호 일치 + 이름 일치 / Then 비회원 슬롯이 회원 슬롯으로 자동 승격되고 "초대받은 작품 N개가 내 계정과 연결되었어요" 토스트가 표시된다
- AC-05: Given 가입 후 온보딩 완료 시 / When 전화번호 일치 + 이름 불일치 / Then 자동 매칭이 차단되고 홈 도착 시 본인 확인 모달(수락/거부/나중에)이 자동으로 표시된다

### 엣지케이스

- EC-01: 만료된 초대 링크로 진입 → 유효하지 않은 초대 안내 표시 (Phase 2에서 토큰 기반 만료 정식 구현 예정)
- EC-02: 이미 다른 계정으로 가입한 사용자가 초대 링크로 진입 → 로그인 상태이므로 전시 상세로 자동 전환, 별도 안내 없음

### Phase 2 보강
- **초대 토큰** 기반 매칭 (URL에 유니크 토큰 포함) — 이름 양식 불일치로 인한 매칭 실패를 원천 제거
- 초대 만료 (예: 30일)
- 초대 코드 별도 입력 옵션 (SMS 못 받은 경우)

## SCR-EX-03 · 작품 공유 랜딩

| 항목 | 값 |
|---|---|
| URL | `/exhibitions/:id?from=work&workId=...` |
| 접근 권한 | 비로그인 가능 |
| 진입 경로 | 작품 상세 모달 "공유" 버튼에서 만든 URL을 누군가 받아서 탭 |
| 레이아웃 | 공통 Layout |

### 화면의 목적
특정 **작품 단위**로 공유받은 사람이 그 작품을 중심으로 전시에 진입하도록. 일반 전시 상세와 달리 공유받은 작품이 상단에 포커스.

### 화면 구성

| 영역 | 요소 |
|---|---|
| 상단 배지 | "○○님이 공유한 작품" |
| 공유받은 작품 | 큰 이미지 + 제목 + 작가 |
| 같은 전시 다른 작품 | 하단에 그리드 |
| [전시 전체 보기] | 탭 시 일반 전시 상세로 전환 |

### 사용자 시나리오
- 작품 한 장 보고 들어온 외부 사용자 (카톡·페북 등 소셜 공유)
- 기대: 공유받은 작품 먼저 보고, 마음에 들면 같은 전시 다른 작품도 둘러보기

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 공유받은 작품 큰 이미지 | 작품 상세 모달 열림 |
| 작가 아바타/이름 | 작가 프로필(`/profile/:artistId`)로 이동 |
| 하단 "같은 전시 다른 작품" 카드 | 작품 상세 모달 열림 (해당 작품에 포커스) |
| **[전시 전체 보기]** | **전시 상세(`/exhibitions/:id`)** 로 이동 (쿼리 없이 기본 진입) |

### 수용기준 (Acceptance Criteria)

- AC-01: Given 작품 공유 URL로 진입 / When 페이지 로드 / Then `workId`에 해당하는 공유받은 작품이 상단에 큰 이미지로 포커스되어 표시된다
- AC-02: Given 작품 공유 랜딩 / When 하단 영역 확인 / Then 같은 전시의 다른 작품들이 그리드로 표시된다
- AC-03: Given 작품 공유 랜딩 / When [전시 전체 보기] 버튼 탭 / Then 쿼리 없는 일반 전시 상세(`/exhibitions/:id`)로 이동한다
- AC-04: Given 검수 대기(pending) 상태의 공유 작품 / When 본인이 접근 / Then 검수 대기 배지가 작품에 표시된다

### 엣지케이스

- EC-01: 삭제된 작품의 공유 URL로 진입 → 404 안내 화면 표시

### Phase 2 보강
- OG 이미지 동적 생성 — 공유 시 메신저에서 작품 썸네일 자동 노출
- 작품 단위 조회수 집계
