# 레퍼런스 대비 제거·통합된 화면 목록 (Phase 1)

> **이 문서의 용도**: 원본 레퍼런스(화면 모음)에는 별도 스펙으로 존재하지만, Phase 1에서 **의도적으로 제거·통합·보류된 화면**을 명시합니다.
> 개발자가 "레퍼런스엔 있는데 docs엔 왜 없지?"를 판단할 때 이 문서를 먼저 확인하세요.
>
> **기준일**: 2026-04-16

---

## 적용 규칙 요약

| 상황 | 판정 |
|---|---|
| 이 문서에 있는 화면 | **구현하지 않음 / 구현해도 별도 스펙 없음** — 레퍼런스의 해당 스펙은 폐기 |
| 이 문서에도, `docs/screens/`에도 없는 레퍼런스 화면 | **레퍼런스 그대로 따름** (기본 규칙) |
| `docs/screens/`에 있는 화면 | 이 문서의 결정이 레퍼런스보다 우선 |

## 1. 별도 화면에서 다른 화면의 일부로 통합된 항목

> 레퍼런스는 화면 단위로 쪼개져 있지만, 제품 결정으로 **상위 화면에 흡수**되어 별도 스펙이 불필요해진 것들.

| 레퍼런스 원본 | 통합된 위치 | 사유 |
|---|---|---|
| **GNB 전체 레이아웃** | `00-globals.md` 헤더(SCR-GLOBAL-01)로 통합 | 헤더 하나로 일원화 |
| **GNB 아바타 드롭다운** | `00-globals.md` 헤더 내 요소로 기술 | 독립 화면 아님 |
| **알림 툴팁** | `00-globals.md` 헤더 내 알림 벨 드롭다운으로 기술 | 별도 스펙 불필요 |
| **강사 프로필** | `03-profile-exhibition.md` 프로필(SCR-PROFILE-01)로 통합 | **강사/작가 구분 없이 단일 프로필** — 강사 여부는 업로드 이력에서 자동 파생 |
| **강사 프로필 — 기본 (전시 탭)** | 동상 | 동일 사유 |

## 2. Phase 1에서 의도적으로 제거·보류된 별도 화면

> 레퍼런스에 스펙이 있지만 Phase 1에 **구현하지 않거나 단순화로 대체**한 항목.

| 레퍼런스 원본 | 결정 | 사유·대체 방식 |
|---|---|---|
| **로그아웃** (별도 화면) | ❌ 화면 단위 제거 | 별도 화면 없이 **헤더 아바타 드롭다운 내 로그아웃 버튼**으로 대체. 버튼 클릭 시 확인 다이얼로그 → 즉시 로그아웃. 동선은 `00-globals.md` 헤더 스펙 참조 |
| **배지 UI 컴포넌트** | ⚠️ 화면 아님 | 디자인 시스템 컴포넌트 — 별도 화면 스펙 대상이 아님. 어느 화면에서 배지가 쓰이는지는 각 화면 스펙 내부에 표기 |
| **장별 패널** | ⚠️ 화면 아님 | 업로드 화면 내부의 작품 메타 입력 영역 — `02-core.md` 업로드 스펙과 `docs/upload_spec.md` 내부에 통합 기술. 별도 화면 스펙 폐기 |
| **참여 완료 확인 화면** (이벤트 응모 후 확인 페이지) | ❌ 화면 단위 제거 | **토스트 + 이벤트 상세 버튼 상태 전환**으로 대체. 별도 확인 페이지 없이 응모 상태가 "응모 완료"로 즉시 바뀜. 동선은 `00-flows.md` F-09 이벤트 응모 흐름 참조 |
| **코치마크 툴팁 안내** | ⏸ Phase 2로 보류 | 시니어 사용자 대상 학습 가이드는 Phase 2 과제. Phase 1은 첫 사용자 온보딩 1회 + FAQ·서비스 소개 문서로 대체 |
| **화면 미리보기** | ⏸ 운영/검수용 도구로 이관 | 레퍼런스엔 일반 기능처럼 분류됐지만, 실제로는 **운영팀 검수 도구**로 분류. 일반 사용자 화면이 아니므로 `06-demo-tools.md` 및 내부 검수 툴킷으로 이동 |
| **회원가입 로그인** (합본 진입 화면) | ❌ 합본 제거 | "회원가입"과 "로그인"을 **별도 진입 화면**으로 분리. `01-auth.md` SCR-AUTH-01(로그인), SCR-AUTH-02(회원가입), SCR-AUTH-03(소셜 회원가입 모달) 각각 참조 |

## 3. 구현은 있으나 docs/screens에 단독 스펙이 없는 항목 (레퍼런스 그대로 적용)

> 아래 화면은 구현되어 있고 레퍼런스 스펙이 여전히 유효합니다. 기본 규칙대로 **레퍼런스 원본을 따르면 됩니다**. 단, Phase 1에서 동작이 달라지는 부분만 여기에 메모합니다.

| 레퍼런스 원본 | 구현 위치 요약 | Phase 1 메모 |
|---|---|---|
| **딥줌 뷰어** | 작품 상세 모달 내부에서 연다 | 레퍼런스 스펙 유지. 큰 이미지에 한해 핀치 줌·드래그 제공 |
| **팔로워 팔로잉 목록** | 프로필 화면 내 모달로 구현 | 레퍼런스 스펙 유지. 프로필 팔로워/팔로잉 숫자 클릭 시 모달로 열림. 본인/타인 모두 조회 가능 |
| **비가입 작업자 초대 화면** | 업로드 중 "함께 올리기"에서 비회원 추가 UI로 구현 | 레퍼런스 스펙 유지. 실명 + 전화번호 입력 → 초대 발송. 상세는 `02-core.md` 업로드 스펙 참조 |
| **작품 수정화면** | 업로드 화면을 편집 모드로 재사용 (`?editWorkId={id}`) | 레퍼런스의 별도 편집 스펙은 폐기. **업로드 화면 단일 스펙**으로 통합 — 편집 모드 차이는 `02-core.md` 업로드 "상태·분기"에 기술 |
| **로그인 유도 모달** | 비로그인 사용자가 좋아요/저장/팔로우 등 인터랙션 시 모달로 제공 | 레퍼런스 스펙 유지. 모달은 `08-modals.md`에도 간단 기술 |
| **쿠키 동의 배너** | 첫 방문 시 하단 배너로 제공 | 레퍼런스 스펙 유지. `00-globals.md` 푸터/전역 영역에 간단 기술 |
| **대시보드** | 어드민 메인 대시보드로 구현 | `07-admin.md` SCR-ADMIN-01 참조 |

## 4. 체크리스트 — 개발자가 레퍼런스를 그대로 따라갈지 판단하는 순서

1. `docs/screens/`에 해당 화면 스펙이 있나? → **있으면 이 문서 우선, 레퍼런스는 참고만**
2. 이 문서(99-removed-from-reference)에 해당 화면이 있나? → **있으면 해당 결정(제거·통합·보류) 따름**
3. 둘 다 없나? → **레퍼런스 그대로 따름** (기본 규칙)

> 애매하면 PM에게 문의. 위 1→2→3 순서로 판정해도 판단이 어려운 경우가 나오면 바로 소통해주세요.
