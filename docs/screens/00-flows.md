# 00. 주요 사용자 흐름 (Flows)

> 화면 스펙은 각 카테고리 문서에 있고, 이 문서는 **end-to-end 흐름 narrative만** 모음. 각 단계 옆의 SCR/MOD ID로 자세한 스펙을 찾아가세요.
> 흐름이 깨지거나 모호한 지점을 빨리 발견하기 위한 검수 도구.

---

## 흐름 인덱스

| # | 이름 | 핵심 사용자 | 진입 → 종착 |
|---|---|---|---|
| F-01 | 신규 가입 — 이메일 | 신규 회원 | 로그인 → 가입 → 온보딩 → 둘러보기 |
| F-02 | 신규 가입 — 소셜 | 신규 회원 | 로그인 → 소셜 모달 → 온보딩 → 둘러보기 |
| F-03 | 비회원 SMS 초대 → 작품 자동 연결 | 비회원 작가 (초대받음) | SMS → 초대 랜딩 → 가입 → 온보딩 → 매칭 |
| F-04 | 작품 업로드 (개인) | 작가 | 둘러보기/내 프로필 → 업로드 → 검수 대기 안내 |
| F-05 | 작품 업로드 (그룹·강사) | 강사 | 업로드(그룹) → 비회원 초대 발송 → 검수 |
| F-06 | 작품 발견 → 좋아요 → 작가 팔로우 | 게스트·회원 | 둘러보기 → 작품 상세 → 작가 프로필 → 팔로우 |
| F-07 | 작품 신고 → 운영 처리 → 작가 자동 정지 | 사용자 + 운영팀 | 신고 → 어드민 신고 관리 → 자동 7일 정지 |
| F-08 | 콘텐츠 검수 → 반려 → 작가 재업로드 | 운영팀 + 작가 | 어드민 검수 → 반려 알림 → 반려 사유 모달 → 업로드 편집 |
| F-09 | 이벤트 응모 → 검수 → 어드민 참여자 표시 | 작가 + 운영팀 | 이벤트 상세 → 업로드 → 어드민 참여자 |
| F-10 | 비밀번호 재설정 (이메일 가입자) | 회원 | 설정 → 재설정 다이얼로그 → 메일 발송 안내 |
| F-11 | 회원 탈퇴 | 회원 | 설정 → 탈퇴 다이얼로그 → 로그아웃 |
| F-12 | 운영팀 회원 정지 | 운영팀 + 회원 | 어드민 회원 관리 → 정지 → 다음 로그인 차단 |

## F-01 · 신규 가입 — 이메일

| 단계 | 화면/모달 | 사용자 행동 / 시스템 반응 |
|---|---|---|
| 1 | SCR-AUTH-01 (로그인) | "회원가입" 링크 탭 |
| 2 | SCR-AUTH-02 (가입 Step 1) | 이메일·비밀번호 입력 → [다음] |
| 3 | SCR-AUTH-02 (가입 Step 2) | 닉네임·생년월일 입력 (만 14세 이상 검증) → [다음] |
| 4 | SCR-AUTH-02 (가입 Step 3) | 약관 2종 필수 동의 → [가입 완료] |
| 5 | (시스템) | 로그인 처리 + 가입 +50 AP + 이메일·닉네임 prefill 보관 |
| 6 | SCR-AUTH-03 (온보딩 Step 1) | "환영" 화면 → [시작하기] |
| 7 | SCR-AUTH-03 (온보딩 Step 2) | 실명·전화 필수 입력 (이메일은 prefill) → [다음] +30 AP |
| 8 | SCR-AUTH-03 (온보딩 Step 3) | 컨페티 + [작품 올리기] 또는 [둘러보기 시작] |
| 9 | SCR-CORE-01 (둘러보기) 또는 SCR-CORE-03 (업로드) | 종착 |

**가능한 이탈 지점**:
- Step 1에서 이미 등록된 이메일 → "이미 가입된 이메일이에요" 에러 → 로그인으로 안내
- Step 2에서 14세 미만 → "만 14세 이상부터 가입 가능해요" 거부
- 온보딩 [나중에] → 프로필 미완성 상태로 둘러보기 진입

## F-02 · 신규 가입 — 소셜

| 단계 | 화면/모달 | 사용자 행동 / 시스템 반응 |
|---|---|---|
| 1 | SCR-AUTH-01 (로그인) | 카카오/구글/애플 버튼 탭 (첫 가입자) |
| 2 | MOD-01 (소셜 첫 가입 모달) | 닉네임 + 약관 + 14세 확인 → [가입 완료] |
| 3 | (시스템) | 로그인 + provider 첫 가입 마킹 + 닉네임·이메일(있으면) prefill 보관 |
| 4 | SCR-AUTH-03 (온보딩 Step 2) | 실명·전화·이메일 필수 입력 (소셜 가입자는 이메일도 필수) |
| 5 | SCR-AUTH-03 (온보딩 Step 3) | 종착 |

**재방문자**: 같은 provider로 두 번째부터는 모달 없이 즉시 로그인 → redirect 경로 또는 둘러보기.

## F-03 · 비회원 SMS 초대 → 작품 자동 연결

| 단계 | 화면/모달 | 사용자 행동 / 시스템 반응 |
|---|---|---|
| 1 | SCR-CORE-03 (업로드 — 작가 A) | 그룹 전시 발행 시 비회원 B를 이름·전화번호로 추가 |
| 2 | MOD-05 (비회원 초대 발송 확인) | 발행 직전 모달 — "실명 정확성 경고" 확인 → [발송] |
| 3 | (시스템) | 작품 저장 + 비회원 슬롯 등록 + SMS 모의 발송 + 작품에서 전화번호 스크럽 |
| 4 | (외부) | B의 휴대폰에 SMS 도착 |
| 5 | SCR-EX-02 (전시 초대 랜딩) | B가 SMS 링크 탭 → "회원님 작품이 전시되었습니다" 환영 → [가입하기] |
| 6 | SCR-AUTH-02 (가입 Step 1~3) | 이메일·비밀번호·닉네임·생년월일·약관 |
| 7 | SCR-AUTH-03 (온보딩 Step 2) | **전화·실명이 prefill됨** (URL에서 전달된 값). 사용자가 수정 가능 |
| 8 | (시스템) | 온보딩 완료 시 자동 매칭 시도 |
| 9-A | (매칭 일치) | "초대받은 작품 N개가 내 계정과 연결되었어요" 토스트 → 둘러보기 도착 |
| 9-B | (이름 불일치) MOD-02 (본인 확인 게이트) | 둘러보기 도착 시 모달 자동 오픈 → [네, 제 작품이에요] / [아니요, 모르는 초대] / [나중에] |

**위험**: 작가 A가 이름을 별명/호칭으로 입력한 경우 → 자동 매칭 실패 → 9-B 경로. 초대 발송 직전 confirm 모달(MOD-05)이 이걸 한 번 막아주지만 완전 차단은 못 함.

## F-04 · 작품 업로드 (개인)

| 단계 | 화면/모달 | 사용자 행동 / 시스템 반응 |
|---|---|---|
| 1 | SCR-CORE-01 또는 SCR-PROF-01 | 헤더 [작품 올리기] 탭 (또는 내 프로필 "새 작품") |
| 2 | (비로그인 시) MOD-06 (로그인 유도) | 로그인 후 업로드로 자동 복귀 |
| 3 | SCR-CORE-03 (업로드) | 유형 = 개인. 이미지 추가(드래그·정렬·삭제) → 전시명 입력 → 원본 동의 체크 → [발행하기] |
| 4 | MOD-04 (세부정보 모달) | 작품별 제목 + 대표 커버 선택 → [완료] |
| 5 | (시스템) | 검증 → 작품 저장 + 발행 +100 AP (첫 작품) 또는 +20 AP |
| 6 | SCR-PROF-01 (내 프로필 전시 탭) | 검수 대기 안내 배너 ("발행 완료! 검수 후 공개돼요. 보통 24시간 내") |

**검수 후속**: 운영팀 검수 → 승인 시 시스템 알림 → 피드 노출. 반려 시 → F-08로.

## F-05 · 작품 업로드 (그룹·강사)

F-04와 동일하되, 그룹 유형 선택 + 추가 단계:

| 단계 | 화면/모달 | 차이점 |
|---|---|---|
| 3a | SCR-CORE-03 (업로드 — 그룹) | 그룹명 입력 (필수) + "저는 강사예요" 체크 + 이미지별 작가 지정 (회원 검색 / 비회원 입력 / 셀프) |
| 3b | (시스템) | 유니크 작가 1명이면 → "개인 전시로 바꿀까요?" 모달 |
| 4a | MOD-04 (세부정보) | 동일 |
| 4b | MOD-05 (비회원 초대 확인) | 비회원 작가 1명 이상 시 발송 직전 confirm |
| 5 | (시스템) | 작품 저장 + 강사 +30 AP (월·강사 조합 첫 1회) + 비회원 SMS 발송 + 전화번호 스크럽 |
| 6 | SCR-PROF-01 | 검수 대기 + **"수강생 작품" 탭 자동 노출** (강사 업로드가 1건이라도 있으면) |

**후속**: 비회원 작가가 SMS 받으면 F-03 흐름으로 이어짐.

## F-06 · 작품 발견 → 좋아요 → 작가 팔로우

| 단계 | 화면/모달 | 사용자 행동 / 시스템 반응 |
|---|---|---|
| 1 | SCR-CORE-01 (둘러보기) | 피드 스크롤 → 마음에 드는 카드 탭 |
| 2 | MOD-03 (작품 상세 모달) | 큰 이미지 감상 → 좋아요 탭 |
| 3 | (비로그인 시) MOD-06 (로그인 유도) | 첫 시도면 모달, 이후 같은 세션엔 토스트만 |
| 4 | (시스템) | 좋아요 상태 토글 |
| 5 | MOD-03 | 작가 아바타 탭 |
| 6 | SCR-PROF-01 (작가 프로필) | 다른 작품·전시 감상 → [팔로우] 탭 |
| 7 | (시스템) | 팔로우 등록. 피드 랭킹에서 해당 작가 작품에 **팔로우 가산**(구현: `feedOrdering.ts`) |

## F-07 · 작품 신고 → 운영 처리 → 작가 자동 정지

| 단계 | 화면/모달 | 행위자 | 행동 |
|---|---|---|---|
| 1 | SCR-CORE-01 / SCR-EX-01 / SCR-PROF-01 | 사용자 | 작품 카드 "…" → 신고 |
| 2 | MOD-07 (신고 모달) | 사용자 | 사유 + 상세 → [신고하기] |
| 3 | (시스템) | — | 신고 접수 + 본인 피드에서 해당 작품 자동 숨김 + 운영팀 알림 |
| 4 | SCR-ADM-12 (어드민 신고 관리) | 운영팀 | 신고 카드 검토 → **[경고]** (3가지 다른 액션도 가능: 삭제 · 기각 · 비공개) |
| 5 | (시스템) | — | 작가에게 경고 +1 카운트. 누적 1·2회는 표시만 |
| 6 | (반복) | 사용자들 | 같은 작가에 신고 누적 |
| 7 | (시스템 자동) | — | **경고 누적 3회 → 자동 7일 정지로 승격**. 즉시 로그아웃 + 다음 로그인 차단 |
| 8 | SCR-AUTH-01 (로그인 시도) | 작가 | 정지 차단 메시지 표시 (사유·해제일·이의제기 안내) |

**유사 흐름 — 허위 신고 차단**: 운영팀이 신고를 [기각]하면 신고자에게 허위 신고 +1. 누적 3회 시 신고자도 7일 차단.

## F-08 · 콘텐츠 검수 → 반려 → 작가 재업로드

| 단계 | 화면/모달 | 행위자 | 행동 |
|---|---|---|---|
| 1 | SCR-ADM-06 (어드민 검수) | 운영팀 | 대기중 작품 카드 선택 → 큰 미리보기 검토 → [반려] |
| 2 | (모달) | 운영팀 | 사유 4분류 중 선택 + 운영팀 메시지 작성 → [반려 처리] |
| 3 | (시스템) | — | 작품에 반려 상태 + 사유·메시지 저장. 작가에게 시스템 알림 발송 |
| 4 | SCR-NOTI-01 (알림) | 작가 | "검수가 완료되었어요" 알림 탭 → 내 프로필 작품 탭으로 |
| 5 | SCR-PROF-01 (작품 탭) | 작가 | 해당 작품에 **반려 배지** 표시 → 탭 |
| 6 | MOD-09 (반려 사유 모달) | 작가 | 사유 + 운영팀 메시지 확인 → [수정해서 다시 올리기] |
| 7 | SCR-CORE-03 (업로드 편집 모드) | 작가 | 기존 데이터 prefill된 상태로 수정 → 다시 발행 |
| 8 | (시스템) | — | 검수 대기로 다시 진입 → 1단계로 |

## F-09 · 이벤트 응모 → 검수 → 어드민 참여자 표시

| 단계 | 화면/모달 | 행위자 | 행동 |
|---|---|---|---|
| 1 | SCR-EV-01 (이벤트 목록) | 작가 | 진행 중 이벤트 카드 탭 |
| 2 | SCR-EV-02 (이벤트 상세) | 작가 | [참여하기] 탭 |
| 3 | (검사) | — | 비로그인 → 로그인 유도. 이미 응모 → "이미 참여했어요" |
| 4 | SCR-CORE-03 (업로드 + 이벤트 연결) | 작가 | 작품 업로드 후 발행. 작품에 이벤트 ID 자동 연결 |
| 5 | SCR-ADM-05 (어드민 이벤트 참여자) | 운영팀 | 해당 이벤트 카드에 응모자 +1 자동 카운트 + 테이블에 표시 |
| 6 | SCR-ADM-06 (어드민 검수) | 운영팀 | 검수 → 승인 시 참여자 상태 "참여 완료"로 자동 업데이트 |

## F-10 · 비밀번호 재설정 (이메일 가입자)

| 단계 | 화면/모달 | 행위자 | 행동 |
|---|---|---|---|
| 1 | SCR-SET-01 (설정) | 회원 | 계정 섹션 "비밀번호 재설정" 탭 |
| 2 | (다이얼로그) | 회원 | "○○@example.com로 재설정 링크를 보내드려요" 안내 → [재설정 링크 보내기] |
| 3 | (시스템) | — | 토스트 "○○로 재설정 링크를 보냈어요" |
| 4 | (외부) | 회원 | 메일함에서 링크 클릭 |
| 5 | SCR-AUTH-04 (비밀번호 재설정 — `?demo=new_password`) | 회원 | 새 비밀번호 2회 입력 → [완료] |
| 6 | SCR-AUTH-04 (완료 안내) | 회원 | "변경 완료" 안내 + [로그인하기] |

**소셜 가입자**: 1단계에서 "소셜 계정은 각 서비스에서 비밀번호를 관리해주세요" 토스트만 표시되고 다이얼로그 안 열림.

## F-11 · 회원 탈퇴

| 단계 | 화면/모달 | 행위자 | 행동 |
|---|---|---|---|
| 1 | SCR-SET-01 (설정) | 회원 | 위험구역 "회원 탈퇴" 탭 |
| 2 | MOD-12 (회원 탈퇴 다이얼로그) | 회원 | 사유 라디오 5종 중 선택 + (이메일 가입자) 비밀번호 입력 → [탈퇴하기] |
| 3 | (파괴적 확인) | 회원 | "정말 탈퇴할까요? 되돌릴 수 없어요" → 확인 |
| 4 | (시스템) | — | 탈퇴 상태 표시 + 즉시 로그아웃 |
| 5 | SCR-AUTH-01 (로그인) | — | 다음 진입 시 일반 로그인 화면. 같은 이메일·전화로 30일 내 재가입 차단 (Phase 2 정책) |

**부수효과**: 탈퇴한 작가의 프로필을 다른 사용자가 진입하면 빈 화면 + "이 작가는 더 이상 활동하지 않습니다" 안내.

## F-12 · 운영팀 회원 정지

## 흐름 점검 체크리스트

새 흐름을 추가하거나 기존 흐름을 수정할 때 확인:

- [ ] 모든 단계에 화면/모달 ID가 명시되어 있나
- [ ] 비로그인·로그인·정지 상태별 분기가 빠지지 않았나
- [ ] 시스템 부수효과(포인트 적립·알림 발송·통계)가 표시되어 있나
- [ ] 가능한 이탈 지점·에러 경로가 메모되어 있나
- [ ] 흐름 종착이 명확한가 (어디에 도착하면 끝인가)
- [ ] 다른 흐름과의 연결 (예: F-05 → F-03)이 표시되어 있나
