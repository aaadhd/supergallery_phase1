# 06. PM·QA 데모 도구

> 이 문서 단독으로 이해 가능하도록 작성. 레퍼런스 명세를 이 문서 기준으로 업데이트해주세요.
> **이 카테고리 화면들은 실서비스에 노출하지 않음 — 배포 전 제거 또는 숨김 권장.**

---

## 사전 지식

### 이 카테고리 화면이 존재하는 이유

Phase 1은 PM이 PD·개발자·스테이크홀더에게 **플로우와 명세 일치 여부를 시연**해야 하므로:

1. 모든 화면 경로를 외울 수 없음 → 바로가기 맵 필요
2. 알림·정지·검수 같은 상태 변화를 즉시 시뮬해 보여줘야 함
3. 레퍼런스 명세에 있는 토큰·지역 분기·이메일 템플릿 등이 실제로 구현되어 있는지 한눈에 검증 가능해야 함

이 두 화면은 그 목적의 **PM 사이드 도구**이며, 일반 사용자에겐 노출하지 않음.

### 공통 정책

- 실서비스 배포 시 **라우트 제거 또는 환경 설정으로 숨김** 권장
- QA 바로가기 플로팅 버튼도 동일하게 환경 설정으로만 노출

## SCR-DEMO-01 · 플로우 데모 툴즈

| 항목 | 값 |
|---|---|
| URL | `/demo` |
| 접근 권한 | 누구나 (라우팅엔 제한 없음). **프로덕션 비노출 권장** |
| 진입 경로 | URL 직진입 · QA 플로팅 버튼 (개발·QA 환경에서만 보임) |
| 레이아웃 | 공통 Layout |

### 화면의 목적
PM이 시연할 때 **플로우 단축 맵 + 알림 시뮬레이터 + 어드민 게이트 해제 토글**을 한 페이지에서 제공.

### 사용자 시나리오
- 주 사용자: PM, QA, 데모 진행자
- 기대: 경로를 외우지 않고도 탭 한 번으로 화면 이동, 알림 시뮬로 UI 검증, 어드민 화면 빠른 진입

### 화면 구성

| 영역 | 요소 |
|---|---|
| 상단 | "PM 플로우 데모" 타이틀 + ⚠️ "프로덕션 비권장" 경고 배너 |
| **플로우 맵 블록 (15개 섹션)** | 각 섹션: 제목 + 경로 리스트. 경로는 고정폭 폰트로 표시, 탭 시 해당 화면으로 이동 |
| **알림 시뮬레이터** | 버튼 5종: 새 작품 / 좋아요 / Pick 선정 / 시스템 공지 / 이벤트. 탭 시 해당 타입 알림 추가 |
| **어드민 게이트 해제** | 스위치 — 프로덕션 빌드에서도 어드민 접근을 브라우저별로 열어두는 토글 |
| 추가 데모 링크 | 정지 시뮬·지역 분기 데모 등 시나리오별 서브뷰 단축 진입 |

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 플로우 맵 카드 내 경로 링크 | 해당 경로로 이동 (예: `/login`, `/upload`) |
| 알림 시뮬 버튼 5종 | 해당 타입 데모 알림 추가 — 화면 이동 없음. 헤더 벨 카운트 즉시 증가 |
| 어드민 게이트 해제 스위치 | 토글 — 화면 이동 없음. 이후 `/admin` 진입 시 비밀번호 게이트 우회 |
| 추가 데모 링크 | 시나리오별 서브뷰 경로로 이동 (예: `/login?demo=suspended`) |

### 플로우 맵 블록 형식

각 섹션은 텍스트 리소스로 정의:

```
섹션 제목
/path1 | 짧은 설명1
/path2 | 짧은 설명2
```

- 첫 줄 = 섹션 제목
- 이후 각 줄 = `경로 | 설명` (구분자 `|`)
- 렌더 결과: 카드 내부에 경로는 고정폭 폰트로 링크, 설명은 작은 회색 텍스트

15개 섹션 예시 (내용은 운영팀이 수정):

| # | 주제 |
|---|---|
| 1 | 인증·온보딩 |
| 2 | 둘러보기·검색 |
| 3 | 업로드 |
| 4 | 프로필 |
| 5 | 전시 |
| 6 | 이벤트 |
| 7 | 알림·설정 |
| 8 | 어드민 진입 |
| ... | ... |

### 알림 시뮬레이터 동작

각 버튼 탭 시 해당 타입의 데모 알림 추가:
- 알림 화면에 즉시 반영 (헤더 벨 카운트 증가)
- 데모 알림은 **알림 설정 토글과 무관하게 항상 표시**
- 시뮬에 사용되는 발신 작가는 시드 작가에서 가져옴

### 어드민 게이트 해제 토글

- ON → 현재 브라우저에 영구 해제 마킹 → 프로덕션 빌드에서도 어드민 진입 가능
- OFF → 해제 마킹 제거 → 다음 어드민 진입 시 비밀번호 게이트 다시 표시
- 초기값은 현재 마킹 상태 조회

### 검증·정책
- **실서비스 제거 권장**. 현재는 라우팅 항상 활성 (PM 시연 편의 우선)

### Phase 2 처리
- 실서비스 빌드 자동 제외
- QA 전용 계정만 접근 허용 (어드민 권한과는 별도)

## SCR-DEMO-02 · 레퍼런스 검수 툴킷

| 항목 | 값 |
|---|---|
| URL | `/demo/reference` |
| 접근 권한 | 누구나. **프로덕션 비노출 권장** |
| 진입 경로 | URL 직진입 · QA 플로팅 버튼 |
| 레이아웃 | 공통 Layout |

### 화면의 목적

레퍼런스 폴더 명세에 등장하는 **세션 토큰 관리 / 지역 분기 시뮬 / 이메일 템플릿 / 운영 체크리스트** 등을 개발자·QA가 한 화면에서 검수할 수 있도록 모은 도구.

### 사용자 시나리오
- 주 사용자: 개발자, QA, 운영팀 검수자
- 기대: 레퍼런스 명세 항목이 실제로 구현되어 있는지 빠르게 확인. 인수인계 시 "이거 되는 거 보여주세요" 시연용

### 탭·액션 결과 (네비게이션)

이 화면은 상단 탭 또는 수직 섹션으로 구분되어 한 페이지 안에서 모든 도구를 사용. **외부 이동 없음** — 모든 섹션이 같은 화면에서 동작. 이메일 템플릿 카드 탭 시 같은 화면 위에 미리보기 모달만 열림.

### 섹션 구성

#### 🔐 세션 토큰 섹션

| 요소 | 동작 |
|---|---|
| 현재 세션 표시 | 현재 로그인된 사용자의 토큰 정보 (주체·액세스·리프레시·만료시각) |
| 토큰 단축 표시 | 긴 토큰은 뒷부분 "..."으로 줄임 |
| 만료 여부 배지 | 만료면 빨간 배지 |
| [토큰 재발급] | 새 액세스 토큰 발행 |
| [세션 클리어] | 로그아웃 상태로 |

목적: 레퍼런스 명세의 액세스/리프레시 토큰 분리 정책이 동작하는지 확인.

#### 🌐 지역 분기 시뮬

| 요소 | 동작 |
|---|---|
| [IP 조회] | 현재 접속자의 국가·대륙·타임존 반환 |
| 캐시 정책 | 1시간 (같은 세션에서 반복 호출 방지) |

목적: "한국 접속자에겐 카카오 우선, 그 외에겐 구글/애플 우선" 같은 지역 분기 로직 시연.

#### 📧 이메일 템플릿 미리보기

10종 템플릿을 카드 그리드로, 각 카드 탭 시 본문 미리보기:

| 키 | 용도 |
|---|---|
| 가입 환영 | 가입 직후 |
| 이메일 인증 | 소유권 확인 |
| 비밀번호 재설정 | 분실 시 |
| 비밀번호 변경 완료 | 변경 후 |
| Pick 선정 축하 | 큐레이션 선정 시 |
| 주간 다이제스트 | 주간 활동 요약 |
| 약관 변경 안내 | 정책 변경 시 |
| 마케팅 | 선택 수신자만 |
| 신고 접수 | 신고자 |
| 신고 처리 결과 | 신고자 (Phase 2) |

각 템플릿은 제목·본문의 다국어 리소스 키 쌍. 미리보기는 단순 텍스트 (XSS 방지 정책 준수 — HTML 주입 안 함).

목적: 레퍼런스 명세에 정의된 10종 템플릿 카피가 모두 다국어 리소스에 들어 있는지 검증.

#### ✅ 운영 체크리스트

레퍼런스 항목을 표로 나열, 각 행에 수동 체크. 어드민 "출시 체크리스트"와 별개로 PM·개발자가 레퍼런스 대조용으로 사용.

### 검증·정책
- 실서비스 빌드 비노출 권장
- 실 API 연동 시 일부 섹션은 그대로 유지, 일부(이메일 템플릿 검수)는 어드민으로 이전

### Phase 2 처리

| 섹션 | Phase 2에서 |
|---|---|
| 세션 토큰 | 실 OAuth·세션 라이브러리 도입 시 도구는 폐기 (개발자 콘솔로 충분) |
| 지역 분기 | 실 API 연동 시 캐시 정책만 유지 |
| 이메일 템플릿 | 어드민 "이메일 발송 관리"로 이전, 실 발송 통계도 |
| 체크리스트 | 어드민 "출시 체크리스트"로 통합 |
