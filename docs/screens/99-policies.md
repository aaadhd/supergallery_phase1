# 99. 정책·수치 시트 (외부 전달용)

> 모든 화면 문서에 흩어져 있는 **확정 정책·수치**를 한 곳에 모은 부록.
> 받는 개발자가 비즈니스 룰을 한 번에 검토하고 본인 구현 기준으로 변환할 때 참조.
> **이 문서의 수치가 정본** — 화면 문서와 차이가 있다면 이 문서가 우선.

---

## 1. 가입·온보딩

| 항목 | 값 | 적용 |
|---|---|---|
| 비밀번호 | 8자 이상 + 영문 1자 + 숫자 1자 이상 | 모든 가입 경로 |
| 닉네임 | 2~20자, 비속어 사전 차단 | 모든 가입 경로 |
| 닉네임 중복 | 다른 사용자와 같으면 차단 | 모든 가입 경로 |
| 나이 제한 | 만 14세 이상 | 생년월일 검증으로 차단 |
| 이메일 형식 | 기본 정규식 (`@` + 도메인에 `.`) | 모든 입력 |
| 실명 | 모든 가입 경로에서 필수, 2자 이상 | 온보딩 |
| 전화번호 | 모든 가입 경로에서 필수, 숫자 10자 이상 + 중복 차단 | 온보딩 |
| 이메일 (온보딩) | 소셜 가입자만 필수 (이메일 가입자는 가입 폼에서 받음) | 온보딩 |
| 이메일 중복 가입 | 같은 이메일로 재가입 차단 | 가입 + 온보딩 |
| 전화 중복 가입 | 같은 전화로 재가입 차단 | 온보딩 |
| 약관 동의 | 이용약관 + 개인정보처리방침 필수 + 14세 확인 | 모든 가입 경로 |
| 마케팅 동의 | 이메일·푸시 각각 선택 | 가입 시 |

## 2. 작품 업로드

| 항목 | 값 |
|---|---|
| 전시당 이미지 수 | **최대 10장** |
| 파일 크기 | 10MB 초과 차단 |
| 허용 포맷 | JPEG / PNG / WebP / GIF |
| 최소 해상도 | 단변 **800픽셀** 이상 |
| 카메라 원본 | JPEG 메타데이터에 카메라 정보 있으면 거부 (휴대폰 직촬 차단) |
| 전시 제목 | 최대 **20자** |
| 그룹명 | 그룹 전시 시 필수 + 자동완성 (정규화 매핑) |
| 작품 제목 | 최대 **20자** (전시명·그룹명과 동일 상한, `TITLE_FIELD_MAX_LEN`) |
| 비속어 검사 | 전시명 · 그룹명에 적용 |
| 원본 동의 | 발행 전 체크박스 필수 ("본인 창작 + 저작권 보유") |
| 그룹 학생 동의 | 그룹 전시 발행 전 추가 체크박스 ("수강생 게시 동의받음") |

### 작품 자동 제목 생성 규칙

| 상황 | 자동 제목 |
|---|---|
| 그룹 + 작가 지정 있음 | "[작가명] 1번 그림" |
| 그룹 + 작가 자유 (셀프 등) | "1번 그림" |
| 개인 + 1장 | 전시명 사용 |
| 개인 + 여러 장 | "(무제)" |

### 검수 정책
- **모든 작품은 발행 직후 운영팀 검수 대기 → 승인 → 피드 노출**
- 작가 프로필 전시 탭 도착 시 "검수 후 공개 (보통 24시간 내)" 안내 배너 (닫기 가능)
- 데모·PM 시연용 검수 우회 모드 있지만 실서비스에선 끈 상태 필수

### 발행 후 동작
- 비회원 작가가 있으면 **발송 직전 confirm 모달** (실명 정확성 경고)
- 비회원 SMS 발송 후 작품에서 **전화번호 즉시 스크럽** (이름만 남김)
- 한국 번호는 카카오 알림톡, 그 외는 SMS

## 4. 피드 (둘러보기)

### 버킷·노출
같은 작품은 **하나의 버킷에만** 배정된다. 버킷은 위→아래 우선순위로 선점된다:

| 우선순위 | 버킷 | 진입 조건 | 상한 |
|---|---|---|---|
| 1 | Artier's Pick | 어드민이 현재 활성 Pick으로 지정 | 10 |
| 2 | 테마전 | 현재 테마전 목록에 포함된 작품 | 없음 |
| 3 | 추천 작가(featured) | 어드민이 추천 작가로 지정한 작가의 작품 | 없음 |
| 4 | 팔로우 개인화 | 로그인 유저가 팔로우 중인 작가의 작품 | 없음 |
| 5 | 신규 | 업로드 후 14일 이내 | 없음 |
| 6 | 일반 | 위 어디에도 해당하지 않음 | 없음 |

> `editorsPick`(이력 배지)은 Pick 버킷 진입 조건이 **아니다**. 배지만 남는 과거 Pick 수상 표시용.

### 버킷 내 정렬 (정확한 수식)
각 작품의 최종 점수는 다음 합산으로 계산한다.

```
작품 점수
  = log1p(좋아요 수)  × 2.2
  + log1p(저장 수)    × 3.2
  + (팔로우 중 작가이면 +3.5, 아니면 0)

최종 점수 = 작품 점수 + 소스 부스트 + 랜덤 노이즈
```

- **소스 부스트** (버킷별 고정값): Pick +14 · 테마전 +8 · 추천 작가 +6 · 팔로우 개인화 +7 · 신규 +3 · 일반 0
- **랜덤 노이즈** (새로고침마다 순서 흔듦, 균등분포 ±범위): Pick ±5 · 테마전 ±6 · 추천 작가 ±6.5 · 신규 ±7.5 · 일반 ±10 (우선순위 버킷일수록 노이즈 좁음)
- **댓글은 사용하지 않음** (Phase 1 Out of Scope)
- **이미 본 작품**(세션 노출 이력) ID는 같은 버킷 안에서 뒤로 밀림

### 최종 인터리브 패턴
각 버킷을 위에서 아래로 단순 합치지 않고, 아래 패턴을 한 바퀴씩 돌며 하나씩 꺼내어 섞는다:

```
Pick → 일반 → 테마전 → 일반 → 팔로우 → 일반 → 추천 → 일반 → 신규 → 일반  (반복)
```

우선순위 버킷 사이에 매번 "일반"이 끼어들어, Pick만 연속으로 쏟아지는 현상을 방지한다.

### 순서 재계산 타이밍
둘러보기 피드 순서는 다음 경우에만 다시 계산한다(좋아요·저장 같은 상호작용 때는 **재계산 안 함** — 시니어 UX에서 카드가 점프하지 않도록):
- 피드 내 작품 수 변화
- 사용자 수동 새로고침
- 팔로우 수 변화

### 페이지네이션
- 한 번에 24개 카드 로드
- 스크롤 바닥 감지 시 24개씩 추가

### 노출 제외
- 작가가 비공개 처리한 작품
- 검수 대기 / 반려 작품
- 운영팀이 숨김 처리한 작품
- 본인이 신고한 작품 (같은 사용자에겐 자동으로 안 보임)

## 7. 신고·정지

### 신고 처리 액션 4종 (어드민)

| 액션 | 효과 | 색 |
|---|---|---|
| **삭제** | 작품 영구 삭제 (작품 신고 한정) | 빨강 |
| **경고** | 작가에게 경고 +1 누적 (3회 누적 → 자동 7일 정지) | 주황 |
| **기각** | 신고자에게 허위 신고 +1 누적 (3회 누적 → 신고자 7일 차단) | 중립 |
| **비공개** | 작품을 피드·검색에서 숨김 (작가 본인 프로필엔 보임) | 회색 |

### 정지 단계 4종 (어드민 회원 관리)

| 레벨 | 기간 | 표시 |
|---|---|---|
| 주의 | — | 정지 없이 경고 표시만. 누적 시 정지로 승격 |
| 7일 | 7일 | 시한부 |
| 30일 | 30일 | 시한부 |
| 영구 | ∞ | 무기한 |

### 자동 정지 승격 룰
- 경고 누적 **3회** → 자동 7일 정지로 전환
- 허위 신고 누적 **3회** → 신고자 7일 차단

### 중복 방지
- (신고자 + 대상 + 사유) 조합으로 같은 신고 두 번 차단 → 토스트 "이미 신고하셨습니다"

### 신고자 피드백 정책
- Phase 1: 신고 처리 결과를 신고자에게 알리지 않음 (운영팀 부담·역풍 방지)
- Phase 2: 재검토

## 17. Phase 1 → Phase 2 외부 연동 매핑

받는 개발자가 실서비스 구축 시 Phase 1 모의를 어떻게 실연동으로 바꿔야 하는지:

| 항목 | Phase 1 (현재) | Phase 2 (실연동) |
|---|---|---|
| 소셜 로그인 (카카오·구글·애플) | 버튼 누르면 즉시 통과 | 실 OAuth + 토큰 교환 |
| 이메일 로그인 | 어떤 이메일이든 통과 | 이메일 소유권 인증 + 비밀번호 검증 |
| 비밀번호 재설정 | 토스트만 표시 | 실 메일 발송 + 토큰 링크 (30분 만료) |
