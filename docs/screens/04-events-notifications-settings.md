- **확인 다이얼로그**: 앱 자체 디자인. 브라우저 네이티브 금지

## SCR-EV-01 · 이벤트 목록

| 항목 | 값 |
|---|---|
| URL | `/events` |
| 접근 권한 | 비로그인 가능 |
| 진입 경로 | 하단 네비 "이벤트" · 둘러보기 배너 · 푸터 |
| 레이아웃 | 공통 Layout |

### 화면의 목적
운영팀이 등록한 이벤트(공모전·캠페인·신청 등)를 진행 중·예정·종료별로 묶어 보여주기.

### 사용자 시나리오
- 주 사용자: 참여 의사 있는 작가
- 기대: 진행 중인 이벤트 한눈에, 마음에 드는 것 탭해 상세 보기

### 화면 구성

| 영역 | 요소 |
|---|---|
| 상단 배너 캐러셀 | 자동 전환(3~5초). 시니어 친화 — 일시정지 가능 |
| **진행 중 섹션** | 진행 중 상태 이벤트 카드 그리드 |
| **예정 섹션** | 예정 상태 — 시작일 배지 |
| **종료 섹션 (접힘)** | 종료 — "더 보기"로 펼침 |
| 빈 상태 | "진행 중인 이벤트가 없어요" + 다른 화면 추천 |

### 카드 요소
- 배너 이미지
- 제목
- 기간 (시작일 ~ 종료일)
- 상태 배지 (진행 중 / 예정 / 종료 — 색상 구분)

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 상단 배너 캐러셀 | 해당 이벤트 상세(`/events/:id`)로 이동 |
| 캐러셀 좌우 화살표 / 스와이프 | 같은 화면에서 다른 배너 보기 |
| 진행 중·예정 섹션 카드 | 이벤트 상세(`/events/:id`)로 이동 |
| 종료 섹션 "더 보기" | 같은 화면에서 종료 이벤트 목록 펼침 |
| 빈 상태 CTA (있다면) | 둘러보기(`/`)로 이동 |

### 수용기준 (Acceptance Criteria)
- AC-01: Given 이벤트가 1건 이상 존재할 때 / When 사용자가 `/events`에 진입하면 / Then 진행 중·예정·종료 섹션이 상태별로 분리되어 표시된다
- AC-02: Given 진행 중인 이벤트가 있을 때 / When 화면이 렌더링되면 / Then 진행 중 섹션이 최상단(배너 캐러셀 아래)에 노출되고, 각 카드에 배너 이미지·제목·기간·상태 배지가 표시된다
- AC-03: Given 종료된 이벤트가 존재할 때 / When 화면이 렌더링되면 / Then 종료 섹션은 기본 접힌 상태이며 "더 보기"를 탭하면 펼쳐진다
- AC-04: Given 이벤트가 0건일 때 / When 사용자가 `/events`에 진입하면 / Then "진행 중인 이벤트가 없어요" 빈 상태 메시지가 표시된다
- AC-05: Given 상단 배너 캐러셀이 노출될 때 / When 3~5초가 경과하면 / Then 다음 배너로 자동 전환되며, 일시정지 버튼으로 자동 전환을 멈출 수 있다
- AC-06: Given 이벤트 카드를 탭할 때 / When 진행 중 또는 예정 카드를 탭하면 / Then 해당 이벤트 상세(`/events/:id`)로 이동한다
- AC-07: Given 비로그인 사용자일 때 / When `/events`에 진입하면 / Then 로그인 없이 이벤트 목록을 정상 조회할 수 있다

### 엣지케이스
- EC-01: 이벤트 종료일이 현재 시각과 정확히 같은 경우 → "종료" 상태로 판정한다 (종료일 ≤ 현재)
- EC-02: 시작일과 종료일이 같은 이벤트 → 해당 일자에만 "진행 중"으로 표시되고, 이후 즉시 "종료"로 전환된다
- EC-03: 배너 캐러셀에 이미지가 1건뿐인 경우 → 자동 전환·좌우 화살표 비노출, 단일 배너만 표시
- EC-04: 진행 중 이벤트만 있고 예정·종료가 없는 경우 → 예정·종료 섹션 자체를 숨기고 빈 구역을 남기지 않는다
- EC-05: 이벤트 제목이 매우 긴 경우 → 카드에서 말줄임(ellipsis) 처리하여 레이아웃 깨짐을 방지한다

### Phase 2 보강
- 알림 받기 신청 → 이벤트 시작 시 푸시
- 카테고리 필터
- 즐겨찾기

## SCR-EV-02 · 이벤트 상세

| 항목 | 값 |
|---|---|
| URL | `/events/:id` |
| 접근 권한 | 비로그인 가능 (참여는 로그인) |
| 진입 경로 | 이벤트 목록 카드 · 배너 · 검색 |
| 레이아웃 | 공통 Layout |

### 화면의 목적
이벤트 본문 + 참여 방법 + 관련 작품 갤러리 + 참여 CTA.

### 화면 구성

| 영역 | 요소 |
|---|---|
| 헤로 | 큰 배너 + 그라디언트 오버레이 + 상태 배지 + 제목 + 부제 |
| [목록으로] | 좌상단 뒤로가기 링크 |
| 메타 카드 | 기간 + 대상 |
| 안내 | "참여 방법" 제목 + 본문 |
| 알림 안내 | "이벤트 시작·종료를 알림으로 받으려면" + 알림 설정 링크 |
| **[참여하기]** | 진행 중 상태에서만 활성. 종료면 "종료된 이벤트입니다" 비활성 박스 |

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| [목록으로] (좌상단 뒤로가기) | **이벤트 목록(`/events`)**으로 이동 |
| 알림 안내 영역 "알림 설정" 링크 | **설정 알림 섹션(`/settings#notifications`)**으로 이동 |
| **[참여하기]** | 아래 "참여 액션 동작" 흐름 진행 |
| 종료 상태 참여 버튼 | 비활성화 — 탭해도 반응 없음 ("종료된 이벤트입니다" 표시) |

### 참여 액션 동작

1. 비로그인 → 로그인 유도 모달 (로그인 성공 시 현재 화면으로 복귀)
2. 이미 같은 이벤트에 작품 제출했음 → 화면 이동 없이 토스트 "이미 참여했어요"
3. 통과 시 → **업로드 화면(`/upload?event={id}&eventTitle=...`)**으로 이동. 업로드 발행 시 작품이 해당 이벤트와 자동 연결

### 이벤트 응모 → 어드민 추적
사용자가 이벤트 상세에서 "참여"로 업로드한 작품은 자동으로 해당 이벤트와 연결됨. 어드민 "이벤트 참여자" 화면에서 실 업로드 작품을 참여자로 자동 조회 가능 (07 문서 SCR-ADM-05 참조).

### 알림 받기 (Phase 2 정식 도입 예정)

현재 이벤트 상세 안내 영역에 "이벤트 시작·종료를 알림으로 받으려면" 텍스트 + 알림 설정 링크가 있음. **이벤트별 개별 알림 구독은 Phase 2 영역**:

| 항목 | Phase 1 | Phase 2 |
|---|---|---|
| 알림 받기 신청 버튼 | 없음 (안내 텍스트만) | 이벤트 카드/상세에 [알림 받기] 버튼 |
| 시작 알림 | 운영팀이 수동으로 일괄 발송 | 신청자에게 자동 푸시 |
| 종료/마감 임박 알림 | 동일 | 자동 |
| 구독 관리 | 없음 | 설정 화면에서 내가 알림 받는 이벤트 목록 조회·해제 |

### 수용기준 (Acceptance Criteria)
- AC-01: Given 진행 중인 이벤트 상세에 진입했을 때 / When 화면이 렌더링되면 / Then 배너 이미지·상태 배지·제목·부제·기간·대상·안내 본문이 표시되고 [참여하기] 버튼이 활성 상태이다
- AC-02: Given 종료된 이벤트 상세에 진입했을 때 / When 화면이 렌더링되면 / Then [참여하기] 버튼이 비활성 상태이며 "종료된 이벤트입니다" 텍스트가 표시된다
- AC-03: Given 비로그인 사용자가 [참여하기]를 탭했을 때 / When 로그인 유도 모달이 표시되면 / Then 로그인 성공 후 현재 이벤트 상세 화면으로 복귀한다
- AC-04: Given 로그인 사용자가 [참여하기]를 탭했을 때 / When 해당 이벤트에 이미 작품을 제출한 적이 있으면 / Then 화면 이동 없이 "이미 참여했어요" 토스트가 표시된다
- AC-05: Given 로그인 사용자가 [참여하기]를 탭했을 때 / When 미참여 상태이면 / Then 업로드 화면(`/upload?event={id}&eventTitle=...`)으로 이동하며 발행 시 이벤트와 자동 연결된다
- AC-06: Given 좌상단 [목록으로] 링크를 탭했을 때 / When 탭하면 / Then 이벤트 목록(`/events`)으로 이동한다
- AC-07: Given 알림 안내 영역의 "알림 설정" 링크를 탭했을 때 / When 탭하면 / Then 설정 알림 섹션(`/settings#notifications`)으로 이동한다

### 엣지케이스
- EC-01: 이벤트 상세 진입 시점에는 진행 중이었으나 체류 중 종료 시각이 지난 경우 → [참여하기] 탭 시 서버/스토어 상태 재확인 후 "종료된 이벤트입니다" 안내를 표시한다
- EC-02: 비로그인 → 로그인 유도 모달에서 로그인하지 않고 닫은 경우 → 이벤트 상세 화면이 그대로 유지된다
- EC-03: 이벤트 배너 이미지 로드 실패 시 → 기본 플레이스홀더 이미지를 표시하고 나머지 콘텐츠는 정상 렌더링한다
- EC-04: 삭제되었거나 존재하지 않는 이벤트 ID로 직접 URL 진입 시 → 빈 화면 대신 "이벤트를 찾을 수 없어요" 안내 + 목록 이동 링크를 표시한다

### Phase 2 보강
- 1인당 작품 개수 제한
- 마감 임박 알림 (위)
- 참여자 수 라이브 표시
- 알림 받기 구독 (위)

## SCR-NOTI-01 · 알림

| 항목 | 값 |
|---|---|
| URL | `/notifications` |
| 접근 권한 | 로그인 필수 |
| 진입 경로 | 헤더 벨 아이콘 (안 읽음 카운트 배지) |
| 레이아웃 | 공통 Layout |

### 화면의 목적
좋아요·팔로우·Pick·시스템·이벤트 알림을 모아보기. 탭 시 원문으로 이동.

### 사용자 시나리오
- 주 사용자: 회원 모두
- 기대: 누가 좋아요/팔로우 했는지, 공지가 왔는지, 내 이벤트 시작했는지 확인

### 알림 타입별 아이콘·예시

| 타입 | 아이콘 | 예시 메시지 |
|---|---|---|
| 좋아요 | 하트 | "○○님이 회원님 작품에 좋아요를 눌렀어요" |
| 팔로우 | 사람+플러스 | "○○님이 회원님을 팔로우해요" |
| Pick | 별 | "회원님 작품이 Pick에 선정되었어요" |
| 시스템 | 벨 | "검수가 완료되었어요 / 정지 안내 / 공지" |
| 이벤트 | 달력 | "관심 이벤트가 시작되었어요" |

### 화면 구성

| 영역 | 요소 |
|---|---|
| 상단 | "알림" 타이틀 + [모두 읽음] 버튼 |
| 알림 카드 리스트 | 아바타(있으면) + 메시지 + 상대 시간 ("3시간 전"). 안 읽음은 배경 강조 |
| 비어 있음 | "새 알림이 없어요" |
| 하단 링크 | "알림 설정" |

### 보관·정리 정책
- 화면 진입 시 90일 지난 항목 자동 제거
- 200건 초과면 오래된 것부터 잘림

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 알림 카드 — **좋아요/저장** | 해당 **작품 상세 모달** 열림 + 읽음 처리 |
| 알림 카드 — **팔로우** | 해당 **작가 프로필(`/profile/:id`)** 로 이동 + 읽음 처리 |
| 알림 카드 — **Pick** | 해당 작품 상세 모달 열림 (또는 둘러보기 Pick 섹션) + 읽음 처리 |
| 알림 카드 — **시스템(검수 결과)** | 해당 작품 있으면 내 프로필 작품 탭으로 이동 (반려 배지 확인 유도) |
| 알림 카드 — **이벤트** | 해당 **이벤트 상세(`/events/:id`)** 로 이동 + 읽음 처리 |
| [모두 읽음] | 확인 다이얼로그 → 전체 읽음 처리 (화면 이동 없음) |
| 슬라이드·롱프레스 (Phase 2) | 개별 삭제 |
| 하단 "알림 설정" | **설정 알림 섹션(`/settings#notifications`)** 으로 이동 |

### 알림 설정과 연동
알림 설정에서 OFF한 타입은 **새로 쌓이지 않음** (기존 히스토리는 남음). 시스템 타입은 OFF 불가 (중요 공지 보장).

### 수용기준 (Acceptance Criteria)
- AC-01: Given 로그인 사용자가 `/notifications`에 진입했을 때 / When 알림이 1건 이상 존재하면 / Then 알림 카드 리스트가 최신순으로 표시되며 안 읽은 항목은 배경 강조로 구분된다
- AC-02: Given 안 읽은 알림이 있을 때 / When 해당 알림 카드를 탭하면 / Then 읽음 처리되고 타입에 따라 올바른 화면으로 이동한다 (좋아요→작품 상세, 팔로우→작가 프로필, Pick→작품 상세, 시스템→내 프로필 작품 탭, 이벤트→이벤트 상세)
- AC-03: Given [모두 읽음] 버튼을 탭했을 때 / When 확인 다이얼로그에서 확인을 누르면 / Then 모든 알림이 읽음 처리되고 배경 강조가 해제된다
- AC-04: Given 알림이 0건일 때 / When `/notifications`에 진입하면 / Then "새 알림이 없어요" 빈 상태 메시지가 표시된다
- AC-05: Given 화면 진입 시 / When 90일 이상 지난 알림이 존재하면 / Then 해당 항목이 자동 제거된다
- AC-06: Given 알림이 200건을 초과할 때 / When 새 알림이 추가되면 / Then 가장 오래된 항목부터 잘려 200건을 유지한다
- AC-07: Given 비로그인 사용자가 `/notifications`에 접근할 때 / When 접근하면 / Then 로그인 화면으로 리다이렉트된다
- AC-08: Given 하단 "알림 설정" 링크를 탭했을 때 / When 탭하면 / Then 설정 알림 섹션(`/settings#notifications`)으로 이동한다

### 엣지케이스
- EC-01: 알림에 연결된 작품이 이미 삭제된 경우 → 탭 시 작품 상세 모달 대신 "삭제된 작품입니다" 안내 토스트를 표시하고 화면 이동하지 않는다
- EC-02: 알림에 연결된 작가가 탈퇴한 경우 → 탭 시 빈 프로필 + 탈퇴 안내가 표시된다
- EC-03: [모두 읽음] 확인 다이얼로그에서 취소를 누른 경우 → 읽음 처리하지 않고 현재 상태를 유지한다
- EC-04: 알림 설정에서 특정 타입을 OFF한 후 다시 ON한 경우 → OFF 기간 중 발생한 알림은 소급 생성되지 않으며 ON 이후 새 항목만 쌓인다
- EC-05: 동일 사용자가 같은 작품에 좋아요를 반복(취소 후 재좋아요)하는 경우 → 알림이 중복 생성되지 않도록 최신 1건만 유지한다

### 화면의 목적
계정·알림·접근성·법적 페이지·회원 탈퇴를 한 화면에 정리. 시니어가 메뉴 깊이 들어가지 않도록 평면 구조.

### 섹션 구성

| 섹션 | 항목 |
|---|---|
| 계정 | 이메일 표시 / **비밀번호 재설정** / 로그아웃 |
| **알림** (앵커 `#notifications`) | 타입별 토글: 좋아요 · 팔로우 · Pick · 시스템(고정 ON, 변경 불가) · 이벤트 · 마케팅 이메일 · 마케팅 푸시 |
| 접근성 | **글자 크기 옵션** — 작게 / 보통 / 크게 (라디오 3단) |
| 언어 | 한국어 / 영어 토글 |
| 법적 | 이용약관 / 개인정보처리방침 / 고객센터 / FAQ / 공지사항 (각 화면 링크) |
| 데이터 (Phase 2) | 데이터 내보내기 / 데이터 삭제 요청 |
| **위험구역** | 회원 탈퇴 |

### 비밀번호 재설정 동작

계정 섹션의 "비밀번호 재설정"을 탭하면:

- **이메일 가입자**: 현재 로그인된 계정 이메일을 보여주며 "이 이메일로 재설정 링크를 보내드려요" 안내 다이얼로그 → [재설정 링크 보내기] 버튼 → 토스트 "○○로 재설정 링크를 보냈어요"
- **소셜 가입자**: "소셜 계정은 각 서비스(카카오·구글·애플)에서 비밀번호를 관리해주세요" 토스트 (다이얼로그 안 열림)

레퍼런스에는 비밀번호 재설정 화면으로 바로 링크 이동이었지만, **로그인 상태에서 비로그인 전용 화면으로 이동하는 어색한 UX**를 정리한 결정.

### 글자 크기

- 옵션: 작게 / 보통(기본) / 크게 3단
- 선택 즉시 전체 앱 텍스트에 반영 + 토스트로 확인 ("글자 크기가 변경됐어요")
- 시니어 접근성의 핵심 기능

### 회원 탈퇴 흐름

1. "회원 탈퇴" 탭 → 다이얼로그
2. **탈퇴 사유 라디오 선택** (5종):
   - 잘 안 쓰여서
   - 다른 서비스 이용
   - 개인정보 우려
   - 사용이 어려움
   - 기타
3. 비밀번호 확인 입력 (이메일 가입자만)
4. 파괴적 확인 다이얼로그로 최종 확인
5. 탈퇴 처리:
   - 이 사용자는 "탈퇴" 상태로 표시 → 다른 사용자가 이 프로필 진입 시 빈 화면 + 안내
   - 즉시 로그아웃
6. Phase 2에선 서버 삭제 요청까지 (Phase 1은 로컬 플래그만)

### 알림 설정 영속화
타입별 ON/OFF 상태는 기기에 저장. 시스템 타입은 항상 ON.

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| **비밀번호 재설정** 행 | 이메일 가입자는 같은 화면 위에 **비밀번호 재설정 안내 다이얼로그** 열림. 소셜 가입자는 화면 이동 없이 안내 토스트만 |
| 다이얼로그 [재설정 링크 보내기] | 다이얼로그 닫힘 + 토스트 "○○로 재설정 링크를 보냈어요" (화면 이동 없음) |
| **로그아웃** 행 | 로그아웃 처리 → **로그인(`/login`)** 으로 이동 |
| 알림 토글 (각 항목) | 화면 이동 없음 — 즉시 저장 |
| 글자 크기 라디오 (작게/보통/크게) | 화면 이동 없음 — 전체 앱에 즉시 반영 + 확인 토스트 |

| 법적 섹션 — **개인정보처리방침** | `/privacy`로 이동 |

| **회원 탈퇴** | 같은 화면 위에 **탈퇴 다이얼로그** 열림. 사유 선택 + 비밀번호(이메일 가입자) + 최종 파괴적 확인 → 처리 후 로그아웃 → 로그인(`/login`)으로 이동 |

### 수용기준 (Acceptance Criteria)
- AC-01: Given 로그인 사용자가 `/settings`에 진입했을 때 / When 화면이 렌더링되면 / Then 계정·알림·접근성·언어·법적·위험구역 섹션이 평면 구조로 모두 표시된다
- AC-02: Given 계정 섹션에서 이메일 가입자가 "비밀번호 재설정"을 탭했을 때 / When 안내 다이얼로그가 열리면 / Then 현재 로그인 이메일이 표시되고 [재설정 링크 보내기] 버튼이 있다
- AC-03: Given 소셜 가입자가 "비밀번호 재설정"을 탭했을 때 / When 탭하면 / Then 다이얼로그 없이 "소셜 계정은 각 서비스에서 비밀번호를 관리해주세요" 토스트만 표시된다
- AC-04: Given 언어 토글을 변경했을 때 / When 한국어↔영어를 전환하면 / Then 화면 이동 없이 전체 앱 텍스트가 즉시 전환된다
- AC-05: Given 글자 크기 라디오를 변경했을 때 / When 작게/보통/크게 중 하나를 선택하면 / Then 전체 앱 텍스트 크기가 즉시 반영되고 "글자 크기가 변경됐어요" 토스트가 표시된다
- AC-06: Given 알림 타입별 토글을 변경했을 때 / When ON/OFF를 전환하면 / Then 즉시 저장되며 시스템 타입은 항상 ON 고정으로 토글 불가하다
- AC-07: Given "로그아웃"을 탭했을 때 / When 로그아웃 처리되면 / Then 로그인 화면(`/login`)으로 이동한다
- AC-08: Given 회원 탈퇴를 진행할 때 / When 사유 선택 → 비밀번호 확인(이메일 가입자) → 파괴적 확인 다이얼로그 확인 순서를 거치면 / Then 탈퇴 처리 후 즉시 로그아웃되고 로그인 화면으로 이동한다
- AC-09: Given 법적 섹션 링크를 탭했을 때 / When 이용약관·개인정보처리방침·고객센터·FAQ·공지사항 중 하나를 탭하면 / Then 각각 올바른 경로(`/terms`, `/privacy`, `/contact`, `/faq`, `/notices`)로 이동한다
- AC-10: Given `#notifications` 앵커로 진입했을 때 / When `/settings#notifications`로 접근하면 / Then 알림 섹션이 화면 상단에 스크롤되어 보인다
- AC-11: Given 비로그인 사용자가 `/settings`에 접근할 때 / When 접근하면 / Then 로그인 화면으로 리다이렉트된다

### 엣지케이스
- EC-01: 회원 탈퇴 다이얼로그에서 사유를 선택하지 않고 다음 단계로 진행하려는 경우 → 사유 미선택 시 진행 버튼이 비활성 상태를 유지한다
- EC-02: 이메일 가입자가 탈퇴 시 비밀번호를 잘못 입력한 경우 → 오류 메시지("비밀번호가 일치하지 않아요")를 표시하고 재입력을 유도한다

## SCR-POINTS · 포인트 잔액·원장 (Phase 2 정식 공개 예정)

| 항목 | 값 |
|---|---|
| URL | `/points` |
| 현재 동작 | **둘러보기(`/`)로 즉시 리다이렉트** — Phase 1에선 가려져 있음 |

### 화면의 목적 (Phase 2 시점)
사용자가 적립한 **AP(활동 포인트)**·**PP(프리미엄 포인트)** 잔액 + 적립·차감 원장을 조회. 가입·온보딩·발행·일일 둘러보기 등으로 적립되는 포인트를 투명하게 확인.

### Phase 1 상태
- 포인트 적립·회수 로직은 동작 중 (가입 +50, 온보딩 +30, 작품 발행 +100/+20 등)
- 잔액·원장 데이터도 기록되고 있음
- 다만 **사용자 노출 화면은 Phase 2까지 가려둠** — 상품·교환 시스템이 같이 갖춰져야 의미 있음

### Phase 2 화면 구성 (예정)

| 영역 | 요소 |
|---|---|

### 수용기준 (Acceptance Criteria)
- AC-01: Given Phase 1에서 사용자가 `/points`로 직접 진입했을 때 / When URL이 로드되면 / Then 즉시 둘러보기(`/`)로 리다이렉트된다
- AC-02: Given 포인트 적립 이벤트(가입·온보딩·발행 등)가 발생했을 때 / When 적립 로직이 실행되면 / Then 잔액·원장 데이터가 정상 기록된다 (화면 미노출이나 데이터 정합성은 유지)
- AC-03: Given 작품 발행 후 24시간 이내 삭제했을 때 / When 삭제가 실행되면 / Then AP -20 회수가 원장에 기록된다
