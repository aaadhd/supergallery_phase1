# 01. 인증·온보딩

> 이 문서 단독으로 이해 가능하도록 작성. 레퍼런스 명세를 이 문서 기준으로 업데이트해주세요.

---

## 사전 지식

### 이 카테고리에 적용되는 공통 정책

- **주 사용자**: 50대 액티브 시니어 순수미술 작가. 터치 영역과 가독성에 민감. 단계가 깊으면 이탈.
- **레이아웃**: 로그인·가입·온보딩·비밀번호 재설정·점검 5개 화면은 **헤더/푸터/하단 네비게이션 없는 집중 플로우**. 화면 정중앙 단일 컬럼(약 440px 너비).
- **만 14세 미만 가입 불가**: 모든 가입 경로에서 생년월일 검증으로 차단.
- **비속어 차단**: 닉네임·실명 입력 시 사전 기반 검사.
- **비밀번호 정책**: 8자 이상 + 영문 1자 + 숫자 1자 이상.
- **중복 가입 차단**: 이메일·전화번호 모두 중복 불가 (가입 완료된 이메일·전화 레지스트리 기준).

### 가입 경로 세 가지

| 경로 | 진입 → 가입 → 온보딩 |
|---|---|
| **이메일 가입** | 가입 화면 폼 입력 → 온보딩 |
| **소셜 첫 가입** | 로그인 화면에서 카카오/구글/애플 클릭 → 약관·닉네임 받는 모달 → 온보딩 |
| **SMS 비회원 초대 가입** | 작가가 비회원의 전화번호로 초대 발송 → 초대받은 사람이 SMS 링크 클릭 → 초대 랜딩 → 가입하기 → 가입 폼 → 온보딩 |

**모든 가입 경로의 종착지는 온보딩 화면**. 온보딩에서 **실명·전화는 모든 경로 필수**, **이메일은 소셜 가입자만 필수** (이메일 가입자는 가입 폼에서 이미 받음).

### 가입 경로 → 온보딩 Prefill

가입 과정에서 이미 받은 정보는 온보딩 입력란에 자동 채워져 있음 (사용자가 수정 가능):

| 경로 | Prefill되는 필드 |
|---|---|
| 이메일 가입 | 이메일 (가입 폼에서 입력한 값) |
| 소셜 첫 가입 | 닉네임 (약관 모달에서 입력한 값) + provider가 제공했으면 이메일 |
| SMS 초대 | 전화번호 + 실명 (초대 시 작가가 입력한 표시명) |

온보딩 완료 시 이 prefill 정보는 일괄 정리됨.

## SCR-AUTH-01 · 로그인

| 항목 | 값 |
|---|---|
| URL | `/login` |
| 쿼리 파라미터 | `?redirect={path}` (성공 시 이동할 경로) |
| 접근 권한 | 비로그인 전용 (이미 로그인이면 redirect 경로로 자동 이동) |
| 진입 경로 | 헤더 "로그인" 버튼 · 비로그인 사용자가 좋아요/저장/팔로우/업로드 시도 시 로그인 유도 모달에서 이동 · 세션 만료 자동 리다이렉트 |
| 레이아웃 | 집중 플로우 (헤더/푸터 없음) |
| 다국어 | 한국어·영어 |

### 화면의 목적
기존 회원의 재진입 주 경로. 소셜 로그인을 1차로 크게 노출하고 이메일 로그인은 접힘 상태(2차)로 숨김. 시니어가 한 번의 탭으로 빠르게 진입하는 것이 우선.

### 사용자 시나리오
- 재방문 회원의 대부분은 소셜 로그인 사용 → 로고 보고 자기 소셜 버튼 한 번 탭 → 원래 보던 화면 복귀
- 이메일 가입자(소수)는 "이메일로 로그인" 버튼 펼쳐서 사용

### 화면 구성

| 영역 | 요소 | 비고 |
|---|---|---|
| 상단 브랜드 | 로고 + "Artier" 텍스트 | 탭 시 둘러보기로 이동 |
| 소셜 버튼 3종 (1차) | 카카오 · 구글 · 애플 | **순서는 접속 언어에 따라 분기**: 한국어는 카카오·구글·애플 / 영어 등은 구글·애플·카카오 |
| 구분선 | "또는" | 상하 separator |
| 이메일 로그인 토글 (2차) | 메일 아이콘 + "이메일로 로그인" 버튼 | 탭 시 폼 펼침 |
| 이메일 폼 (펼친 후) | 이메일 · 비밀번호(보임/숨김 토글) · 로그인 버튼 · "비밀번호 찾기" 링크 | 펼침 시 이메일 입력란에 자동 포커스 |
| 연령 안내 | "만 14세 미만은 가입할 수 없습니다" | 작은 글씨 |
| 회원가입 안내 | "계정이 없으신가요? 회원가입" | 회원가입 화면으로 이동 |

### 상태·분기

| 상태 | 조건 | 표현 |
|---|---|---|
| 로딩 | 버튼 탭 후 잠시 | 버튼 텍스트가 "로그인 중..."으로 바뀌고 비활성화 |
| 이메일 오류 | 빈 값 / 형식 불일치 | 이메일 입력란 아래 빨간 텍스트 |
| 비밀번호 오류 | 빈 값 / 8자 미만 | 비밀번호 입력란 아래 빨간 텍스트 |
| 정지 상태 | 계정이 정지되어 있음 | 폼 하단 중앙 빨간 텍스트로 사유·해제일·이의제기 안내 조합 표시. 로그인 시도 자체 차단 |
| 소셜 첫 가입 분기 | 처음 누른 provider | 약관·닉네임 모달 오픈 |
| 소셜 재방문 | 이전에 그 provider로 가입한 적 있음 | 모달 없이 즉시 로그인 후 redirect 경로로 이동 |

### 정지 상태 차단 메시지 조합
- "이 계정은 일시 정지되었어요"
- (사유가 있으면) "사유: ○○○"
- (해제일이 있으면) "해제 예정: 2026-04-22" (사용자 언어에 맞는 날짜 표기)
- "이의가 있으시면 고객센터로 문의해주세요"

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 로고 | 둘러보기(`/`)로 이동 |
| 카카오/구글/애플 버튼 | **재방문자**: 즉시 로그인 → `?redirect=` 경로 or 둘러보기(`/`)로 이동. <br>**첫 가입자**: 같은 화면 위에 **소셜 첫 가입 모달** 열림. 모달 완료 시 **온보딩(`/onboarding`)**으로 이동 |
| "이메일로 로그인" 버튼 (접힌 상태) | **같은 화면에서 폼 펼침** — URL 변화 없음 |
| 이메일 폼 [로그인] | 검증 통과 시 로그인 처리 후 `?redirect=` 경로 or `/`로 이동 |
| 비밀번호 보임/숨김 아이콘 | 같은 화면에서 입력란 타입만 전환 |
| "비밀번호 찾기" | **비밀번호 재설정(`/reset-password`)**으로 이동 |
| "회원가입" | **가입 화면(`/signup`)**으로 이동 |
| 정지 상태에서 로그인 시도 | 화면 이동 없음 — 폼 하단에 정지 안내 메시지 표시 |

### 검증·정책

- 이메일: 기본 형식 (`@` 포함, 도메인에 `.` 포함)
- 비밀번호: 8자 이상 (단순 길이)
- 정지 차단은 소셜·이메일 모두 적용

### Phase 2 보강
- 카카오·구글·애플 실제 OAuth 연동
- 이메일·비밀번호 실서버 검증
- 로그인 실패 누적 시 잠금 (브루트포스 방지)
- 2단계 인증 옵션

## SCR-AUTH-02 · 회원가입

| 항목 | 값 |
|---|---|
| URL | `/signup` |
| 쿼리 파라미터 | `?invited=1` (SMS 초대 경로 — 플래그 세팅용) |
| 접근 권한 | 비로그인 전용 |
| 진입 경로 | 로그인 화면 "회원가입" 링크 · SMS 초대 랜딩의 "가입하기" |
| 레이아웃 | 집중 플로우 |

### 화면의 목적
이메일 가입 3단계 폼. **소셜 가입자는 이 화면을 거치지 않음** (약관 모달에서 닉네임만 받고 온보딩으로 바로).

### 사용자 시나리오
- 이메일·비밀번호로 가입하려는 신규 사용자 (전체의 약 10%)
- 기대: 단계별 진행으로 부담 분산. 필수 약관만 동의, 마케팅은 선택.

### 진행 표시
상단에 "회원가입" 타이틀 + 3단계 dot 표시 (1/3, 2/3, 3/3). 스텝 전환 시 우→좌 슬라이드 애니메이션.

### Step 1: 계정 정보

| 필드 | 규칙 |
|---|---|
| 이메일 | 기본 형식 통과 + 이미 가입된 이메일이 아닐 것 |
| 비밀번호 | 8자 이상 + 영문 1자 + 숫자 1자 |

[다음] 버튼 — 두 필드 검증 통과해야 활성화.

### Step 2: 기본 정보

| 필드 | 규칙 |
|---|---|
| 닉네임 | 2~20자 + 비속어 사전 차단 |
| 생년월일 | 연·월·일 드롭다운 3개. 모두 선택 시 유효한 날짜인지 + 만 14세 이상인지 검증 |

[이전] [다음] 버튼.

### Step 3: 약관 동의

| 항목 | 필수 여부 | 비고 |
|---|---|---|
| 이용약관 동의 | **필수** | "(보기)" 링크로 약관 화면 열림 |
| 개인정보처리방침 동의 | **필수** | "(보기)" 링크 |
| 마케팅 이메일 수신 | 선택 | |
| 마케팅 푸시 수신 | 선택 | |

상단에 **전체동의 마스터 체크박스** (3상태: 전부/부분/없음). 탭 시 전부 On/Off.

[이전] [가입 완료] 버튼.

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| Step 1 [다음] | 검증 통과 시 **같은 화면에서 Step 2로 전환** (슬라이드 애니메이션) |
| Step 2 [이전] | 같은 화면에서 Step 1로 전환 |
| Step 2 [다음] | 검증 통과 시 Step 3로 전환 |
| Step 3 [이전] | Step 2로 전환 |
| Step 3 [가입 완료] | 검증 통과 시 아래 "가입 완료 시 동작" 실행 → **온보딩(`/onboarding`)**으로 이동 |
| 이용약관 "(보기)" | **이용약관(`/terms`)**으로 이동 (새 탭 또는 같은 탭 — 정책 확정 필요) |
| 개인정보처리방침 "(보기)" | **개인정보처리방침(`/privacy`)**으로 이동 |
| 전체동의 체크박스 | 같은 화면에서 모든 약관 토글 |
| 하단 "로그인" 링크 | 로그인(`/login`)으로 이동 |

### 가입 완료 시 동작

1. 로그인 처리 + 세션 저장
2. 가입 축하 포인트 +50 적립
3. 입력한 이메일·닉네임을 온보딩 prefill용으로 보관
4. **온보딩(`/onboarding`)으로 이동**

### 검증·정책
- **중복 가입 차단**: 이미 등록된 이메일이면 "이미 가입된 이메일이에요" 에러
- **만 14세 미만**: 생년월일 검증에서 거부 — "만 14세 이상부터 가입 가능해요"
- **비속어 닉네임**: 사전 매칭되면 거부

### Phase 2 보강
- 이메일 소유권 인증 (가입 후 메일 발송 → 링크 클릭으로 활성화)
- 약관 동의 시점 이력 보관
- 실패 누적 잠금

## SCR-AUTH-03 · 온보딩

| 항목 | 값 |
|---|---|
| URL | `/onboarding` |
| 접근 권한 | 로그인 직후 + 온보딩 미완료 사용자 |
| 진입 경로 | 가입 폼 제출 후 · 소셜 약관 모달 완료 후 · SMS 초대 가입 경로 |
| 레이아웃 | 집중 플로우 (상단 진행바 + 중앙 카드) |

### 화면의 목적
가입 직후 **회원 프로필 정보 보완** + **SMS 초대 작품 자동 연결** + 첫 업로드 유도. 가입 경로에서 이미 받은 정보는 prefill로 채워 재입력 부담 최소화.

### 사용자 시나리오
- 방금 가입한 사용자
- 기대: 이미 입력한 정보는 자동 채워져 있고, 빠진 정보만 채우면 됨. 한 번에 끝나는 느낌.

### 진행 표시
상단에 1/3, 2/3, 3/3 진행바. 스텝 전환 애니메이션.

### Step 1: 환영

- 로고 + "Artier에 오신 걸 환영해요" 타이틀 + 짧은 안내
- [시작하기] 버튼 → Step 2로
- [나중에] 버튼 → 완료 플래그만 세팅 + 둘러보기로 이동 (프로필 미완성 상태)

### Step 2: 프로필 설정 (메인 입력 화면)

**프로필 이미지 (선택)**
- 카메라 아이콘 + "사진 등록" 텍스트
- 탭 시 파일 선택 → 미리보기

**경로별 안내 배너 (상단)**

| 경로 | 배너 문구 |
|---|---|
| SMS 초대 | "초대 문자에 기재된 이름·번호와 같게 입력해주세요" |
| 소셜 첫 가입 | "소셜 계정으로 가입했어요. 회원 정보 보완을 위해 실명·전화·이메일을 입력해주세요" |
| 일반 이메일 가입 | 배너 없음 |

**입력 필드**

| 필드 | 필수 여부 | 검증 | Prefill 소스 |
|---|---|---|---|
| 실명 | **모든 경로 필수** | 2자 이상 | SMS 초대일 때 — 초대 시 작가가 입력한 표시명 |
| 닉네임 | 필수 | 2~20자 | 이메일 가입 / 소셜 가입일 때 |
| 전화번호 | **모든 경로 필수** | 숫자 10자 이상 + 중복 가입 차단 | SMS 초대일 때 — 초대받은 번호 |
| 이메일 | **소셜 가입자만 필수** (이메일 가입자는 가입 폼에서 받음) | 기본 형식 + 중복 차단 (단, prefill과 동일하면 통과) | 이메일 가입의 가입 폼 값 / 소셜 provider가 제공한 이메일 |

**필수 표시**: 라벨 옆 빨간 (필수) 텍스트 + 스크린리더용 라벨.

[이전] [다음] 버튼 — 모든 검증 통과 시 Step 3로.

### Step 3: 첫 업로드 유도

- 컨페티(축하) 애니메이션 + "환영합니다, ○○님!" 타이틀
- [작품 올리기] → 업로드 화면
- [둘러보기 시작] → 둘러보기

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| Step 1 [시작하기] | Step 2로 전환 |
| Step 1 [나중에] | 온보딩 완료 플래그만 세팅 → **둘러보기(`/`)로 이동** (프로필 미완성 상태) |
| Step 2 프로필 이미지 탭 | 기기 파일 선택 창 → 같은 화면에서 미리보기 |
| Step 2 [이전] | Step 1로 전환 |
| Step 2 [다음] | 검증 통과 시 +30 AP 적립 → Step 3로 전환 |

| Step 3 [둘러보기 시작] | 완료 동작 실행 → **둘러보기(`/`)**로 이동 |

### 완료 시 동작

1. 입력한 정보로 프로필 저장
2. 이메일·전화를 가입 레지스트리에 등록 (이후 같은 이메일·전화로 재가입 차단)
3. **SMS 초대 매칭 시도**: 전화·실명이 모두 있으면 → 같은 조합의 비회원 초대를 자동 찾아 작품의 비회원 슬롯을 회원 슬롯으로 승격
   - 성공 시 "초대받은 작품 N개가 내 계정과 연결되었어요" 토스트 (다음 화면에서)
   - **전화 일치, 이름 불일치**: 홈 도착 시 본인 확인 모달이 자동 뜸 (08 문서 MOD-02)
4. 온보딩 완료 플래그 세팅
5. 모든 prefill·핸드오프 정보 일괄 정리
6. 위 네비게이션 표에 따라 이동

### 포인트 적립
- Step 2 완료(다음 버튼 통과) 시: +30 AP

### Phase 2 보강
- "나중에" 건너뛴 사용자에게 프로필 미완성 배너 노출
- 프로필 페이지 저장 시 SMS 초대 재매칭 실행 (늦깎이 매칭)
- SMS 초대 토큰 기반 매칭 → 이름 불일치 문제 자체 제거

## SCR-AUTH-04 · 비밀번호 재설정

| 항목 | 값 |
|---|---|
| URL | `/reset-password` |
| 접근 권한 | 비로그인 (로그인 상태여도 접근 가능) |
| 진입 경로 | 로그인 화면 "비밀번호 찾기" · 재설정 메일의 링크 클릭 |

### 화면의 목적
비밀번호 분실 시 재설정 이메일 요청. 별도로 링크 만료·새 비밀번호 설정 화면도 제공.

### 화면 구성 (기본 뷰)

| 영역 | 요소 |
|---|---|
| 타이틀 | "비밀번호 재설정" |
| 설명 | "가입한 이메일을 입력하시면 재설정 링크를 보내드려요" |
| 이메일 입력 | 메일 아이콘 + 형식 검증 |
| [재설정 메일 보내기] | 검증 통과 시 발송 |
| 돌아가기 | "로그인으로 돌아가기" |

### 상태별 전환

| 상태 | 표현 |
|---|---|
| 초기 | 기본 폼 |
| 발송 완료 | "메일을 보냈습니다" 안내 + 로그인 링크 |
| 링크 만료 진입 | "링크가 만료되었어요" + [재발송] |
| 새 비밀번호 설정 진입 | 새 비밀번호 2회 입력 폼 |
| 새 비밀번호 완료 | "변경 완료" 안내 + 로그인 링크 |

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| [재설정 메일 보내기] | 같은 화면에서 "메일을 보냈습니다" 상태로 전환 (URL 변화 없음) |
| "로그인으로 돌아가기" | **로그인(`/login`)**으로 이동 |
| 링크 만료 화면 [재발송] | 메일 발송 상태로 전환 |
| 새 비밀번호 설정 [완료] | "변경 완료" 상태로 전환 |
| 변경 완료 화면 [로그인하러 가기] | 로그인(`/login`)으로 이동 |

### 검증
- 이메일: 기본 형식
- 새 비밀번호: 8자 이상 + 영·숫자 각 1자 + 두 입력 일치

### Phase 2 보강
- 실 이메일 발송
- 토큰 기반 링크 (30분 만료, 1회 사용)
- 재설정 이력 보관

## SCR-AUTH-05 · 점검

| 항목 | 값 |
|---|---|
| URL | `/maintenance` |
| 접근 권한 | 모두 (비로그인 포함) |
| 진입 경로 | 운영팀이 점검 시 모든 트래픽을 이 경로로 전환 |

### 화면의 목적
정기·긴급 점검 동안 사용자에게 안내.

### 화면 구성

| 영역 | 요소 |
|---|---|
| 아이콘 | 공구 아이콘 + 둥근 사각 카드 |
| 타이틀 | "잠시 점검 중이에요" |
| 본문 | 여러 줄 안내 텍스트 |
| ETA | "재개 예정: ○시간 후" 표기 |
| 연락처 | "문의: [고객지원 이메일]" — 메일 링크 |

**고객지원 이메일**은 서비스 전역에서 동일한 값 사용 (문서 상단 공통 정책 참조).

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 이메일 링크 | 기기의 메일 앱 열림 (mailto:) — 화면 이동 없음 |

이 화면엔 앱 내 이동 버튼이 없음 — 점검이 끝난 뒤 사용자가 새로고침하거나 도메인 재진입 시 자동으로 둘러보기 등 정상 라우팅으로 돌아감.

### Phase 2 보강
- 서버·배포 파이프라인과 연동해 점검 모드 자동 토글
- 점검 종료 시각을 시스템에서 주입 → 실시간 카운트다운
- 상태 페이지 링크 추가
