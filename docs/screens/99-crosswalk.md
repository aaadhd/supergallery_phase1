# 레퍼런스 ↔ docs/screens 크로스워크 (Phase 1)

> **이 문서의 용도**: 레퍼런스 원본(화면 모음)의 화면 이름을 `docs/screens/`의 SCR-ID로 빠르게 찾기 위한 대조표.
>
> **읽는 법**: 레퍼런스 화면명을 찾고 → `Docs SCR-ID`와 `문서 위치` 열로 이동 → 해당 문서의 앵커 섹션 확인.
>
> **기준일**: 2026-04-17

---

## 범례

| 상태 | 의미 |
|---|---|
| ✅ **수정됨** | 레퍼런스 기반 스펙이 `docs/screens/`에서 수정·재정의됨 — 이 docs를 우선 적용 |
| 🔁 **통합됨** | 레퍼런스의 독립 화면이 다른 화면에 흡수됨 — `99-removed-from-reference.md` §1 참조 |
| ❌ **제거됨** | Phase 1에서 의도적으로 제거·보류 — `99-removed-from-reference.md` §2 참조 |
| 📎 **레퍼런스 유지** | 구현은 있으나 docs에서 재정의하지 않음, 레퍼런스 그대로 따름 — `99-removed-from-reference.md` §3 참조 |

## 사용 팁

- 레퍼런스 스펙에서 출발해 개발할 때 → 이 표로 Docs SCR-ID를 찾은 후 해당 문서의 스펙을 먼저 적용.
- 기존 구현 코드를 docs에 매칭할 때 → 이 표와 `99-removed-from-reference.md`를 함께 보면 "이게 원래 어떤 레퍼런스 화면에 해당하는지"가 명확해짐.
- 신규 화면 요청이 들어오면 → 먼저 "docs 신규" 섹션과 레퍼런스 양쪽에 이미 있는지 확인.
