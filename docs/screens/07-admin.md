## 사전 지식

### 어드민 콘솔이 존재하는 이유

운영팀이 일상적으로 처리하는 일:
1. **콘텐츠 검수** — 업로드된 작품 승인/반려 (Phase 1 기본은 모든 작품이 발행 후 검수 대기)
2. **신고 처리** — 사용자 신고를 4가지 액션으로 처리 + 허위 신고 추적
3. **회원 관리** — 정지/해제 (4단계: 주의·7일·30일·영구)
4. **큐레이션** — 배너·Pick·테마전·추천 작가
5. **이벤트** — 등록·기간 관리
6. **운영 진행 추적** — 미해결 이슈·출시 체크리스트·파트너 작가

### 공통 정책

- **터치 영역**: 44×44 픽셀 (시니어 작가 외 운영팀이 사용하지만 앱 일관성 유지)
- **다국어**: 어드민은 **한국어 전용** (Phase 1)
- **확인 다이얼로그**: 앱 자체 디자인 (브라우저 네이티브 금지)
- **파괴적 작업**: 빨간 확인 버튼 (삭제·강제 정지 등)

---

## 공통 접근 제어

`/admin` 진입 시 `canAccessAdminRoutes()` 함수가 다음 순서로 검사:

1. **로그인 여부**: 미로그인 시 즉시 `/`로 리다이렉트
2. **환경 변수 우회**: `VITE_ADMIN_OPEN=true`이면 role 체크 없이 통과 (프리뷰·CI용)
3. **운영팀 역할**: `hasOperatorRole()` — `artier_admin_session_v1` 키가 `'1'`인지 확인

비밀번호 게이트는 없음. Phase 2에서 서버 사이드 권한 검사로 대체 예정.

### 공통 레이아웃 (AdminLayout)

- **좌측 사이드바**: 13개 메뉴 항목 + 현재 활성 표시
- **사이드바 헤더**: "사용자 앱으로" 링크 + "운영 콘솔" 라벨 + 운영팀 역할 배지
- **본문**: 우측 메인 영역

### 공통 네비게이션

| 탭 대상 | 결과 |
|---|---|
| 좌측 메뉴 항목 | 해당 어드민 화면으로 이동 (같은 레이아웃, 본문만 교체) |
| 사이드바 헤더 "사용자 앱으로" | 일반 사용자 앱의 둘러보기(`/`)로 이동 |

## SCR-ADM-01 · 어드민 대시보드

| URL | `/admin` |
|---|---|

### 화면의 목적
운영팀 홈. 미해결 이슈 / 체크리스트 진행률 / 파트너 작가 현황을 한눈에.

### 화면 구성

| 영역 | 요소 |
|---|---|
| 상단 KPI 카드 4개 | 미결 이슈 건수 · 체크리스트 진행률 % · 파트너 작가 수 · 파트너 작품 수 |
| 최근 이슈 | 상위 5건 — 우선순위 배지 + 제목 |
| 체크리스트 요약 | 카테고리별 진행률 바 |
| 파트너 단계 요약 | 스테이지별 분포 |

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| KPI 카드 (미결 이슈·체크리스트·파트너 작가) | 각각 해당 화면으로 이동. 파트너 작품 카드는 링크 없이 정보만 표시 |
| 최근 이슈 행 | 미해결 이슈 화면(`/admin/issues`)으로 이동 (해당 이슈 선택된 상태 권장) |
| 체크리스트 요약 바 | 출시 체크리스트 화면(`/admin/checklist`)으로 이동 |
| 파트너 단계 요약 | 파트너 작가 화면(`/admin/partners`)으로 이동 |

### 수용기준 (Acceptance Criteria)

- AC-01: Given 어드민 대시보드 진입 / When 페이지 로드 완료 / Then KPI 카드 4개(미결 이슈 건수·체크리스트 진행률·파트너 작가 수·파트너 작품 수)가 실시간 데이터로 표시된다
- AC-02: Given KPI 카드 중 "미결 이슈" 탭 / When 카드 클릭 / Then `/admin/issues` 화면으로 이동한다
- AC-03: Given KPI 카드 중 "체크리스트 진행률" 탭 / When 카드 클릭 / Then `/admin/checklist` 화면으로 이동한다
- AC-04: Given KPI 카드 중 "파트너 작가 수" 탭 / When 카드 클릭 / Then `/admin/partners` 화면으로 이동한다
- AC-05: Given 대시보드 최근 이슈 영역 / When 페이지 로드 / Then 우선순위 배지 + 제목이 포함된 최근 이슈 상위 5건이 표시된다
- AC-06: Given 최근 이슈 행 탭 / When 행 클릭 / Then 미해결 이슈 화면으로 이동한다

### 엣지케이스

- EC-01: 미해결 이슈가 0건일 때 → KPI 카드에 "0"으로 표시하고 최근 이슈 영역에 빈 상태 안내
- EC-02: 체크리스트 항목이 모두 완료(100%)일 때 → 진행률 바에 100% 표시

## SCR-ADM-02 · 미해결 이슈

| URL | `/admin/issues` |
|---|---|

### 화면의 목적
개발·운영 중 발견된 이슈 추적. 상태·우선순위·카테고리 필터.

### 상수 정의

| 상수 | 값 |
|---|---|
| 상태 | 미결 / 진행 중 / 해결됨 / 보류 |
| 우선순위 | 긴급 / 높음 / 보통 / 낮음 |
| 카테고리 | 디자인 / 개발 / 콘텐츠 / 법무 / 마케팅 / 운영 |

### 화면 구성

| 영역 | 요소 |
|---|---|
| 필터 | 드롭다운 3개 (상태 / 우선순위 / 카테고리) |
| 테이블 | ID · 제목 · 카테고리 · 우선순위(배지) · 상태(배지) · 담당자 · 마감일 |

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 필터 드롭다운 변경 | 같은 화면에서 테이블만 필터링 |
| 테이블 행 탭 | 같은 화면 위에 **편집 다이얼로그** 열림 (상태·담당자·메모 수정) |
| 다이얼로그 [저장] | 다이얼로그 닫힘 + 테이블 갱신 |

### 수용기준 (Acceptance Criteria)

- AC-01: Given 미해결 이슈 화면 진입 / When 상태 드롭다운을 "진행 중"으로 변경 / Then 테이블이 "진행 중" 상태 이슈만 필터링되어 표시된다
- AC-02: Given 미해결 이슈 화면 / When 우선순위 드롭다운을 "긴급"으로 변경 / Then 긴급 이슈만 표시된다
- AC-03: Given 미해결 이슈 화면 / When 카테고리 드롭다운을 "법무"로 변경 / Then 법무 카테고리 이슈만 표시된다
- AC-04: Given 3개 필터를 동시에 설정 / When 필터 조합 적용 / Then 모든 조건을 만족하는 이슈만 표시된다
- AC-05: Given 테이블 행 탭 / When 특정 이슈 행 클릭 / Then 편집 다이얼로그가 열려 상태·담당자·메모를 수정할 수 있다
- AC-06: Given 편집 다이얼로그에서 상태 변경 후 / When [저장] 클릭 / Then 다이얼로그 닫히고 테이블이 갱신된다

### 엣지케이스

- EC-01: 필터 조합 결과가 0건일 때 → 빈 상태 안내 표시 ("조건에 맞는 이슈가 없어요")
- EC-02: 필터 3개를 모두 "전체"로 돌렸을 때 → 전체 이슈 다시 표시

### Phase 2 보강
- 외부 트래커 연동 (Linear·Jira)
- 이슈 댓글·변경 이력

## SCR-ADM-03 · 출시 체크리스트

| URL | `/admin/checklist` |
|---|---|

### 화면의 목적
런칭 전 필수 항목 체크. 진행률 표시 + 카테고리 필터.

### 카테고리
기술 · 디자인 · QA · 법무 · 마케팅 · 운영

### 화면 구성

| 영역 | 요소 |
|---|---|
| 상단 진행 바 | 완료 / 전체 비율 + % |
| 카테고리 필터 | 드롭다운 |
| 체크리스트 행 | 체크박스 + 항목 제목 + 담당자 + 메모 |
| [리셋] | 초기 상태로 복원 (데모용) |

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 카테고리 필터 | 같은 화면에서 리스트 필터링 |
| 체크박스 | 같은 화면에서 상태 토글 — 진행 바도 즉시 갱신 |
| [리셋] | 파괴적 확인 다이얼로그 → 확인 시 시드 상태 복원 |

### 수용기준 (Acceptance Criteria)

- AC-01: Given 출시 체크리스트 화면 진입 / When 페이지 로드 / Then 상단 진행 바에 완료/전체 비율과 퍼센트가 표시된다
- AC-02: Given 체크리스트 항목 / When 체크박스 토글 / Then 진행 바가 즉시 갱신된다
- AC-03: Given 카테고리 필터 / When "QA"로 변경 / Then QA 카테고리 항목만 표시된다
- AC-04: Given [리셋] 버튼 탭 / When 파괴적 확인 다이얼로그에서 확인 / Then 모든 체크박스가 시드 초기 상태로 복원되고 진행 바도 갱신된다
- AC-05: Given [리셋] 버튼 탭 / When 파괴적 확인 다이얼로그에서 취소 / Then 아무 변화 없이 현재 상태 유지

### 엣지케이스

- EC-01: 모든 항목이 이미 완료 상태에서 리셋 → 진행 바가 초기 비율로 돌아감
- EC-02: 필터 적용 중 체크박스 토글 → 진행 바는 전체 항목 기준으로 갱신 (필터 범위가 아닌 전체)

## SCR-ADM-04 · 파트너 작가

| URL | `/admin/partners` |
|---|---|

### 화면의 목적
서비스 런칭 초기 파트너십 파이프라인 관리.

### 상수
- **단계**: 후보 / 연락 완료 / 온보딩 중 / 활성 / 비활성
- **제출 상태**: 미제출 / 제출 완료 / 검토 중 / 승인 / 반려

### 화면 구성

| 영역 | 요소 |
|---|---|
| 필터 | 단계 드롭다운 + 제출 상태 드롭다운 |
| 테이블 | 작가명 · 단계 · 제출 상태 · 진행률 · 최근 컨택일 · 비고 |

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 필터 변경 | 같은 화면에서 테이블 필터링 |
| 테이블 행 | 같은 화면 위에 편집 다이얼로그 열림 |
| 다이얼로그 [저장] | 다이얼로그 닫힘 + 테이블 갱신 |

### 수용기준 (Acceptance Criteria)

- AC-01: Given 파트너 작가 화면 / When 단계 드롭다운을 "온보딩 중"으로 변경 / Then 온보딩 중 단계 파트너만 테이블에 표시된다
- AC-02: Given 파트너 작가 화면 / When 제출 상태 드롭다운을 "제출 완료"로 변경 / Then 제출 완료 파트너만 표시된다
- AC-03: Given 단계 + 제출 상태 필터 동시 적용 / When 두 필터 조합 / Then 양쪽 조건을 모두 만족하는 행만 표시된다
- AC-04: Given 테이블 행 탭 / When 파트너 행 클릭 / Then 편집 다이얼로그가 열린다
- AC-05: Given 편집 다이얼로그에서 단계를 "활성"으로 변경 후 / When [저장] / Then 다이얼로그 닫히고 테이블에 변경된 단계 반영

### 엣지케이스

- EC-01: 필터 결과가 0건일 때 → 빈 상태 안내 표시
- EC-02: "비활성" 단계 파트너 편집 시 → 편집 가능하나 특별 제한 없음

## SCR-ADM-05 · 이벤트 참여자

| URL | `/admin/events` |
|---|---|

### 화면의 목적
각 이벤트의 **참여자 목록 + 응모 작품** 조회. (이벤트 자체 등록·수정은 SCR-ADM-11)

### 화면 구성

| 영역 | 요소 |
|---|---|
| 이벤트 요약 카드 그리드 | 이벤트별 카드: 제목 + 기간 + "전체 N명" + "완료 N명" 배지 |
| 필터 | 이벤트 드롭다운 · 상태 드롭다운 (참여 완료 / 대기 중 / 취소) |
| 참여자 테이블 | 이름 · 이메일 · 이벤트 · 상태(배지) · 참여일 |

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 이벤트 요약 카드 | 같은 화면에서 필터가 해당 이벤트로 설정됨 (테이블 필터링) |
| 이벤트·상태 필터 | 같은 화면에서 테이블 필터링 |
| 참여자 행 (Phase 2) | 현재 Phase 1은 상세 모달 없음 — Phase 2에서 참여 작품 미리보기 모달 예정 |

### 데이터 소스

참여자 목록은 **실 업로드 작품 + 데모 시드 병합**:
- 이벤트 상세에서 "참여" 버튼으로 실제 업로드된 작품이 자동으로 참여자로 잡힘 (작품의 연결 이벤트 필드 기반)
- 상태 매핑: 승인된 작품 → 참여 완료, 반려/삭제된 작품 → 취소, 그 외 → 대기 중
- 데모 시드도 유지되어 한 표에서 같이 노출 (ID 충돌 방지)

**중요**: 레퍼런스에는 참여자 목록이 별도 데이터원으로 되어 있지만, Phase 1에선 **작품의 연결 이벤트 필드로 일원화**. 작품이 곧 참여 기록.

### 수용기준 (Acceptance Criteria)

- AC-01: Given 이벤트 참여자 화면 진입 / When 페이지 로드 / Then 이벤트별 요약 카드(제목 + 기간 + 전체 N명 + 완료 N명)가 그리드로 표시된다
- AC-02: Given 이벤트 요약 카드 탭 / When 특정 이벤트 카드 클릭 / Then 필터가 해당 이벤트로 설정되고 참여자 테이블이 필터링된다
- AC-03: Given 참여자 테이블 / When 상태 드롭다운을 "참여 완료"로 변경 / Then 참여 완료 상태 참여자만 표시된다
- AC-04: Given 작품이 승인 상태 / When 참여자 테이블 조회 / Then 해당 참여자의 상태가 "참여 완료"로 매핑되어 표시된다
- AC-05: Given 작품이 반려·삭제 상태 / When 참여자 테이블 조회 / Then 해당 참여자의 상태가 "취소"로 매핑되어 표시된다
- AC-06: Given 작품이 대기 중 상태 / When 참여자 테이블 조회 / Then 해당 참여자의 상태가 "대기 중"으로 매핑되어 표시된다

### 엣지케이스

- EC-01: 참여자가 0명인 이벤트 → 요약 카드에 "전체 0명" 표시, 테이블 필터 시 빈 상태 안내

## SCR-ADM-06 · 콘텐츠 검수

| URL | `/admin/content-review` |
|---|---|

### 화면의 목적
업로드된 작품의 승인/반려. Phase 1 핵심 운영 화면.

### 화면 구성

| 영역 | 요소 |
|---|---|
| 필터 | 상태 드롭다운 (전체 / 대기중 / 승인 / 반려) + 기간 필터 (시작일·종료일) |
| 작품 테이블 | 썸네일 · 작품명 · 작가 · 업로드일 · 상태 배지 · 작업(승인/반려 버튼) |
| **[승인]** | 승인 처리 + 작가에게 알림 발송 |
| **[반려]** | 사유 선택 모달 → 반려 처리 + 알림 |

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 상태 필터 | 같은 화면에서 작품 카드 그리드 필터링 |
| 작품 카드 | 같은 화면 우측에 선택된 작품 상세가 큰 미리보기로 표시 |
| [승인] | 화면 이동 없음 — 처리 후 현재 작품을 리스트에서 제거 (다음 작품 자동 선택) |
| [반려] | 같은 화면 위에 **반려 사유 선택 모달** 열림 → 사유·메시지 입력 후 [반려 처리] → 모달 닫힘 + 리스트에서 제거 |

### 반려 사유 (4분류)

| 키 | 라벨 |
|---|---|
| copyright | 저작권 의심 |
| inappropriate | 부적절한 콘텐츠 |
| lowQuality | 품질 미달 |
| other | 기타 (직접 입력) |

### 알림 → 작가 후속 흐름
승인·반려 후 시스템 알림 발송 → 작가 프로필 작품 탭에 **반려 배지** 표시 → 작가가 배지 탭 시 사유·메시지 모달 + "수정해서 다시 올리기" 버튼 → 업로드 편집 모드로 이동

### 수용기준 (Acceptance Criteria)

- AC-01: Given 콘텐츠 검수 화면 / When 상태 필터를 "대기중"으로 설정 / Then 검수 대기 작품만 카드 그리드에 표시된다
- AC-02: Given 작품 카드 탭 / When 특정 작품 카드 클릭 / Then 우측에 큰 미리보기 + 메타 정보가 표시된다
- AC-03: Given 선택된 작품에서 / When [승인] 버튼 클릭 / Then 작품 상태가 approved로 변경되고 작가에게 알림이 발송되며 리스트에서 제거 후 다음 작품이 자동 선택된다
- AC-04: Given 선택된 작품에서 / When [반려] 버튼 클릭 / Then 반려 사유 선택 모달이 열린다
- AC-05: Given 반려 사유 모달에서 사유(4분류 중 1개) + 메시지 입력 후 / When [반려 처리] 클릭 / Then 작품이 반려 처리되고 작가에게 사유·메시지 포함 알림 발송, 모달 닫힘 + 리스트에서 제거
- AC-06: Given 상태 필터를 "승인"으로 변경 / When 필터 적용 / Then 이미 승인된 작품 목록이 표시된다

### 엣지케이스

- EC-01: 검수 대기 작품이 0건일 때 → 빈 상태 안내 ("검수할 작품이 없어요")
- EC-02: 마지막 대기 작품을 승인/반려했을 때 → 리스트 비움 + 빈 상태 안내 표시
- EC-03: 반려 사유 "기타" 선택 시 → 직접 입력 필드가 표시되어 구체적 사유를 작성해야 함

## SCR-ADM-08 · Pick 관리

| URL | `/admin/picks` |
|---|---|

### 화면의 목적
Pick 큐레이션 관리 화면. 현재 Pick(주간 운영, 최대 10개)은 둘러보기 피드 1번째 버킷에 노출.

### 정책
- **최대 10건 고정**
- 현재 Pick은 주간 단위로 운영하며 최대 10개를 유지한다.
- 한 번 Pick 선정된 작품은 배지 이력을 유지하며, 현재 Pick에서 제거되어도 배지는 남는다.

### 화면 구성

| 영역 | 요소 |
|---|---|
| 탭 | 현재 Pick / Pick 이력 |
| 현재 Pick 목록 | 카드 리스트 + 드래그 핸들로 정렬 변경 |
| Pick 이력 목록 | 과거 포함 선정 작품 리스트 + 현재 Pick 재편입 액션 |
| 검색 입력 | 작품명·작가 검색 → 결과 드롭다운 → 추가 |
| [추가] | 10건 도달 시 비활성 |
| 운영 정책 안내 | "현재 Pick은 주간 운영, 선정 이력 배지는 유지" 작은 회색 텍스트 |

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 탭 전환 (현재 Pick / Pick 이력) | 같은 화면에서 목록만 전환 (URL 변화 없음) |
| 검색 입력 | 같은 화면에서 결과 드롭다운 노출 |
| 검색 결과 행 → [추가] | 같은 화면에서 Pick 목록에 추가 |
| Pick 카드 드래그 핸들 | 같은 화면에서 순서 재정렬 |
| Pick 카드 삭제 | 같은 화면에서 제거 (확인 다이얼로그 생략 가능 — 10건 제한이라 복구 쉬움) |
| Pick 이력 카드 → 현재 Pick 재편입 | 현재 Pick 최대 10건 조건에서 즉시 재편입 |

### 수용기준 (Acceptance Criteria)

- AC-01: Given Pick 관리 화면 / When "현재 Pick" 탭 선택 / Then 현재 Pick 목록(최대 10건)이 드래그 가능한 카드 리스트로 표시된다
- AC-02: Given 현재 Pick이 10건 미만 / When 검색으로 작품 찾아 [추가] 클릭 / Then 현재 Pick 목록에 추가된다
- AC-03: Given 현재 Pick이 이미 10건 / When [추가] 버튼 확인 / Then 버튼이 비활성 상태이다
- AC-04: Given 현재 Pick 카드 / When 드래그 핸들로 순서 변경 / Then 변경된 순서가 저장되고 둘러보기 피드 노출 순서에 반영된다
- AC-05: Given "Pick 이력" 탭 선택 / When 과거 Pick 작품 목록 조회 / Then 이력 목록이 표시되고 현재 Pick 재편입 액션이 있다
- AC-06: Given Pick 이력의 작품에서 재편입 액션 / When 현재 Pick이 10건 미만일 때 클릭 / Then 해당 작품이 현재 Pick에 즉시 추가된다
- AC-07: Given Pick에서 제거된 작품 / When 둘러보기에서 확인 / Then Pick 배지 이력은 유지된다

### 엣지케이스

- EC-01: 현재 Pick이 10건인 상태에서 이력 재편입 시도 → 추가 불가 안내
- EC-02: 이미 현재 Pick에 있는 작품을 검색·추가 시도 → 중복 추가 방지
- EC-03: 현재 Pick에서 모든 작품 제거 → 빈 목록 표시, 검색으로 새로 추가 가능

| 버전 | 일자 | 업데이트 내용 |
|---|---|---|
| v1.1 | 2026-04-16 | Pick 관리 화면을 현재 Pick/이력 Pick 분리 탭 구조와 재편입 동작 기준으로 동기화. |

- AC-06: Given 배너 행 탭 / When 편집 다이얼로그에서 배지 타입을 "HOT"으로 변경 후 [저장] / Then 다이얼로그 닫히고 배지 표시가 갱신된다

### 엣지케이스

- EC-01: 모든 배너를 비활성으로 전환 → 사용자 화면에 배너 영역 자체가 노출되지 않음
- EC-02: 배너 삭제 확인 다이얼로그에서 취소 → 배너 유지
- EC-03: 배너 1개만 남은 상태에서 삭제 → 빈 목록 표시

## SCR-ADM-11 · 이벤트 관리

| URL | `/admin/managed-events` |
|---|---|

### 화면의 목적
이벤트 CRUD. 일정·배너·설명 관리.

### 화면 구성

| 영역 | 요소 |
|---|---|
| 이벤트 리스트 | 상태별 그룹 (진행 중 / 예정 / 종료) |
| [+ 새 이벤트] | 생성 다이얼로그 |
| 생성/편집 다이얼로그 | 제목 · 부제 · 기간(시작일/종료일) · 배너 이미지 URL · 설명 · 대상 라벨 |
| 행 액션 | 편집 · 삭제(파괴적 확인) |

### 상태 자동 판정
상태는 저장 필드가 아니라 **현재 시각 기준 자동 판정**. 별도로 상태 드롭다운 입력 받지 않음.

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| [+ 새 이벤트] | 같은 화면 위에 생성 다이얼로그 열림 |
| 이벤트 행 편집 아이콘 | 편집 다이얼로그 열림 |
| 이벤트 행 삭제 아이콘 | 파괴적 확인 다이얼로그 → 제거 (사용자 이벤트 목록에서도 즉시 사라짐) |
| 다이얼로그 [저장] | 다이얼로그 닫힘 + 리스트 갱신 + 사용자 앱에도 즉시 반영 |

### 수용기준 (Acceptance Criteria)

- AC-01: Given 이벤트 관리 화면 / When 페이지 로드 / Then 이벤트가 상태별(진행 중 / 예정 / 종료) 그룹으로 분류되어 표시된다
- AC-02: Given 이벤트 상태 / When 현재 시각이 시작일~종료일 사이 / Then 해당 이벤트가 "진행 중"으로 자동 분류된다
- AC-03: Given 이벤트 상태 / When 현재 시각이 시작일 이전 / Then "예정"으로 자동 분류된다
- AC-04: Given 이벤트 상태 / When 현재 시각이 종료일 이후 / Then "종료"로 자동 분류된다
- AC-05: Given [+ 새 이벤트] 클릭 / When 생성 다이얼로그에서 제목·기간 등 입력 후 [저장] / Then 이벤트가 생성되고 사용자 앱에도 즉시 반영된다
- AC-06: Given 이벤트 삭제 아이콘 탭 / When 파괴적 확인 다이얼로그에서 확인 / Then 이벤트 삭제되고 사용자 앱 이벤트 목록에서도 즉시 제거된다

### 엣지케이스

- EC-01: 시작일과 종료일을 같은 날로 설정 → 당일만 "진행 중", 다음 날부터 "종료"
- EC-02: 종료일을 시작일보다 앞서 입력 → 입력 검증 에러 표시
- EC-03: 이벤트 0건일 때 → 각 상태 그룹에 빈 상태 안내

## SCR-ADM-12 · 신고 관리

| URL | `/admin/reports` |
|---|---|

### 화면의 목적
사용자 신고 처리. **4가지 액션**. 허위 신고는 신고자에게 카운트 누적.

### 4가지 액션

| 액션 | 효과 | 시각 |
|---|---|---|
| **삭제** | 작품 신고 전용. 작품 영구 삭제 + 처리 상태 기록 | 빨강 |
| **경고** | 신고 대상 작가에게 경고 카운트 +1 누적. **3회 누적 시 자동 7일 정지로 승격** | 주황 |
| **기각** | 신고가 부당으로 판정. 신고자에게 허위 신고 카운트 +1. **3회 누적 시 신고자 7일 차단** | 중립 |
| **비공개** | 작품 피드·검색 제외 (작가 본인 프로필엔 보임) | 회색 |
| (목록에서 제거) | 액션 없이 큐에서만 빼기 (레거시 호환) | — |

### 화면 구성

| 영역 | 요소 |
|---|---|
| 필터 | 상태 (대기·처리됨·숨김·삭제됨·경고됨·기각됨) · 타입 (작품 / 작가) |
| 신고 카드 | 신고 대상(썸네일+제목) + 신고자 + 사유 + 상세(있으면) + 시각 |
| 액션 버튼 4개 + 제거 | 위 |

### 자동 정지 승격 로직

- 경고 누적 3회 → 자동 7일 정지로 전환
- 허위 신고 누적 3회 → 신고자 7일 차단
- 카운터는 서버 없이 현재는 로컬 누적

### 신고자 피드백 정책 (Phase 1 결정)

**신고 처리 결과를 신고자에게 알리지 않음** — 운영팀 부담·역풍 방지. Phase 2에서 재검토.

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 상태·타입 필터 | 같은 화면에서 카드 리스트 필터링 |
| 신고 대상 썸네일·제목 | 해당 **작품 상세 모달** 또는 **작가 프로필(`/profile/:id`)** 로 이동 (대상 타입에 따라) |
| [삭제] | 파괴적 확인 다이얼로그 → 작품 영구 삭제 → 카드에서 제거 |
| [경고] | 즉시 처리 (확인 다이얼로그 없음) → 신고 대상에게 경고 +1 → 카드 상태 "경고됨"으로 |
| [기각] | 즉시 처리 → 신고자에게 허위 +1 → 카드 상태 "기각됨"으로 |
| [비공개] | 즉시 처리 → 작품 숨김 → 카드 상태 "숨김"으로 |

### 수용기준 (Acceptance Criteria)

- AC-01: Given 신고 관리 화면 / When 상태 필터를 "대기"로 설정 / Then 미처리 신고만 카드 리스트에 표시된다
- AC-02: Given 작품 신고 카드에서 / When [삭제] 클릭 후 파괴적 확인 다이얼로그에서 확인 / Then 작품이 영구 삭제되고 카드 상태가 "삭제됨"으로 변경된다
- AC-03: Given 신고 카드에서 / When [경고] 클릭 / Then 신고 대상 작가에게 경고 카운트 +1이 누적되고 카드 상태가 "경고됨"으로 변경된다
- AC-04: Given 경고가 3회 누적된 작가 / When 3번째 경고 처리 / Then 해당 작가에게 자동 7일 정지가 적용된다
- AC-05: Given 신고 카드에서 / When [기각] 클릭 / Then 신고자에게 허위 신고 카운트 +1이 누적되고 카드 상태가 "기각됨"으로 변경된다
- AC-06: Given 허위 신고가 3회 누적된 신고자 / When 3번째 기각 처리 / Then 해당 신고자에게 자동 7일 차단이 적용된다
- AC-07: Given 신고 카드에서 / When [비공개] 클릭 / Then 작품이 피드·검색에서 제외되고(작가 본인 프로필엔 보임) 카드 상태가 "숨김"으로 변경된다

### 엣지케이스

- EC-01: 작가 신고에 대해 [삭제] 시도 → 삭제는 작품 신고 전용이므로 작가 신고에는 [삭제] 버튼 미노출
- EC-02: 이미 처리된 신고 카드에서 액션 재시도 → 액션 버튼 비활성 또는 미노출
- EC-03: 신고 대상 작품이 이미 다른 경로로 삭제된 경우 → 해당 신고 카드에 "원본 삭제됨" 안내

## SCR-ADM-13 · 회원 관리

| URL | `/admin/members` |
|---|---|

### 화면의 목적
회원 상태 조회 + **4단계 정지/해제**.

### 정지 4단계

| 레벨 | 기간 | 표시 |
|---|---|---|
| **주의** | — | 정지 없이 경고만. 누적 시 정지로 승격 |
| **7일** | 7일 | 시한부 |
| **30일** | 30일 | 시한부 |
| **영구** | ∞ | 무기한 |

### 화면 구성

| 영역 | 요소 |
|---|---|
| 회원 테이블 | ID · 이름 · 이메일 · 가입일 · 상태 배지 · 작품 수 |
| 행 액션 | **정지** (4단계 라디오 모달) · **해제** |
| 정지 상세 표시 | 사유 · 기한 |

### 데모 사용자 글로벌 적용

Phase 1은 서버가 없어 다른 회원들에게 정지를 실제로 적용할 수 없음. 대신 **데모용 사용자(시드의 대표 작가)**에 대해서만 **글로벌 정지 적용**:
- 정지 즉시 해당 사용자 로그아웃
- 다음 로그인 시 로그인 화면에서 차단 + 사유·해제일 안내

기타 목업 회원은 UI 상태 배지만 변경 (실 로그인 차단 안 됨 — Phase 1 한계).

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 회원 행 탭 | 상세 정보 다이얼로그 (옵션) — Phase 2 |
| 정지 아이콘 | 같은 화면 위에 **4단계 라디오 모달** 열림 (주의/7일/30일/영구 + 사유 입력) |
| 4단계 모달 [적용] | 모달 닫힘 + 회원 상태 배지 갱신. 데모 사용자면 즉시 로그아웃 처리 |
| 해제 아이콘 | 파괴적 확인 다이얼로그 → 정지 해제 |
| 회원 이름 탭 | 해당 회원 프로필(`/profile/:id`)로 이동 (새 탭) — 참고용 |

### 수용기준 (Acceptance Criteria)

- AC-01: Given 회원 관리 화면 / When 회원 행의 정지 아이콘 클릭 / Then 4단계 라디오 모달(주의/7일/30일/영구 + 사유 입력)이 열린다
- AC-02: Given 4단계 모달에서 "7일" 선택 + 사유 입력 후 / When [적용] 클릭 / Then 모달 닫히고 회원 상태 배지가 "7일 정지"로 갱신된다
- AC-03: Given 데모 사용자(시드 대표 작가)에게 정지 적용 / When [적용] 클릭 / Then 해당 사용자가 즉시 로그아웃되고 다음 로그인 시 차단 + 사유·해제일 안내가 표시된다
- AC-04: Given 데모 사용자가 아닌 목업 회원에게 정지 적용 / When [적용] 클릭 / Then UI 상태 배지만 변경되고 실 로그인 차단은 적용되지 않는다 (Phase 1 한계)
- AC-05: Given 정지된 회원 행의 해제 아이콘 / When 파괴적 확인 다이얼로그에서 확인 / Then 정지가 해제되고 상태 배지가 정상으로 복귀한다
- AC-06: Given 회원 이름 탭 / When 클릭 / Then 해당 회원 프로필(`/profile/:id`)이 새 탭으로 열린다

### 엣지케이스

- EC-01: 이미 정지된 회원에게 다시 정지 적용 → 기존 정지를 새 단계로 덮어씌움
- EC-02: "영구" 정지 후 해제 → 정상 상태로 복귀 가능
- EC-03: 정지 모달에서 사유 미입력 시 → [적용] 버튼 비활성 또는 기본 사유 적용

### Phase 2 보강
- 실 회원 DB 연동 (모든 회원에게 글로벌 적용)
- 정지 이력 로그 (감사 추적)
- 운영팀 다중 계정 + 권한 분리 (슈퍼어드민 / 검수자 / 마케팅)
- 정지 알림 자동 발송 (이메일 + SMS)
