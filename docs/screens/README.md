### 이 문서의 위상
이 폴더의 MD 파일들은 **레퍼런스 명세의 수정본**입니다. 원래 reference 폴더(217개 MD)의 명세대로 만들어진 구현이 있고, PM이 바이브코딩 과정에서 여러 판단으로 명세를 조정했습니다. 이 조정 결과가 최종 기준이므로 다음 원칙으로 적용해주세요:

| 상황 | 적용 |
|---|---|
| 레퍼런스와 **다른** 부분 | 이 문서의 결정이 우선 |
| 레퍼런스에 **없는** 부분 | 이 문서가 새로 추가된 요구사항 |
| 레퍼런스에는 있지만 이 문서에 **없는** 부분 | `99-removed-from-reference.md`를 먼저 확인 → 제거·통합이 아니면 레퍼런스 그대로 따르기 |

> **빠른 매칭을 위해**: 레퍼런스 화면명으로 시작해 이 docs의 스펙을 찾을 때는 **[99-crosswalk.md](99-crosswalk.md)**를 가장 먼저 열면 됩니다. 화면명 → SCR-ID → 문서 위치가 한 표에 정리되어 있습니다.

## 화면 스펙 공통 포맷

각 화면은 아래 섹션으로 구성:

1. **헤더** — URL · 접근 권한 · 진입 경로 · 레이아웃 · 다국어
2. **화면의 목적** — 이 화면이 왜 존재하나
3. **사용자 시나리오** — 주 사용자와 기대 동선
4. **화면 구성** — 요소별 표 (영역 / 요소 / 비고)
5. **상태·분기** — 로딩/빈 상태/에러/권한별 뷰
6. **액션·이벤트** — 버튼·상호작용별 결과
7. **검증·정책** — 입력 제한, 비즈니스 룰
8. **Phase 2 보강** — 이번 단계에 안 들어간 향후 과제

각 문서 상단에 "사전 지식" 블록이 있어 해당 카테고리의 데이터/정책/용어를 요약해둠. 이 README를 건너뛰어도 각 문서만으로 이해할 수 있도록 설계.

## 전체 공통 정책 (모든 화면 적용)

### 타겟 사용자와 디자인 원칙
- **주 타겟**: 50대 액티브 시니어 순수미술 작가 (그림 그리는 분들)
- **시니어 친화**: 모든 탭·버튼 등 누르는 영역은 최소 44×44 픽셀 크기 확보
- **시각 일관성**: 앱 내에서 브라우저 네이티브 confirm/alert 대화상자를 쓰지 않음. 앱 자체 디자인의 확인 다이얼로그만 사용
- **작품 이미지 표시**: 작품 비율 유지 (잘라내기 금지) — 원본이 세로형이든 가로형이든 그대로

### 다국어
- **한국어·영어 지원**
- 모든 텍스트는 다국어 리소스에서 가져옴 (하드코딩 금지)
- 사용자가 언어를 바꾸면 즉시 전체 화면에 반영

### 연령·비속어
- **만 14세 미만 가입 불가** — 생년월일 검증으로 차단
- **비속어 검사 위치**: 전시명·그룹명·닉네임·프로필 편집 (사전 기반 차단)

### 비밀번호 정책
- 8자 이상 + 영문 1자 + 숫자 1자 이상

### 콘텐츠 검수
- **모든 작품은 발행 후 운영팀 검수 대기 → 승인 → 피드 노출**이 원칙
- 데모·PM 시연 편의를 위해 검수 우회 모드가 있지만, 실서비스에서는 검수 필수

### 확인 다이얼로그 규칙
- 파괴적 작업(삭제 등): 빨간 확인 버튼
- 일반 작업: 기본 버튼

### 중복 방지
- 같은 이메일·전화번호로 중복 가입 불가
- 같은 대상·사유로 중복 신고 불가
- 같은 이벤트에 중복 응모 불가

## 데이터·영속성 (레퍼런스와의 차이 요약)

레퍼런스는 서버 DB를 전제로 하지만, Phase 1 현재는 **서버 없이 브라우저에만 저장**. 실서비스로 전환 시 개발자가 이 저장 포인트들을 서버 API로 바꿔야 함:

| 항목 | Phase 1 저장 위치 | 실서비스 |
|---|---|---|
| 작품 데이터 | 브라우저 로컬 저장소 단일 키 | 서버 DB |
| 검색 히스토리 | 브라우저 로컬 저장소 (계정별) | 서버 DB |
| 배너·이벤트·Pick·큐레이션 | 브라우저 로컬 저장소 | 서버 어드민 |
| 비회원 전화번호 | 초대 발송 후 **즉시 스크럽(제거)** — 작품엔 이름만 남김 | 동일 정책 유지 |
| 신고·경고·허위신고 카운터 | 브라우저 로컬 저장소 | 서버 + 감사 로그 |

### Phase 1 외부 연동 상태 (Phase 2 예정)

| 기능 | Phase 1 | Phase 2에 필요한 실연동 |
|---|---|---|
| 소셜 로그인 (카카오·구글·애플) | 버튼 누르면 즉시 로그인된 것처럼 처리 | 실제 OAuth |
| 이메일 로그인 | 어떤 이메일이든 즉시 통과 | 이메일 소유권 인증 |
| 이메일 발송 | 템플릿만 있고 실제 발송 안 됨 | 이메일 서비스 연동 |
| SMS·카카오 알림톡 | 콘솔 로그만 (5% 확률로 실패 시뮬) | 실제 메시징 게이트웨이 |
| DB 서버 | 브라우저 로컬 저장소 | 실서버 |
| OG 이미지 | 정적 메타태그만 | 동적 생성 |

이 수준의 목업은 PM 시연 목적상 의도적 유보입니다. 실서비스 전에 모두 실연동 필요.
