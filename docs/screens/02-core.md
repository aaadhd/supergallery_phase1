# 02. 핵심 기능 — 둘러보기 · 검색 · 업로드

> 이 문서 단독으로 이해 가능하도록 작성. 레퍼런스 명세를 이 문서 기준으로 업데이트해주세요.

---

## 사전 지식

### 작품(Work)이 가지는 개념 필드

| 필드 | 설명 |
|---|---|
| 제목 | 짧은 이름 (최대 20자, `TITLE_FIELD_MAX_LEN`) |
| 이미지 | 단일 작품 1장, 또는 그룹전시면 여러 장 |
| 커스텀 커버 | 이미지 배열과 별도로 카드 대표 이미지를 외부 파일로 지정 가능 (전시 작품 수에 안 들어감) |
| 소유자/작가 | 업로드한 회원 |
| 전시명 | 최대 20자 |
| 그룹명 | 그룹 전시일 때 필수 |
| 업로드 유형 | 개인 / 그룹 |
| 강사 업로드 여부 | 그룹 전시에서 업로더가 "저는 강사예요" 체크한 경우 참 |
| 이미지별 작가 | 그룹 전시에서 이미지 각각의 작가 지정 (회원 / 비회원 / 셀프) |
| 비공개 여부 | 작가가 직접 비공개 처리 |
| 숨김 여부 | 운영팀이 신고 처리로 비공개 처리 |
| 검수 상태 | 대기 / 승인 / 반려 |
| 반려 사유·메시지 | 운영팀이 작성 |
| 업로드 시각 | |
| 연결 이벤트 | 이벤트 응모 시 |

### 이 카테고리 공통 정책

- **시니어 친화 터치 영역**: 최소 44×44 픽셀
- **작품 이미지 비율 유지**: 잘라내기 금지 (시니어 미술 작가의 작품 훼손 방지)
- **작품 카드**: 하단에 좋아요·저장은 **상태 아이콘만** 노출, 숫자는 노출하지 않음. 댓글 기능은 Phase 1 범위 밖
- **확인 다이얼로그**: 앱 자체 디자인. 브라우저 네이티브 금지
- **모든 작품은 발행 → 검수 대기 → 운영팀 승인 → 피드 노출**이 원칙. 실서비스 전 검수 우회 모드는 꺼야 함

## SCR-CORE-01 · 둘러보기

| 항목 | 값 |
|---|---|
| URL | `/` (서비스 홈) |
| 접근 권한 | 비로그인 가능. 좋아요·저장·팔로우 같은 상호작용은 로그인 유도 |
| 진입 경로 | 앱 진입 시 기본 · 로고 탭 · 뒤로가기 |
| 레이아웃 | 공통 Layout (헤더 + 푸터 + 모바일 하단 네비게이션) |

### 화면의 목적
작품 발견의 메인 피드. 새 작품·작가를 자연스럽게 만나는 무한 스크롤. 개인 전시(작품 1장)와 그룹 전시(여러 작품) 카드가 혼합.

### 사용자 시나리오
- 주 사용자: 모든 회원 + 게스트
- 기대: "보러 들어왔다" → 큰 이미지로 감상 → 마음에 드는 것 좋아요/저장 → 작가 프로필로 더 보기

### 화면 구성

| 영역 | 요소 | 비고 |
|---|---|---|
| 상단 배너 캐러셀 | 자동 전환. 운영팀이 등록한 배너가 있으면 그것, 없으면 진행 중 이벤트 | 시니어 친화 — 일시정지 가능, 좌우 수동 스와이프 |
| 카테고리 탭 | 전체 / 개인 / 그룹 | 필터 역할 |
| 피드 카드 리스트 | 개인전시(이미지 1장+제목+작가) + 그룹전시(여러 이미지 묶음) 혼합 | 작품 비율 유지 |
| 카드 내 요소 | 커버 · 전시명(20자 컷) · 작가 아바타+이름 · 좋아요/저장 **상태 아이콘만** · "…" 메뉴(신고/공유) | 숫자 노출 금지 |
| 페이지네이션 | 스크롤 바닥 감지 시 24장씩 추가 로드 | |
| 빈 상태 | "아직 작품이 없어요" + 로그인 시 업로드 유도 | |

### 피드 정렬 알고리즘 (구현 기준: `feedOrdering.ts`)

작품은 **한 버킷에만** 배정된다. 풀은 대략 다음으로 나뉜다: Pick(최대 10) · 테마전 · 추천 작가 · **팔로우 작가 개인화**(로그인 시) · 신규(14일 이내) · 일반.

버킷별로 후보를 랭킹한 뒤, **고정 패턴으로 인터리브**하여 한 목록으로 섞는다(단순히 1→5번 버킷을 위에서 아래로 붙이지 않음).

버킷 내 정렬 요지:
- 좋아요·저장 건수는 **log1p 스케일**에 가중하고, 팔로우 중인 작가 작품에 추가 가산한다.
- 소스별 부스트·약한 랜덤 노이즈로 새로고침마다 순서가 다소 바뀐다.
- 이미 본 작품 ID(`seenIds`)는 같은 버킷 안에서 뒤로 밀린다.
- 댓글은 랭킹에 사용하지 않는다(Phase 1 범위 밖).

### 피드 노출 제외 조건

- 작가가 비공개 처리한 작품
- 검수 대기 / 반려 작품
- 운영팀이 숨김 처리한 작품
- 본인이 신고한 작품 (같은 사용자에겐 자동으로 안 보임)

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 | 비고 |
|---|---|---|
| 상단 배너 | 배너에 설정된 링크로 이동 (주로 이벤트 상세 `/events/:id` 또는 외부 링크) | — |
| 카테고리 탭 (전체/개인/그룹) | **같은 화면에서 필터만 변경** (URL 변화 없음) | 스크롤 최상단으로 |
| **피드 카드 (개인 전시)** 탭 | **작품 상세 모달** 열림 (풀스크린 감상, 화면 이동 없음) — 자세한 구성은 작품 상세 모달 스펙 참조 | 비로그인도 가능 |
| **피드 카드 (그룹 전시)** 탭 | 작품 상세 모달 열림 — 그룹 작품 중 첫 번째에 포커스, 좌우 화살표로 같은 전시 내 이동 | 동일 |
| 카드 내 작가 아바타/이름 | **작가 프로필 (`/profile/:artistId`)로 이동** | — |
| 카드 내 좋아요 버튼 | **화면 이동 없음** — 상태 토글만 | 비로그인이면 로그인 유도 모달 |
| 카드 내 저장 버튼 | 화면 이동 없음 — 상태 토글 | 동일 |
| 카드 내 팔로우 (해당 작가의 첫 피드에만 있음) | 화면 이동 없음 — 상태 토글 | 동일 |
| 카드 내 "…" → 신고 | 신고 모달 오픈 (같은 화면 위) | 로그인 필수 |
| 카드 내 "…" → 공유 | 기기 공유 시트 또는 링크 복사 — 화면 이동 없음 | — |
| 빈 상태 CTA (로그인 시) | 업로드 화면 (`/upload`)으로 이동 | — |
| 뒤로가기 | 이전 화면으로 복귀. 이 화면 재진입 시 이전 스크롤 위치 복원 | — |

**비로그인 상호작용 정책 (세션 1회 모달)**:
- 처음 시도 → 로그인 유도 모달 오픈
- 모달 닫은 이후 같은 세션 동안 같은 유형 시도 → 토스트만 ("로그인 후 사용 가능합니다")
- 새 세션(브라우저 재시작) → 다시 모달

### 부수 효과

| 시점 | 동작 |
|---|---|
| 진입 | 일일 둘러보기 포인트 +5 AP (하루 1회) |
| 진입 | 이전 스크롤 위치 복원 |

| 이탈 | 현재 스크롤 위치 저장 |

## SCR-CORE-02 · 검색

| 항목 | 값 |
|---|---|
| URL | `/search?q={키워드}` |
| 접근 권한 | 비로그인 가능 |
| 진입 경로 | 헤더 검색 아이콘 · 최근 검색 칩 선택 |
| 레이아웃 | 공통 Layout |

### 화면의 목적
**작가·작품(전시 단위)** 을 키워드로 찾는다. 자동완성 후보 + 최근 검색 + (Enter 후) 결과 목록.

### 사용자 시나리오
- 특정 작가·작품·전시명을 알고 찾으려는 사용자
- 기대: 입력 중 자동완성 후보 확인 → Enter로 검색 → 결과에서 프로필 또는 전시로 이동

### 화면 구성 (현 구현 기준)

| 영역 | 요소 |
|---|---|
| 상단 검색바 | 입력란(자동 포커스) + 우측 ✕(입력 지우기) |
| 자동완성 (검색 실행 전) | 작품·전시·그룹명·작가명 풀에서 부분 일치 후보, **최대 8건** |
| 검색 전 | 최근 검색 칩(전체 삭제 시 **확인 다이얼로그**) |
| 검색 후 | 상단 **Top matches**(첫 작가 1명 + 작품 2건) → **작가** 섹션(전체) → **작품** 섹션(그리드, 매칭 전체) |
| 빈 결과 | 안내 문구 + 최근 검색 칩 + **둘러보기 이동** (`/browse`는 `/`로 리다이렉트) |

### 검색 동작

- 검색 실행: **Enter** (또는 자동완성 항목 클릭). 타이핑만으로는 결과 갱신 없음.
- **랭킹** (`searchRank.ts`): 작품·작가 각각 구현된 규칙에 따름.
- **이벤트·탭 분리**: Phase 1 현재 UI에는 **없음** (단일 결과 흐름).

### 최근 검색 히스토리

- **상한 10개**
- 게스트 키와 로그인 계정 키 분리
- **로그인 시**: 게스트 히스토리를 계정 키로 병합한 뒤 **게스트 키는 삭제**

### 탭·액션 결과 (네비게이션)

| 대상 | 결과 |
|---|---|
| 작가 행 | `/profile/:artistId` |
| 작품 카드 | `/exhibitions/:workId` (전시·작품 동일 URL 체계) |

### Phase 2 보강
- 인기 검색어, 이벤트·탭별 결과, 오타 교정, 타이핑 디바운스 검색

## SCR-CORE-03 · 업로드

| 항목 | 값 |
|---|---|
| URL | `/upload` |
| 쿼리 파라미터 | `?editWorkId={id}` (편집 모드) · `?event={id}` (이벤트 응모) |
| 접근 권한 | **로그인 필수** — 비로그인이면 로그인으로 자동 이동 후 돌아옴 |
| 진입 경로 | 헤더 "작품 올리기" · 온보딩 완료 후 · 내 프로필 "새 작품" · 이벤트 상세 "참여" |
| 레이아웃 | 공통 Layout |

### 화면의 목적
개인 또는 그룹 전시 작품 발행 전 과정. 한 화면에 모든 입력을 펼쳐서 시니어가 단계 이동 부담 없이 진행.

### 사용자 시나리오
- **개인 작가**: 본인 작품 업로드
- **동호회·클래스 강사**: 수강생들 작품을 묶어 그룹 전시로 발행
- 기대: 이미지 드래그로 순서 바꾸기, 실수해도 쉽게 수정, 발행 전에 모든 항목 한 번에 확인

### 화면 구성 (단일 세로 스크롤)

#### 0. 업로드 유형
- 토글: **개인 전시** vs **그룹 전시** (라디오 카드)
- 그룹 선택 시 추가 영역(그룹명·작가 지정) 표시

#### 1. 이미지 추가
- 드롭존 + 파일 선택 버튼
- 추가된 이미지는 카드 리스트로 (썸네일 + 작품 제목 입력 + 작가 지정(그룹) + 삭제 + 드래그 핸들)
- **드래그 정렬**: 카드 위아래 이동 가능

**이미지 검증 규칙**

| 항목 | 규칙 | 위반 시 |
|---|---|---|
| 파일 수 | 최대 **10장** | "최대 10장까지 올릴 수 있어요" |
| 파일 크기 | 10MB 초과 차단 | "파일 크기가 너무 커요" |
| 포맷 | JPEG / PNG / WebP / GIF | "지원하지 않는 형식이에요" |
| 해상도 | 단변 800픽셀 이상 | "해상도가 너무 작아요. 800px 이상 권장" |
| 카메라 원본 | JPEG 메타데이터에 카메라 정보 있으면 거부 | "직접 촬영한 사진은 업로드할 수 없어요. 작품 사진을 올려주세요" |

카메라 원본 차단은 **시니어 작가의 휴대폰 카메라 직촬을 막고 작품 정식 사진 유도** 목적.

#### 2. 전시 정보
- **전시명** — 최대 20자 (필수)
- **그룹명** — 그룹 전시일 때 필수. 자동완성 (이전에 사용한 것 + 다른 작가가 만든 것 정규화 매핑)
- **강사 체크박스** — 그룹 전시에서 "저는 이 클래스 강사예요" 체크 시 → 강사 업로드로 표시됨. 강사가 1건이라도 발행하면 자동으로 프로필에 "수강생 작품" 탭 노출

#### 3. 작가 지정 (그룹 전시만)
이미지 카드별로 작가 지정 — 3가지 옵션:
- **회원 검색**: 가입된 작가 목록에서 선택
- **비회원 직접 입력**: 이름(실명) + 전화번호. 발행 후 이 사람에게 SMS 초대장이 자동 발송됨
- **셀프**: 업로더 본인을 작가로

#### 4. 원본 확인
체크박스 (필수): "본인이 직접 창작했으며 저작권을 보유하고 있습니다"
그룹 전시일 땐: "수강생들로부터 작품 게시 동의를 받았습니다"

#### 5. 발행 버튼
- [임시저장] — 한 번에 1개 슬롯 (현재 작업 보관용)
- [발행하기]

### 탭·액션 결과 (네비게이션)

| 탭 대상 | 결과 |
|---|---|
| 뒤로가기 / 헤더 로고 | 이미지가 있으면 "업로드를 취소하시겠어요?" 파괴적 확인 다이얼로그 → 확인 시 이전 화면으로 |
| 유형 토글 (개인/그룹) | **같은 화면에서 섹션 구성 변경** (URL 변화 없음) |
| 이미지 추가 드롭존/파일 선택 | 기기 파일 선택 창 열림 — 같은 화면에서 리스트 갱신 |
| 이미지 카드 드래그 | 순서 재정렬 — 화면 이동 없음 |
| 이미지 카드 삭제 아이콘 | 파괴적 확인 다이얼로그 → 확인 시 해당 카드 제거 |
| 그룹명 입력 | 드롭다운으로 자동완성 후보 노출 (같은 화면) |
| 작가 지정 - "회원 검색" 탭 | 검색 결과 드롭다운 (같은 카드 안) |
| 작가 지정 - "비회원 직접 입력" 탭 | 이름·전화 입력란으로 전환 (같은 카드 안) |
| [임시저장] | 현재 작업 저장 + 토스트 "임시저장되었어요". 화면은 유지 |
| **[발행하기]** | 아래 "발행 흐름" 단계 진행 (여러 모달 거침) |

### 발행 흐름

1. **기본 검증**: 최소 1장 이상 + 원본 체크 + 비속어(전시명·그룹명) + 그룹명 필수(그룹) + 작가 지정 필수(그룹)
2. **개인 전시 전환 제안**: 그룹 전시인데 유니크 작가가 1명뿐이면 → 모달 "한 작가의 작품만 있어요. 개인 전시로 바꿀까요?" → 동의 시 자동 전환
3. **세부정보 모달** (08 문서 MOD-04 참조): 작품별 제목 + 대표 커버 이미지 선택
4. **비회원 초대 발송 직전 확인 모달** — 비회원 작가가 있을 때만:
   ```
   초대장을 발송할까요?
   다음 분께 SMS/알림톡으로 초대장이 발송됩니다.
   • 김민수
   • 이영희

   ⚠️ 입력하신 이름이 그분의 실명(가입 시 입력할 이름)과 정확히 같아야 
   작품이 자동 연결됩니다. 호칭·별명(예: "민수쌤", "엄마")으로는 
   연결되지 않아요.

   [취소] [발송]
   ```
   취소 시 발행 자체 중단
5. 작품 저장 + 그룹명 정규화 등록
6. **포인트 적립**:
   - 첫 작품: +100 AP
   - 이후 작품: +20 AP (하루 2회까지)
   - 같은 달 4번째 발행: +50 AP 보너스
   - 그룹 전시 강사 업로드: +30 AP (월·강사 조합당 1회)
   - 그룹 전시 비강사 참여: +15 AP (월 5회까지)
7. **비회원 SMS 초대 발송**:
   - 한국 번호면 카카오 알림톡, 그 외는 SMS
   - 발송 후 작품에서 **전화번호 스크럽** — 회원 전화는 작품에 영구 저장 안 함 (프라이버시)
8. **발행 성공 → 자동 이동**: 내 프로필 전시 탭 (`/me?tab=exhibition`)으로 이동 + 상단에 "검수 후 공개돼요. 보통 24시간 내" 안내 배너 자동 노출 (dismissible). 편집 모드에서 재발행 시엔 배너 없음.
9. 편집 모드(`?editWorkId=`)로 진입한 경우 발행 성공 시엔 배너 없이 내 프로필 전시 탭으로 이동 (수정 완료 토스트만)

### 검수 정책
- 기본은 발행 직후 **검수 대기 상태** → 운영팀 승인 시 피드 노출
- 작가 프로필 전시 탭 도착 시 "검수 후 공개 (보통 24시간)" 안내 배너 (dismissible)
- PM 시연용으로 검수 우회 모드가 있지만 실서비스에서는 끈 상태로 운영

### 24시간 내 삭제 시 포인트 회수
발행 후 24시간 이내에 작품 삭제하면 적립된 AP가 **-20 회수**. 어뷰징 방지.
