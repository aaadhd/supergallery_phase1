## 0. 문서 읽는 법

### 0.1 대상 플랫폼

Artier Phase 1은 **웹 단일 플랫폼**이다. 네이티브 앱 래퍼 없음.

- **사용자 앱**: 모바일 우선(375px 기준) · 데스크톱 반응형. 루트 `/` 트리.
- **운영 어드민**: 데스크톱 우선(1280px 기준). `/admin` 하위.
- **인증·온보딩**: 독립 레이아웃(헤더·푸터 없음). `/login`·`/signup`·`/onboarding` 등.

### 0.2 다국어 전제

UI 기본 언어는 한국어(KO)이며 영어(EN) 전환 지원. 한국 사용자는 전화번호 `+82`로 판별하여 카카오 알림톡·SMS 대상. 그 외 지역은 이메일 중심. 상세는 [Policy §1 알림 채널](Policy_v1.md#1-알림-채널-정책) 참조.

### 0.3 이 문서의 범위

- 화면 ID 체계 (§1)
- 사용자 앱 모든 화면·모달·오버레이 (§2) — 10 영역 70+ 카드
- 운영 어드민 모든 화면·모달 (§3) — 12 영역 25+ 카드
- 공통 팝업·다이얼로그 (§4)
- Phase 1 · Phase 2 우선순위 (§5)
- 해소된 결정 · 열린 항목 (§6)
- 다음 문서 연결 (§7)

## 1. 화면 ID 체계

### 1.1 트랙 접두어

| 접두어 | 트랙 | 대상 사용자 |
|---|---|---|
| **USR-** | 사용자 앱 | 시니어·중장년 작가, 감상자 |
| **ADM-** | 운영 어드민 | 운영팀(Operator 권한) |
| **CM-** | 트랙 공통 팝업·오버레이 | 두 트랙이 모두 사용하거나 특정 트랙·화면에 귀속되지 않음 |

| **AUT** | 인증·온보딩 |

| 코드 | 영역 |
|---|---|
| **DSH** | 대시보드 |

| **ISU** | 미결 이슈 |

### 1.4 화면 카드 포맷

```
### [ID] · [화면 이름]
- 경로: [URL 또는 진입 동선]
- 구성: [화면 내 UI 블록을 영역 단위로]
- 연결: [이동 가능한 다음 화면 또는 모달 ID]
- 엔티티·정책: [ERD 엔티티 / Policy 섹션 / SystemArch 섹션]
- 우선순위: [P0 / P1 / P2]
```

필요 시 `**노트**:` 로 특이사항 추가.

### 1.5 우선순위 플래그

| 플래그 | 의미 |
|---|---|
| **P0** | 런칭 필수. 이 화면 없이 서비스 불가. |
| **P1** | 런칭 권장. 1~3개월 내 도입. |
| **P2** | 로드맵. Phase 2 이후. |

### 2.2 USR-BRW · 둘러보기 (1 카드)

- 경로: `/` (별칭 `/browse` → `/` 리다이렉트)
- 구성:
  - **상단 Header** (공통): 로고 · Home·Events·Search 텍스트 링크(PC) · 업로드 버튼(Plus, primary) · 알림(Bell, unread 배지) · 아바타 드롭다운 · 언어 토글(Globe)

  - **탭 바**: 전체 / 혼자 올리기 / 함께 올리기 (URL `?tab=all|solo|group` 갱신)
  - **작품 카드 그리드**: 모바일 2열 · 태블릿 3열 · 데스크톱 4~5열. 24건씩 무한 스크롤 + skeleton. 카드 = 썸네일(작품 톤 배경) + 좋아요·저장 상태 아이콘(카운트 비노출) + Pick·강사 배지 + 다장 이미지 개수 배지

  - **Footer**(조건부 숨김 on `/upload`): 정책·문의·FAQ 링크
- 연결: USR-EXH-01, USR-PRF-01, USR-EVT-02(이벤트 배너 탭), USR-AUT-02(비로그인 시), CM-02, CM-06
