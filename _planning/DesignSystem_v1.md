## §1. 공통 디자인 토큰 (전 제품 공유)

### 1.1 디자인 방향

**톤**: 시니어 작가의 작품이 주인공. UI는 뒤로 물러난다.
**키워드**: 차분 · 여유 · 존중.
**악센트 철학**: 로고 틸을 고정 악센트로 쓰지 않고, **모호·딥 플럼(plum, hue 328°)** 을 공용 primary로 사용한다. 온라인 갤러리의 차분한 공기감을 위한 선택이며, 로고 틸은 로고 자체에만 국소 적용한다.

모든 색 토큰은 **OKLCH** 로 정의해 다크 모드 전환 시 채도·명도를 독립적으로 조절한다. Figma Variables → 코드 변수 1:1 동기화.

### 1.2 컬러 팔레트

**라이트 모드 (기본)**

| 토큰 | 값(OKLCH) | 용도 |
|---|---|---|
| `background` | `oklch(0.97 0.014 85)` | 페이지 배경. 따뜻한 크림톤. |
| `foreground` | `oklch(0.21 0.022 280)` | 본문 기본 텍스트. |
| `card` | `oklch(0.995 0.008 85)` | 카드·모달 바닥면. 배경보다 약간 밝은 크림. |
| `card-foreground` | `oklch(0.21 0.022 280)` | 카드 내 텍스트. |
| `primary` | `oklch(0.46 0.075 328)` | 주요 CTA·강조. **딥 플럼**. WCAG AA ≈ 5.0:1. |
| `primary-foreground` | `oklch(0.985 0.01 328)` | primary 위 텍스트. |
| `secondary` | `oklch(0.955 0.018 85)` | 보조 배경(탭 비활성, 약한 블록). |
| `secondary-foreground` | `oklch(0.28 0.022 280)` | secondary 위 텍스트. |
| `muted` | `oklch(0.935 0.022 85)` | 조용한 배경(배지·비활성 입력). |
| `muted-foreground` | `oklch(0.43 0.022 280)` | 보조 텍스트. 배경 대비 ≈ 5.2:1. |
| `accent` | `oklch(0.93 0.035 328)` | 호버·활성 하이라이트 배경. |
| `accent-foreground` | `oklch(0.32 0.07 328)` | accent 위 텍스트. |
| `destructive` | `oklch(0.577 0.245 27.325)` | 복구 불가 액션(삭제·탈퇴). 코랄 레드. |
| `destructive-foreground` | `oklch(0.985 0 0)` | destructive 위 텍스트. |
| `border` | `oklch(0.875 0.018 85)` | 구분선·테두리. |
| `input` | `oklch(0.875 0.018 85)` | 입력 테두리. |
| `input-background` | `oklch(0.995 0.008 85)` | 입력 필드 바닥면. |
| `switch-background` | `oklch(0.815 0 0)` | 토글 오프 상태. |
| `ring` | `oklch(0.52 0.08 328)` | 포커스 링. |

**차트**: `chart-1 ~ chart-5` — 무채색 5단 그라데이션(라이트는 어두움→밝음, 다크는 반대).

**다크 모드**

같은 토큰 이름, OKLCH 값만 달라진다. `.dark` 클래스가 조상에 있으면 자동 전환. Phase 1은 시스템 토글 미제공(향후 설정 화면에서 선택하게 함).

**사용 원칙**
- 직접 HEX·RGB 쓰지 말 것. 언제나 토큰 이름으로 참조.
- `background` / `card` / `popover` / `muted` 는 **같은 패밀리**지만 레이어를 분리하려고 명도만 약간 띄운 것 — 계층을 무너뜨리려 새 토큰 만들지 말 것.

### 1.3 타이포그래피

**폰트 패밀리**

1. **Pretendard** (주)
2. Apple SD Gothic Neo / Malgun Gothic / 시스템 기본 sans-serif (폴백)

Pretendard는 한글·영문 타이포 균형이 맞춰져 있어 별도 폰트 믹스 금지. 영문 전용 폰트를 섞지 않는다.

**스케일**

| 역할 | 크기(rem → px) | Line-height | Letter-spacing | Weight |
|---|---|---|---|---|
| H1 | 1.375rem → 22px | 1.35 | -0.02em | 600 (medium) |
| H2 | 1.25rem → 20px | 1.4 | -0.015em | 600 |
| H3 | 1.0625rem → 17px | 1.45 | -0.012em | 600 |
| H4 | 0.9375rem → 15px | 1.5 | -0.008em | 600 |
| Label | 0.9375rem → 15px | 1.5 | — | 600 |
| Button | 0.9375rem → 15px | 1.45 | — | 600 |
| Body | 1rem → 16px | 1.5 | — | 400 |
| Input | 1rem → 16px | 1.5 | — | 400 |

> **iOS 확대 방지**: 입력 필드(input·textarea·select) 전부 `font-size ≥ 16px` 필수. 이보다 작으면 iOS Safari가 자동 줌.

**폰트 두께**: normal 400, medium 600. 두께 600을 medium으로 부르고 실제 사용한다(Pretendard 700은 제목 강조용만). 시니어 UX상 가늘게(300/350) 금지.

**글꼴 크기 조절 (사용자 설정)**

`USR-STG-01`(설정 홈)에서 **작게 / 보통 / 크게** 3단 토글. 루트 폰트 크기를 100% / 110% / 120%로 조정해 전 스케일이 비례 확대된다.

### 1.4 보더 라디우스

| 토큰 | 값 | 용도 |
|---|---|---|
| `radius` (base) | 0.5rem (8px) | 기본 카드·버튼 |
| `radius-sm` | base − 4px = 4px | 입력·소형 칩 |
| `radius-md` | base − 2px = 6px | 버튼 메디엄 |
| `radius-lg` | base = 8px | 카드 표준 |
| `radius-xl` | base + 4px = 12px | 모달·바텀시트 |

리스트 아이템은 12px 고정. 원형은 프로필·아바타 한정.

### 1.5 그림자

shadcn 기본 팔레트를 따르되, **카드는 최소한의 그림자**만 사용해 갤러리 톤을 유지한다.

| 토큰 | 용도 |
|---|---|
| shadow-sm | 기본 카드 |
| shadow | 호버·선택 상태 |
| shadow-lg | 모달·시트 |

작품 썸네일 카드는 그림자 대신 **1px 보더 + 배경 레이어**로 깊이감을 준다(고해상 이미지가 주인공).

### 1.6 모션

| 프리셋 | 지속 | 이징 | 용도 |
|---|---|---|---|
| 인터랙티브 기본 | 200ms | ease | 버튼·카드 배경·그림자 전환 |
| transform | 200ms | ease | hover scale, active press |
| fade-in-up | 300~500ms | ease | 신규 콘텐츠 등장 |
| slow-pan | 20s | ease-in-out infinite alternate | 배경 이미지 느린 팬 |
| slow-zoom | 15s | ease-in-out infinite alternate | 스플래시·히어로 배경 |
| slow-drift | 30s | ease-in-out infinite | 이벤트 히어로 |
| confetti-pop | 0.95s | ease-out forwards | 업로드 완료 등 축하 |

**원칙**
- 시니어 대상: 화면 전환·스크롤 애니메이션은 **짧고 느리게**(200ms 내외). 번쩍이는 효과 금지.
- `prefers-reduced-motion: reduce` 를 존중(현재는 수동, v1.1에서 자동 처리 추가 예정).

### 1.7 아이콘

- **라이브러리**: 라인·스트로크 기반 단일 팩 사용. 복수 팩 혼용 금지.
- **크기**: 버튼 내 16px, 툴바·카드 액션 20px, 히어로 강조 24px.
- **스트로크 두께**: 시니어 가시성 목적으로 기본 2px 유지. 1.5 이하로 낮추지 말 것.

### 1.8 핵심 시각 법칙 (깨지면 일관성 붕괴)

1. **터치 타깃 44px** — 모든 인터랙티브 요소 최소 44×44px. 아이콘 버튼도 예외 없음.
2. **폰트 ≥ 16px** — 입력 필드는 예외 없이 16px 이상(iOS 줌 방지).
3. **호버는 데스크톱만** — 포인터 정밀(fine) + 호버 가능 + 뷰포트 1024px 이상에서만 호버 효과. 모바일·태블릿에선 호버 효과 없음, 대신 `active` 상태에서 opacity 0.8 + scale 0.98.
4. **포커스 가시성** — 모든 인터랙티브에 3px 포커스 링(primary 50%). 2px solid 아웃라인 + 2px 오프셋이 기본.
5. **색상 단독 정보 금지** — 상태(승인/반려/정지)는 항상 아이콘 + 라벨 동반.
6. **확인 다이얼로그 일원화** — 브라우저 네이티브 confirm/alert 금지. 모든 확인은 공통 다이얼로그(CM-01) 사용, 복구 불가 액션은 빨간 확정 버튼.
7. **문자열 하드코딩 금지** — 모든 UI 텍스트는 다국어 훅 경유.

## §2. UA 가이드 (사용자 앱 — 모바일 375px)

사용자 앱은 시니어(50대 이상) 작가·감상자가 주 타깃이다. 아래 규칙은 Phase 1 전체에 적용된다.

### 2.1 레이아웃

- **최대 폭**: 본문 컨테이너 640px 기준. 1024px 이상에서만 2열·3열 그리드 허용.
- **그리드 갭**: 작품 그리드는 8px(모바일) → 12px → 16px(데스크톱)로 점진 확장.
- **패딩**: 페이지 좌우 16px(모바일) / 24px(태블릿 이상). 카드 내부 16px.
- **safe-area**: 하단 스티키 영역은 `safe-area-inset-bottom` 반영.

### 2.2 네비게이션

**PC(≥ 1024px) — Header 고정형**
- 좌측: 로고(Artier) → USR-BRW-01
- 중앙: Home · Events · Search 텍스트 링크
- 우측: 업로드 버튼(primary), 알림(unread 배지 동반), 아바타 드롭다운(설정·로그아웃), 언어 토글(KO/EN)

**모바일(< 1024px)**
- 상단 Header: 로고 + 알림 + 아바타 + 업로드
- 하단: Footer(USR-UPL-02 경로에선 숨김). Footer는 스크롤 시 표시되는 보조 링크(정책·문의·FAQ)로 구성되고, 주요 내비는 상단 Header에만 있음.
- 검색은 상단 검색 탭이 아닌 **전용 화면**(USR-SRC-01) 진입 방식.

> Phase 1에선 하단 탭바(Tabbar) 형태의 글로벌 내비를 쓰지 않는다. 시니어 UX 검토 결과 상단 Header + 명시적 진입이 오조작이 적었다.

### 2.3 컴포넌트

아래는 사용자 앱에서 제공하는 공통 컴포넌트 프리셋. 신규 컴포넌트를 만들기 전, 이 목록에서 대체 가능한지 먼저 확인.

**레이아웃·구조**: Card, AspectRatio, ScrollArea, Separator, Resizable
**내비·디스클로저**: Accordion, Breadcrumb, Collapsible, Menubar, NavigationMenu, Pagination, Tabs, Sheet
**입력**: Input, PasswordInput, Textarea, Select, Checkbox, RadioGroup, Switch, Slider, InputOTP, Label, Form, Toggle, ToggleGroup
**오버레이·피드백**: Alert, AlertDialog, Dialog, Drawer, DropdownMenu, ContextMenu, Popover, Tooltip, Toast, Skeleton, Progress, Command
**콘텐츠**: Avatar, Badge, Button, Calendar, Carousel, Table, HoverCard

**Artier 전용 컴포넌트**
- SplashScreen · ConfirmDialog(CM-01) · LoginPromptModal(CM-02) · ReportModal(CM-03) · CookieConsent(CM-05) · OfflineBanner(CM-06) · ErrorBoundary(CM-07) · SocialSignupModal(USR-AUT-05) · ProfileImageModal(USR-PRF-03) · ExternalLinksEditor · WorkDetailModal(USR-EXH-01) · DeepZoomViewer(대형 이미지 줌 뷰어) · CopyrightProtectedImage(우클릭·드래그 차단) · WorkCard · ExhibitionCard · RequiredMark · Header · Footer

**버튼 변형**

| 변형 | 용도 | 비고 |
|---|---|---|
| default | 기본 primary CTA | 플럼 배경 + 흰 텍스트 |
| destructive | 삭제·탈퇴 | 코랄 레드. 확인 다이얼로그의 복구 불가 액션 스타일과 짝 |
| outline | 보조 CTA | 흰 배경 + 테두리 |
| secondary | 약한 강조 | 크림 배경 |
| ghost | 텍스트 느낌 | 배경 없음, 호버 시 accent |
| toolbar | 오버레이 툴바 | 투명 배경. 작품 뷰어 등 |
| link | 인라인 링크 | 언더라인·primary 색 |

**버튼 크기**

| 크기 | 높이 | 용도 |
|---|---|---|
| default | 44px | 주요 CTA |
| sm | 36px | 툴바·보조 |
| lg | 48px | 스플래시·온보딩 등 랜딩 페이지 메인 CTA |
| icon | 44×44 | **터치 타깃 최소치 — 축소 금지** |

**배지 변형**: default / secondary / destructive / outline

**Artier의 공개 배지 3종**(§1.8 원칙상 색 단독 정보 금지 — 항상 아이콘·라벨 동반):

| 배지 | 조건 | 수명 |
|---|---|---|
| ★ Artier's Pick | Pick 선정 이력 | 영구 |
| 이벤트 선정작 | 이벤트·공모형 테마전 선정 이력 | 영구 |
| 강사 | 강사 역할 업로드 이력 1건 이상 | 자동 파생 |

본인 전용 상태 배지("확인 중"·"게시 불가")는 위와 별개 — 타인에게 보이지 않는다.

### 2.4 작품 카드 패턴

**WorkCard (작품·전시 썸네일)**
- 비율: 4:5 또는 1:1
- 이미지: 우클릭·드래그 차단(CopyrightProtectedImage). 작품은 **잘라내지 말고 원본 비율**(contain)로 보여줌.
- 호버 오버레이: 모바일은 항상 약한 그라데이션, 데스크톱 호버 시 진한 그라데이션
- 하단: **좋아요·저장 상태 아이콘만** (카운트 숫자 비노출)
- 배지: Pick · 강사 · 검수 상태(본인에게만)

**ExhibitionCard (전시 커버)**
- 비율: 4:5. 커스텀 커버가 있으면 전면 표시.
- 오버레이: 전시명 + 그룹명 + 이미지 개수 배지(다장)

### 2.5 알림·피드백

- **토스트**: 성공·에러 텍스트만. 인라인 액션 버튼 없음.
- **인라인 배너**: 업로드 카메라 차단·오프라인·쿠키 동의 — 컴포넌트 상단 고정.
- **확인 다이얼로그**: **제목에 대상 이름**(전시명·파일명 등) 포함, **설명에 영구 삭제·복원 불가 여부** 명시.

## §3. 어드민 가이드 (ADM — 데스크톱 1280px)

운영팀이 사용하는 백오피스. 데이터 밀도를 허용하고 키보드·마우스 워크플로우 중심.

### 3.1 레이아웃

- **기준 폭**: 1280px. 1024px 이하에서도 동작하지만 표·사이드바가 축약되어 장시간 작업엔 부적합.
- **컨테이너**: 좌측 고정 사이드바(240px) + 상단 헤더 + 메인 영역.
- **여백**: 메인 좌우 24px / 상하 16px, 카드 내부 16px.
- **테이블 밀도**: 본문 14px 허용(어드민 한정 예외. 사용자 앱은 ≥ 15px).

### 3.3 컴포넌트 (어드민 특화)

사용자 앱 컴포넌트를 기본으로 쓰되, 어드민 한정으로 다음 패턴을 허용한다.

**데이터 테이블**
- Table 기반. 컬럼 고정(sticky) 허용.
- 행 선택: 단일 체크박스만. **다중 선택 Shift+Click 금지** — 사용자 앱 금지 원칙을 어드민에도 동일 적용.
- 필터 바: 상단에 Select + Input + Date picker 조합. 필터 상태를 URL 쿼리에 반영해 공유 가능.
- 정렬: 한 번에 한 컬럼만.

**액션 다이얼로그**
- 단순 확인 → CM-01(확인 다이얼로그)
- 사유 입력 필요(정지·반려) → 사유 선택·입력 전용 Dialog + 폼
- 복구 불가 액션 → 빨간 확정 + "이 작업은 되돌릴 수 없습니다" 문구 필수

**빈 상태(Empty State)**
- 아이콘 + 안내 문구 + 주요 CTA 1개
- "추가하기" / "새로 만들기" 버튼은 + 아이콘 좌측 + 텍스트

**폼 구성**
- Form + Label + Input 필수 조합. 필수 입력은 빨간 별 + 스크린 리더 전용 "필수" 라벨.
- 긴 폼은 Accordion 또는 Tabs로 섹션 분리. 단일 스크롤 1500px을 넘지 말 것.

### 3.4 색·상태 컨벤션 (어드민 한정)

| 상태 | 배지 색 | 사용처 |
|---|---|---|
| 승인(approved) · 활성 | secondary(회색) · 체크 아이콘 | 검수·이벤트·파트너 |
| 대기(pending) · 시작 전 | outline(회색 보더) | 검수·체크리스트 |
| 반려(rejected) · 차단됨 | destructive(코랄 레드) | 검수·체크리스트 |
| 진행 중(in-progress) | default(플럼) | 체크리스트·이슈 |
| 경고(warning) | 노랑 배지(커스텀) | 신고 경고 누적 표시 |
| 정지(suspended) | destructive 풀배지 | 회원 |

> 어드민이 사용자 앱보다 색 종류가 많지만, **같은 의미면 같은 색**. 다른 의미인데 같은 색으로 재사용 금지.

## §4. 디자인 시스템 관리

### 4.1 토큰 관리 원칙

- **단일 소스**: 모든 색·라디우스·타이포 값은 이 문서의 §1 토큰 표를 단일 소스로 한다. 직접 HEX·RGB·px 값을 화면별로 다르게 쓰지 않는다.
- **다크 모드**: 같은 토큰 이름에 대해 라이트/다크 두 값을 나란히 유지. 새 토큰을 추가하면 두 모드 값을 모두 정의하고 표에 반영.
- **오버라이드**: 컴포넌트 레벨의 예외는 허용하되, 최상위 토큰(예: `primary` 자체)을 화면마다 다르게 재정의하지 말 것.

### 4.2 신규 컴포넌트 추가 규칙

1. 공통 컴포넌트 프리셋(§2.3)으로 해결 가능한지 먼저 확인
2. 커스터마이즈가 2~3줄 내라면 사용처에서 클래스만 확장
3. 재사용 3회 이상 + 구조가 다른 경우에만 새 Artier 전용 컴포넌트 신설
4. 단방향 의존: 공통 UI → 공유 컴포넌트 → 페이지 → 어드민. 역방향 의존 금지.

### 4.3 알려진 제약 (v1.1에서 개선 예정)

- `prefers-reduced-motion` 자동 처리 없음 → 수동으로 motion-safe / motion-reduce 유틸 추가 필요
- 다크 모드 토글 UI 없음 — 토큰은 준비됐으나 설정에서 선택하는 화면 미구현
- 차트 컬러는 무채색 5단만 — 다채색 필요 시 별도 팔레트 설계 필요
