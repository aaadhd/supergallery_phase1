## 0. 문서 사용 안내

### 0.1 카드 포맷

각 화면은 아래 항목을 포함한다. **수용기준(AC)** 은 Given-When-Then 형식으로 작성한다.

```
### [ID] · [화면 이름]

**목적**: 이 화면이 존재하는 이유 (1문장)
**트리거**: 이 화면에 도달하는 진입 동선
**우선순위**: P0 / P1 / P2

#### 입력
- 사용자가 제공하거나 시스템이 읽는 데이터

#### 처리
- 시스템이 수행하는 비즈니스 로직

#### 출력
- 화면에 렌더링되거나 기록되는 결과

#### 수용기준 (Acceptance Criteria)
- AC-01: Given [전제] / When [행위] / Then [기대 결과]

#### 엣지케이스
- EC-01: [상황] → [기대 처리]

#### 의존
- 엔티티: [ERD 엔티티]
- 정책: [Policy §x / SystemArch §x 참조]
- 연결 화면: [이동 가능한 다음 화면 ID]
```

### 0.2 공통 전제

| 항목 | 전제 |
|---|---|
| 기준 해상도 | 모바일 375px 우선 · 데스크톱 반응형 |
| 다국어 | KO/EN 기본. 고정 폭 금지, i18n 키 분리 |
| 터치 타깃 | 최소 44×44px (`[Policy §19.1](Policy_v1.md#19-접근성시니어-친화-정책)`) |
| 입력 폰트 | ≥ 16px (iOS 줌 방지) |
| 확인 다이얼로그 | 공통 다이얼로그 `CM-01` 사용. 브라우저 네이티브 confirm 금지 |
| 인증 | 소셜 3종(카카오/구글/애플) + 이메일. Phase 1 인증은 모의 |
| 수익화 | Phase 1 무료. Phase 2 이후 수익 모델 별도 논의 |

### 0.3 공통 상단바 패턴

| 화면 유형 | 뒤로가기 | 상단바 스타일 | 예시 |
|---|---|---|---|
| **최상위 진입** (헤더의 로고·주요 링크로 바로 접근) | 없음 | 로고 + 내비(Home/Events/Search) + 알림·아바타·업로드 | USR-BRW-01, USR-EVT-01, USR-SRC-01 |
| **모달** (피드·프로필에서 카드 탭) | X 버튼 | 오버레이 전체 배경 | USR-EXH-01, USR-PRF-02, USR-PRF-03 |
| **독립 페이지 (뎁스)** | 있음 | 뒤로가기 + 타이틀 | USR-NTF-01, USR-STG-01, USR-INF-01~07 |
| **인증·온보딩** | 상황별 | 로고만. 헤더·푸터 없음 | USR-AUT-01~07 |

| 항목 | 기준 |

| 화면 전환 응답 | < 300ms (클라이언트 전환) |
| API 응답 | P95 < 500ms (리스트), P95 < 200ms (단건) — Phase 2 백엔드 연동 기준 |
| 이미지 로딩 | Progressive · WebP 변환, 썸네일·원본 이중 해상도 |
| 접근성 | 모든 상호작용 요소에 접근성 라벨. 포커스 가시성 확보 |
| 로깅 | 화면 진입·이탈·주요 CTA 탭 이벤트 → Analytics |

### 0.5 공통 전역 동작

**비로그인 보호 액션(좋아요·저장·팔로우·신고)**
- 비로그인 상태에서 시도 시 `CM-02`(로그인 유도 모달) 오픈.
- 세션당 1회만 모달, 이후는 토스트("로그인이 필요합니다"). 모달 CTA는 `/login?redirect=<현재경로>`.

**검수 상태 노출**
- 본인 프로필에서만 본인 전시에 검수 배지 노출. 타인 프로필·피드에서는 승인 완료 전시만 표시.
- 배지: `pending`(확인 중), `rejected`(게시 불가). 상세는 각 화면 카드 참조.

**로컬 자정 기준**
- 배너 노출 기간·이벤트 상태·업로드 기록일은 모두 **사용자 로컬 시간대 YYYY-MM-DD** 기준.

## 1. USR-AUT · 인증·온보딩

### USR-AUT-01 · 스플래시

**목적**: 앱 초기화, 세션 유효성 검사, 최초 진입 여부 판별 후 적절한 화면으로 라우팅.
**트리거**: 앱 실행(콜드 스타트·웜 스타트).
**우선순위**: **P0**

#### 입력
- 로컬 저장된 세션 상태
- 스플래시 확인 여부 플래그(최초 방문)

#### 처리
1. 세션 유효성 확인 → 유효하면 사용자 정보 로드.
2. 스플래시 확인 여부 플래그 확인:
   - 미확인(최초 진입) → 로고·페이드 연출 ~1.5초
   - 이미 확인 → 짧게 깜박(~0.3초)
3. 인증 상태와 목적 경로에 따라 라우팅.

#### 출력
- 로고 + 배경 연출
- 라우팅 결과: `/`(USR-BRW-01) 또는 `/login`(USR-AUT-02)

#### 수용기준
- AC-01: Given 로그인 세션 유효함 / When 앱 실행 / Then 스플래시 후 `/` 이동.
- AC-02: Given 로그인 세션 없음 / When 앱 실행 / Then 스플래시 후 홈(`/`)에 게스트 상태로 이동. `/login`으로 강제 리다이렉트 하지 않음.
- AC-03: Given 이번 세션에 이미 스플래시 확인 / When 앱 재진입 / Then 스플래시를 짧게 표시하거나 생략.

#### 엣지케이스
- EC-01: 세션 토큰이 유효하지만 프로필 정보 로드 실패 → 로그아웃 상태로 처리하고 `/`로 이동.
- EC-02: 정지 계정 세션 → 로그아웃 후 `/login`으로 이동 + 정지 안내(USR-AUT-07) 자동 노출.

#### 의존
- 엔티티: USER_PROFILE, AUTH_SESSION
- 정책: [Policy §2.4 세션](Policy_v1.md#2-가입본인인증-정책)
- 연결 화면: USR-BRW-01, USR-AUT-02, USR-AUT-07

### USR-AUT-02 · 로그인

**목적**: 기존 회원 로그인 및 신규 가입·비밀번호 재설정 진입점 제공.
**트리거**: 헤더 로그인 버튼, 보호 액션 시도(`CM-02`), 직접 `/login` 진입, 정지 계정 감지 시 리다이렉트.
**우선순위**: **P0**

#### 입력
- 소셜 로그인 버튼 3종(카카오/구글/애플) — 사용자 언어(KO/EN)에 따라 순서 변경
- 이메일·비밀번호(옵션)
- 비밀번호 재설정 링크(USR-AUT-04)
- `redirect` 쿼리 파라미터 (선택)

#### 처리
1. 이미 로그인 상태라면 `/` 또는 `redirect` 경로로 자동 이동.
2. 소셜 로그인 버튼 탭:
   - 기가입자 → 세션 생성 후 `redirect` 경로(없으면 `/`)로 이동.
   - 신규 → USR-AUT-05(소셜 최초 가입 모달).
3. 이메일 로그인:
   - 입력 검증(이메일 형식, 비밀번호 8자+영문·숫자).
   - 성공 시 세션 생성 후 이동. 실패 시 인라인 에러.
4. 정지 계정 감지 시 로그인 차단 + USR-AUT-07 안내.
5. `redirect` 파라미터는 **내부 경로(`/`로 시작)** 만 허용. 외부 URL은 무시.

#### 출력
- 로그인 성공 → 세션 생성, 이전 화면/루트로 이동
- 실패 → 에러 메시지, 입력 필드 포커스 복원
- 만 14세 이상 고지 문구 상시 표시

#### 수용기준
- AC-01: Given 이미 로그인 / When `/login` 접근 / Then `/`로 자동 리다이렉트.
- AC-02: Given 유효한 이메일·비밀번호 / When 로그인 시도 / Then 세션 생성 + `redirect` 경로 또는 `/`로 이동.
- AC-03: Given `redirect=https://evil.com` / When 로그인 성공 / Then `redirect` 무시하고 `/`로 이동.
- AC-04: Given 소셜 로그인 + 기가입자 / When 탭 / Then `/`로 이동.
- AC-05: Given 소셜 로그인 + 신규 / When 탭 / Then USR-AUT-05 모달 오픈.
- AC-06: Given 정지 계정 / When 로그인 시도 / Then USR-AUT-07 안내 후 세션 미생성.

#### 엣지케이스
- EC-01: 소셜 인증 취소(사용자가 외부 창 닫음) → 토스트 "로그인이 취소되었습니다".
- EC-02: 이메일 가입자가 소셜 로그인 시도 → 기존 계정 안내("이 이메일은 이메일 로그인 계정입니다").

#### 의존
- 엔티티: USER_PROFILE, AUTH_SESSION, SANCTION
- 정책: [Policy §2 가입·본인인증](Policy_v1.md#2-가입본인인증-정책) · [Policy §12 신고·제재](Policy_v1.md#12-신고제재-정책)
- 연결 화면: USR-BRW-01, USR-AUT-03~07

### USR-AUT-03 · 회원가입 (이메일)

**목적**: 이메일 기반 신규 계정 생성.
**트리거**: USR-AUT-02에서 "회원가입" 링크 탭.
**우선순위**: **P0**

#### 입력
- **Step 1**: 이메일 + 비밀번호
- **Step 2**: 닉네임 + 생년월일
- **Step 3**: 약관 동의 (필수 2종 + 선택 1종: 마케팅)

#### 처리
1. Step 1: 이메일 형식 검증 + 중복 검사. 비밀번호 8자 + 영문·숫자 각 1자.
2. Step 2: 닉네임 2~20자 + 비속어 필터. 생년월일 만 14세 이상 검증.
3. Step 3: 필수 약관 동의 미체크 시 진행 차단. 전체 동의 체크박스 제공.
4. 완료 시 세션 생성 + 온보딩 프리필 설정(이메일) + USR-AUT-06 이동.

#### 출력
- 진행 표시바 1/3 → 2/3 → 3/3
- Step별 에러 메시지
- 가입 완료 시 세션 + 프리필 → 온보딩 이동

#### 수용기준
- AC-01: Given 이미 등록된 이메일 / When Step 1 제출 / Then 인라인 에러 "이미 가입된 이메일입니다".
- AC-02: Given 비밀번호 7자 / When Step 1 제출 / Then 인라인 에러 "비밀번호는 8자 이상이어야 합니다".
- AC-03: Given 닉네임 "바보" / When Step 2 제출 / Then 비속어 필터 에러.
- AC-04: Given 생년월일 만 13세 / When Step 2 제출 / Then 에러 "만 14세 이상만 가입할 수 있습니다".
- AC-05: Given 필수 약관 미체크 / When Step 3 제출 / Then "필수 약관에 동의해 주세요".

#### 엣지케이스
- EC-01: Step 간 브라우저 뒤로가기 → 이전 Step 값 유지(입력 손실 방지).
- EC-02: 가입 완료 직후 네트워크 단절 → 재시도 토스트 + 세션 재생성 시도.

#### 의존
- 엔티티: USER_PROFILE
- 정책: [Policy §2](Policy_v1.md#2-가입본인인증-정책) · [Policy §9.4 닉네임·실명](Policy_v1.md#9-작품명전시명그룹명-정책) · [Policy §20 확정 수치](Policy_v1.md#20-확정-수치-종합표)
- 연결 화면: USR-AUT-06

### USR-AUT-04 · 비밀번호 재설정

**목적**: 이메일 가입 사용자의 비밀번호 재설정 링크 발송.
**트리거**: USR-AUT-02에서 "비밀번호를 잊으셨나요?" 링크.
**우선순위**: **P0**

#### 입력
- 이메일 주소

#### 처리
1. 이메일 형식 검증.
2. 등록 여부와 무관하게 **동일한 안내**를 보여준다(계정 존재 유무 노출 방지).
3. 실제 재설정 링크는 이메일로 발송(Phase 1은 모의 발송).

#### 출력
- "입력하신 이메일로 재설정 링크를 보냈습니다. 메일함을 확인해 주세요."
- 로그인 화면 복귀 버튼

#### 수용기준
- AC-01: Given 유효한 이메일 입력 / When 제출 / Then 발송 안내 표시 + 모의 이메일 로그 기록.
- AC-02: Given 미등록 이메일 / When 제출 / Then 동일한 발송 안내 표시(계정 존재 노출 없음).

#### 의존
- 엔티티: USER_PROFILE, EMAIL_LOG(모의)
- 정책: [Policy §1.1 법적 의무 알림은 이메일](Policy_v1.md#1-알림-채널-정책)
- 연결 화면: USR-AUT-02

### USR-AUT-05 · 소셜 최초 가입 모달

**목적**: 소셜 인증 성공 시 약관 동의·닉네임을 수집해 Artier 가입을 완결.
**트리거**: USR-AUT-02에서 소셜 로그인 + 미가입자 판정.
**우선순위**: **P0**

#### 입력
- 소셜 제공자 기본 정보(이메일·표시명) — 자동 수신, 사용자 수정 가능
- 약관 동의(전체/필수 2종 + 선택 1종)
- 닉네임(2~20자, 비속어 불가)

#### 처리
1. 닉네임·약관 검증 후 소셜 가입 플래그 + 프리필 임시 값 설정.
2. 계정 생성 + 세션 발급 → 온보딩 이동.
3. 소셜 가입 완료 표시(제공자별 플래그).

#### 출력
- 모달 UI 내부의 3 섹션(안내 문구 · 약관 · 닉네임)
- 완료 시 온보딩 이동

#### 수용기준
- AC-01: Given 소셜 인증 성공(카카오) / When 모달에서 닉네임·약관 입력 / Then 계정 생성 + 온보딩 이동.
- AC-02: Given 닉네임 미입력 / When 완료 시도 / Then 에러 "닉네임을 입력해 주세요".
- AC-03: Given 필수 약관 미체크 / When 완료 시도 / Then 에러.

#### 엣지케이스
- EC-01: 모달 외부 탭·ESC → 닫지 않음(소셜 인증 세션이 이미 존재하므로 취소는 상단 X만 허용).

#### 의존

### USR-AUT-06 · 온보딩

**목적**: 가입 직후 프로필 기본 정보 수집 + 첫 둘러보기로 유도.
**트리거**: USR-AUT-03 완료 / USR-AUT-05 완료 / 초대 링크 랜딩에서 신규 가입.
**우선순위**: **P0**

#### 입력
- **Step 0**: 환영 문구 + 시작
- **Step 1**: 실명·닉네임·전화번호·이메일·프로필 이미지
  - 가입 경로에서 받은 정보가 있으면 프리필(사용자 수정 가능)
- **Step 2**: 완료 + 첫 업로드 유도

#### 처리
1. Step 1 입력 검증:
   - 실명: 2~20자 + 비속어 불가. **모든 경로 필수**.
   - 닉네임: 2~20자 + 비속어 불가.
   - 전화번호: 숫자 7자리 이상 + 중복 가입 검사.
   - 이메일: 형식 + 중복(프리필과 동일하면 허용).
   - 프로필 이미지: 5MB 이하.
2. Step 1 완료 시 AP +30 적립(백그라운드, UI 미노출).
3. 초대 자동 매칭 시도([Policy §3.5](Policy_v1.md#3-비가입자-초대-정책)):
   - 이름·전화 모두 일치 → 초대된 전시에 사용자 ID 바인딩.

4. Step 2에서 "첫 작품 올리기" CTA → USR-UPL-02 또는 "둘러보기" → USR-BRW-01.

#### 출력
- Step 별 폼 · 진행 표시
- 프로필 데이터 저장
- 프리필 임시 값 일괄 정리

#### 수용기준
- AC-01: Given 가입 경로에서 이메일 프리필 있음 / When 온보딩 Step 1 진입 / Then 이메일 필드에 프리필된 값 + 수정 가능.

## 2. USR-BRW · 둘러보기

### USR-BRW-01 · 홈 피드

**목적**: 승인된 전시를 여러 큐레이션 소스(Pick·기획전·추천 작가·팔로잉 신작·신규·일반)에서 엮어 시니어 사용자가 부담 없이 탐색할 수 있는 피드로 제공.
**트리거**: 루트 진입(`/`), 헤더 로고 · Home 링크, 앱 런치 후 기본 진입, 딥링크에서 로그인 성공 후 기본 복귀.
**우선순위**: **P0**

#### 입력
- 전체 전시 중 **승인(`approved`) + 미숨김(`!isHidden`)** 만 후보
- 현재 Pick 목록(최대 10), 기획전 포함 전시, 추천 작가 목록
- 활성 배너(최대 5), 오늘 기준 활성 이벤트
- 사용자의 팔로잉 작가 ID, 좋아요·저장·숨김(신고자 필터) 상태
- 이미 본 작품 기록(최근 N건, 로컬)
- 탭 선택(전체 / 혼자 올리기 / 함께 올리기) — URL 쿼리 `?tab=<all|solo|group>`

#### 처리

**1) 배너 캐러셀 구성**
- 어드민 배너(활성 · 기간 내) + 활성 이벤트를 합쳐 1개 슬라이드 시퀀스로 구성.
- 5초 자동 회전, 드래그·좌우 버튼으로 수동 전환 가능.
- 배너 탭 시 외부 링크 또는 해당 이벤트 상세(USR-EVT-02)로.
- 이벤트 슬라이드는 이벤트 상태에 따라 "참여하기" CTA 활성화.
- **배너·이벤트 모두 0건** → 캐러셀 영역 자체 숨김(§EC-02).

**2) 피드 버킷 분류** ([Policy §16.1](Policy_v1.md#16-피드-랭킹검색-스코어링-정책))
- Pick → Theme(기획전에 포함된 전시) → Featured(추천 작가의 전시) → Personalized(팔로잉 작가의 전시) → Recent(14일 이내 업로드) → Rest(나머지 일반)
- 각 전시는 **우선 순위가 높은 버킷에만 할당**(중복 방지).
- 이미 본 작품은 해당 버킷 **하단**으로 밀려남.

**3) 점수 계산**
```
base = log1p(likes) × 2.2 + log1p(saves) × 3.2
following_bonus = 3.5 (팔로우 중일 때)
bucket_boost = { Pick: +14, Theme: +8, Featured: +6,
                 Personalized: +7, Recent: +3, Rest: 0 }
noise = 버킷별 ±범위 내 난수
total = base + following_bonus + bucket_boost + noise
```
각 버킷 내부는 total 내림차순으로 정렬.

**4) 인터리빙 패턴 적용**
`pick → rest → theme → rest → personalized → rest → featured → rest → recent → rest` 순서로 1개씩 꺼내 최대 2 사이클 반복. 특정 버킷이 소진되면 다음 소스로 이동.

**5) 탭 필터**
- `전체`: 모든 전시.
- `혼자 올리기`: `primaryExhibitionType === 'solo'`.
- `함께 올리기`: `primaryExhibitionType === 'group'`.
- 레거시 전시(`primaryExhibitionType` 미설정)는 `imageArtists` 수로 fallback 분류.

**6) 공개 가시성 필터**
- `feedReviewStatus === 'approved'` 인 것만 노출.
- `isHidden === true` 는 전체 비노출(본인 프로필만 예외).
- 본인이 신고한 전시·작가는 본인 한정 자동 숨김(신고자 숨김).

**7) 무한 스크롤**
- 한 번에 **24건** 씩 로드. 스크롤 끝에 도달하면 다음 페이지.
- 로드 중 skeleton placeholder 표시.

**8) 이미 본 작품 기록**
- 상세 모달(USR-EXH-01) 진입 → 해당 전시 ID를 "이미 본 작품 목록"에 추가(최근 N건 순환).
- 다음 피드 로드 시 해당 전시는 동일 버킷 내 후순위로 밀림.

#### 출력
- **상단 배너 캐러셀** (모바일 가로 100% · 데스크톱 max-width 1024px, 자동 회전 인디케이터 도트)
- **탭 바** (전체/혼자/함께, 활성 탭 언더라인)
- **작품 카드 그리드** (모바일 2열 · 태블릿 3열 · 데스크톱 4~5열)
  - 각 카드: 썸네일(작품 톤 배경) · 좋아요·저장 상태 아이콘(카운트 비노출) · Pick 배지 · 강사 배지 · 다장 이미지 개수 배지
- **로드 상태**: skeleton placeholder
- **빈 상태**: 소개 문구 + 둘러보기 가이드

#### 주요 인터랙션

| 액션 | 동작 |
|---|---|
| 카드 탭 | USR-EXH-01 상세 모달 오픈 + URL `/exhibitions/:id` 갱신 + 피드 이전/다음 네비 가능 |
| 카드 내 좋아요·저장 | 전시 단위 토글. 비로그인 → CM-02 |
| 카드 내 작가 아바타·이름 탭 | USR-PRF-01 프로필 이동 |
| 배너 탭 | 외부 링크 또는 USR-EVT-02 |
| 탭 전환 | URL 쿼리 갱신 + 피드 재구성(스크롤 최상단) |
| 스크롤 끝 도달 | 다음 24건 로드 |
| 카드 길게 누르기 | **동작 없음**(시니어 UX 오조작 방지) |
| 풀 투 리프레시(모바일) | 최신 피드 재로드 |

#### 수용기준

**큐레이션·버킷**
- AC-01: Given Pick 10건이 있음 / When 피드 첫 페이지 로드 / Then Pick 버킷에 최대 10건 노출(중복 없음).
- AC-02: Given 팔로잉 작가 A의 신작 전시 B / When 피드 로드 / Then B가 Personalized 버킷에 포함 + 점수에 `follow +3.5` 반영.
- AC-03: Given 전시가 Pick이면서 기획전 포함 / When 피드 로드 / Then Pick 버킷에만 할당(Theme 버킷 중복 없음).
- AC-04: Given 추천 작가 3명 · 팔로잉 작가 2명 / When 피드 로드 / Then Featured / Personalized 버킷 각각 별도 소스에서 구성.

**공개 가시성**
- AC-05: Given 전시 `feedReviewStatus: pending` / When 피드 로드 / Then 비노출.
- AC-06: Given 전시 `isHidden: true` + 본인 프로필 / When 본인 프로필 로드 / Then 노출.
- AC-07: Given 전시 `isHidden: true` + 피드 / When 피드 로드 / Then 비노출.
- AC-08: Given 사용자가 신고한 전시 / When 피드 로드 / Then 해당 사용자에게만 숨김(다른 사용자에겐 정상 노출).

**인터리빙**
- AC-09: Given 각 버킷에 충분한 전시 / When 피드 로드 / Then 인터리빙 패턴대로 교차 배치 확인(Pick → Rest → Theme → Rest → ...).
- AC-10: Given 이미 본 작품 5개 / When 피드 재로드 / Then 각 작품은 속한 버킷 내 하단으로 밀려남.

**무한 스크롤·탭**
- AC-11: Given 승인된 전시 80건 / When 스크롤 끝 도달 / Then 24 → 24 → 24 → 8 순으로 페이지네이션.
- AC-12: Given 탭 "함께 올리기" 선택 / When 탭 / Then `primaryExhibitionType === 'group'` 만 노출 + URL `?tab=group` 갱신.
- AC-13: Given 탭 전환 / When 탭 / Then 스크롤이 최상단으로 복귀.

**비로그인·인터랙션**
- AC-14: Given 비로그인 + 좋아요 아이콘 탭 / When 탭 / Then CM-02(로그인 유도) 모달 오픈 1회, 같은 세션 재시도 시 토스트.
- AC-15: Given 로그인 완료 후 피드 복귀 / When 복귀 / Then `redirect` 파라미터 경로로 이동.

**배너**
- AC-16: Given 활성 배너 3 + 활성 이벤트 1 / When 피드 로드 / Then 캐러셀 4 슬라이드 + 5초 자동 회전.
- AC-17: Given 배너·이벤트 전부 비활성 / When 피드 로드 / Then 캐러셀 영역 숨김, 피드만 표시.
- AC-18: Given 캐러셀 슬라이드 탭 + 이벤트 슬라이드 / When 탭 / Then USR-EVT-02로 이동.

#### 엣지케이스
- EC-01: 승인된 전시 **0건** + 배너·이벤트 비활성 → 빈 피드 전용 UI("곧 새 전시가 공개됩니다") + 대체 CTA(이벤트 둘러보기 or 둘러보기 가이드).

- EC-05: 스토리지 쿼터 초과로 seen 기록 갱신 실패 → 조용히 무시(사용자 영향 없음).

## 3. USR-EXH · 전시·작품 상세

### USR-EXH-01 · 전시 상세 모달

**목적**: 선택한 전시의 이미지 시퀀스·메타 정보·인터랙션을 제공.
**트리거**: 피드·프로필·검색 결과에서 카드 탭, 또는 `/exhibitions/:id` 직접 진입(기본 분기).
**우선순위**: **P0**

#### 입력
- 전시 ID

- 현재 사용자 정보(팔로우 여부 판정용)

#### 처리
1. 전시 데이터 로드. 커버 이미지가 있으면 첫 슬라이드에 **블랙 배경 + 전시명 라벨**로 표시.
2. 본문 슬라이드: 이미지 배열 순회. 각 이미지에 **작품 톤 배경**(원본 이미지 blur + scale + opacity) 레이어.
3. 인터랙션 버튼(좋아요·저장·공유·신고·팔로우):
   - 비로그인 시도 시 `CM-02`.
   - 탈퇴 작가 전시는 모든 인터랙션 차단.
4. 공유 탭:
   - 모바일: 네이티브 공유 시트 (URL에 `?from=work` 자동 부여).
   - PC: 클립보드 복사 + 토스트.
5. 신고 → `CM-03` 신고 모달.

#### 출력
- 이미지 슬라이더(좌우 네비, 키보드 화살표 지원)
- 전시 메타(전시명·그룹명·작가 프로필 링크·설명)
- 인터랙션 아이콘 바(카운트 숫자 비노출)
- 배지(Pick·강사), 본인 한정 검수 상태 배지

#### 주요 인터랙션
- 이미지 우클릭·드래그 **차단**([Policy §18.4 개인정보](Policy_v1.md#18-데이터-보관저장-정책)는 별도지만 저작권 보호는 기본).

- 좌우 스와이프(모바일) → 피드 내 이전/다음 전시로 네비(피드에서 진입한 경우).

#### 수용기준
- AC-01: Given 피드에서 카드 탭 / When 모달 오픈 / Then URL이 `/exhibitions/:id`로 바뀜 + 뒤로가기로 모달 닫힘.

### USR-UPL-02 · 업로드 메인 (이미지·작가·메타 입력)

**목적**: 전시를 구성하는 이미지·작가·메타 정보를 한 화면에서 입력하고 검수 대기 상태로 발행. 업로드 플로우의 중심 화면이며 서브플로우가 여러 개 연결된다.
**트리거**: USR-UPL-01 유형 선택 완료, 초안 복원(`?draft=<id>`), 이벤트 응모(`?event=<id>`), 전시 수정(`?edit=<id>`).
**우선순위**: **P0**

기능 밀도가 높아 섹션별로 서브플로우를 정의한다.

#### A. 이미지 추가·관리

**파일 추가**
- input(accept=.jpg,.jpeg,.png,.webp,.gif) multiple + 드롭존 클릭·드래그앤드롭
- 동시 최대 10장. 초과 시 토스트 "최대 10장까지 업로드할 수 있습니다" + 초과 분 무시

**검증 순서(각 파일 순차)**
1. **크기 > 10MB** → "파일 크기는 10MB 이하여야 합니다"
2. **포맷 불일치**(MIME 검증) → "지원하지 않는 이미지 형식입니다"
3. **카메라 EXIF 차단** (USR-UPL-07 배너 트리거):
   - JPEG magic bytes(FF D8 FF) 확인
   - 첫 512KB에서 APP1(EXIF) 마커 → TIFF BOM → IFD0의 Make/Model 태그 검사
   - 존재 시 추가 차단 + 배너 표시
4. **해상도**: 짧은 변 < 800px → "짧은 변이 800px 이상이어야 합니다"
5. **WebP 변환**: 자동 변환 후 내부 data URL 저장

**진행률 표시**
- `{ current, total }` 스티키 프로그레스 바

**재정렬·삭제·교체**
- 2장 이상 시 "순서 변경" 버튼 → USR-UPL-05(DnD 모드)
- 카드 휴지통 → 파괴적 확인 다이얼로그
- Replace 아이콘 → 파일 선택 + 검증 반복 + 대표 이미지 인덱스 자동 재정렬

#### B. 작가 지정 (함께 전용 · USR-UPL-06 사이드패널)

- 이미지 카드 하단 "작가 지정" 슬롯 · 초기 미지정
- 패널 탭 2종: **회원 검색** / **직접 입력**

**회원 검색**
- 자동완성 입력 → 작가 리스트 최대 10명 (`name.toLowerCase().includes(검색어)`)
- 강사 역할이면 본인 자동 제외
- 선택 → `{ type: 'member', memberId, memberName, memberAvatar }`

**직접 입력(비회원)**
- 앰버 배경 + 경고 톤
- 이름 필수 20자 + 전화번호 선택(숫자 7자리 이상)
- 헬퍼: "가입 시 전화번호로 자동 매칭됩니다"
- 할당 → `{ type: 'non-member', displayName, phoneNumber }`

#### C. 메타 입력 (USR-UPL-04 세부정보 모달에서 관리)

- **전시명 (필수)**: 20자 카운터 + 비속어 검증
- **그룹명 (그룹 전시 필수)**: 20자 + 자동완성(내 최근 1 + 작품 기반 최대 30) + 중복 허용
- **개별 작품명 (선택)**: 각 20자 · 미입력 시 자동 생성 (혼자: exFinal 또는 "무제" · 그룹: `[작가명] - N`)
- **커스텀 커버**: 이미지 중 인덱스 선택 또는 로컬 파일(data URL, `coverImageIndex: -1`)
- **이벤트 연결 배너**: `?event=` 진입 시 녹색 배너 + 이벤트명
- **원작 확인**: 혼자 = "원작임을 확인합니다" / 함께 = "수강생들의 동의를 얻었습니다". 필수 체크

#### D. 딥링크 분기 (초안·이벤트·수정)

**`?draft=<id>`**
- 진입 시 초안 스토어에서 로드 → 프리필: uploadType · isInstructor · groupName · exhibitionName · contents[] · coverImageIndex · customCoverUrl
- 원작 확인 초기값 유지
- 진입 토스트 "이어서 작업할 수 있습니다"

**`?event=<id>`**
- 이벤트 세부 정보 모달에 녹색 배너 + 이벤트명
- 유형 선택 모달 스킵
- 발행 시 중복 참여 검증
- 발행 성공 후 `/events/<id>`로 리다이렉트(일반 완료 화면 대신)

**`?edit=<id>`**
- 기존 전시 프리필(유형·전시명·이미지·작가·커버·역할·이벤트 연결)
- 상단 "수정 중" 뱃지 + CTA 라벨 "수정 저장"
- 저장 시 **편집 가능 필드만 갱신**한다. 아래 불변 필드는 **보존**:
  - `id`(참조 무결성), `likes`·`saves`(인터랙션 카운터), `uploadedAt`(업로드 시각·피드 정렬 기준), `artistId`·`artist`(업로더 정체성), **`rejectionHistory`(반려 이력 — 감사·재범 추적용)**
- **`feedReviewStatus`는 항상 `pending`으로 재설정**(auto-approve 환경만 `approved`). 재검수 대상.
- **`rejectionReason`은 빈 값으로 초기화**(재발행 시 "현재" 반려 사유만 제거. 이력 배열 `rejectionHistory`는 별도로 보존).
- AP 적립 없음, 중복 이벤트 검증 스킵, 비회원 초대 미발송.
- 삭제된 전시 수정 시도 → 에러 토스트 "수정하려는 작품을 찾을 수 없습니다" + `/upload` 리다이렉트.

#### E. 발행 검증 순서 (에러 메시지 10종, 순차 중단)

|---|---|---|
| 1 | 이미지 0장 | "최소 1장 이상 필요합니다" |
| 2 | 전시명 빈 값 | "전시명을 입력해 주세요" |
| 3 | 원작 체크 미완료 | "원작 확인을 체크해 주세요" (강사: "수강생 동의를 확인해 주세요") |
| 4 | 전시명 비속어 | "부적절한 단어가 포함되어 있습니다(전시명)" |
| 5 | 그룹명 비속어 | "부적절한 단어가 포함되어 있습니다(그룹명)" |
| 6 | 함께인데 그룹명 빈 값 | "그룹명을 입력해 주세요" |
| 7 | 함께인데 특정 이미지 작가 미지정 | "이미지 N, M의 작가를 지정해 주세요" (구체 인덱스) |
| 8 | 함께(참가자)인데 본인 작품 0장 | "본인의 작품이 1점 이상 포함되어야 합니다" |
| 9 | 함께(참가자)인데 모든 이미지 동일 작가 | (모달) "이미지는 모두 동일 작가네요. 혼자 올리기로 전환할까요?" |
| 10 | 중복 이벤트 응모 | "이미 참여한 이벤트입니다" |

첫 위반 지점에서 중단 + 해당 필드 포커스·스크롤.

#### F. 발행 성공 처리

- Work 저장: id UUID · `feedReviewStatus: pending`(auto-approve 시 approved) · `uploadedAt` 로컬 ISO · `linkedEventId`(해당 시)
- 비회원 초대 정보는 `imageArtists[]`에 보존(검수 승인 시 발송)
- **AP 적립** 분기([Policy §7.2](Policy_v1.md#7-포인트ap-정책)):
  - 첫 업로드 +100 (최초 1회)
  - 일반 업로드 +20 (일 2회 상한, 첫 업로드 당일 제외)
  - 월 4회 +50 (월별)
  - 그룹 강사 첫 개설 +30 (월 1회)
  - 그룹 참여 +15 (월 5회)
  - 수정 모드(`?edit=`)는 적립 없음
- 비회원 초대 포함 시 → USR-UPL-08 프리뷰 모달 → 확인 후 최종 발행
- 완료 → USR-UPL-10

#### G. 24시간 내 삭제 회수

- 발행 후 24시간 이내 삭제 시 AP -20 회수(잔액 음수 방지)
- 회수 대상: 첫 업로드·일반 업로드 AP
- 월 4회·그룹 보너스는 회수 제외

#### H. 이탈 방지

- `useBlocker`: hasUnsavedWork = 이미지 > 0 || 전시명 || 그룹명 입력됨
- 경로 이동 감지 → USR-UPL-09 확인 다이얼로그
- 3 버튼: 취소 / 초안 저장(USR-PRF-10 이동) / 나가기(파괴적)

#### 수용기준 (통합 15개)

- AC-01: Given 이미지 11장 시도 / When 드롭 / Then 10장까지만 + 초과 에러 토스트.
- AC-02: Given JPEG EXIF Make "SAMSUNG" / When 추가 / Then 차단 + USR-UPL-07 배너.
- AC-03: Given 짧은 변 799px / When 추가 / Then "800px 이상" 에러 + 차단.
- AC-04: Given 참가자 + 본인 0장 / When 발행 / Then 에러 "본인 작품 1점 이상 필요".
- AC-05: Given 강사 OFF→ON 전환 / When 본인 지정된 슬롯 존재 / Then 슬롯 자동 초기화.

- AC-07: Given 첫 업로드 / When 발행 성공 / Then AP +100 적립(백그라운드, UI 미노출).

- AC-09: Given 24시간 이내 삭제 / When 확정 / Then AP -20 회수(잔액 음수 방지).

- AC-13: Given `?event=<id>` 중복 참여 / When 발행 / Then 에러 "이미 참여한 이벤트입니다".
- AC-14: Given 내용 입력 + 다른 경로 이동 시도 / When 이탈 / Then USR-UPL-09 모달.
- AC-15: Given 초안 저장 후 이탈 / When 프로필 진입 / Then USR-PRF-10에 초안 1건 추가.

#### 엣지케이스 (8)
- EC-01: 스토리지 쿼터 초과 → IndexedDB 오프로드 시도, 실패 시 에러 토스트.

## 5. USR-PRF · 프로필

### USR-PRF-01 · 프로필 홈

**목적**: 본인/타인 프로필의 작품·인터랙션·초안 등을 탭 단위로 제공. 시니어 작가의 "내 갤러리" 역할 + 감상자의 작가 탐색 지점.
**트리거**: 헤더 아바타, 작품 카드·알림의 작가 이름 탭, `/profile/:id` 직접 진입, `/me`, `/me/edit` → `/settings` 리다이렉트.
**우선순위**: **P0**

#### 입력
- 대상 사용자 ID(`/profile`·`/me`는 본인, `/profile/:id`는 타인)
- 선택된 탭(URL 쿼리 `?tab=<exhibition|works|likes|saved|drafts|student-works>`)
- 현재 로그인 사용자의 팔로잉 목록
- 작품 스토어(강사 배지·수강생 작품 탭 파생 기준)
- 탈퇴 작가 목록(익명화 판정)

#### 처리

**1) 헤더 렌더**
- 프로필 사진(기본 이미지 or 사용자 업로드 아바타)
- 이름(최대 20자) + 한 줄 프로필(headline, 최대 20자)
- 바이오(최대 200자, 여러 줄)
- 위치 배지(선택, 드롭다운에서 선택된 국가/지역)
- 관심사 토글 선택분(15종 중 선택된 것만 칩으로 표시 — 예: painting·drawing·digitalArt·printmaking·sculpture 등)
- 외부 링크(최대 6 플랫폼 중 입력된 것: Instagram·YouTube·Facebook·Twitter·Pinterest·Vimeo)
- **강사 배지**(자동 파생): 본인 업로드 전시 중 `isInstructorUpload === true`가 1건 이상.
- 팔로워·팔로잉 숫자(탭 시 USR-PRF-04 모달).
- 본인 프로필: 우상단에 "편집" 연필 아이콘(USR-PRF-02) + "카메라" 아이콘(USR-PRF-03).
- 타인 프로필: 팔로우/언팔로우 버튼(자기 자신·탈퇴 작가는 숨김 또는 비활성).

**2) 탈퇴 작가 처리**
- 대상 사용자 ID가 탈퇴 목록에 있으면:
  - 작가명을 "작가 미상"(deletedUser 라벨, 회색 텍스트)으로 대체
  - 아바타를 기본 이미지로 대체
  - 팔로우·신고 비활성(opacity 40% + `pointer-events-none`)

**3) 탭 가시성**
- 본인: `exhibition` / `works` / `likes` / `saved` / `drafts` + 강사면 `student-works`
- 타인: `exhibition` 단독. 다른 탭 URL 직접 진입 → `exhibition`으로 리다이렉트.
- 본인 탭에서 `?tab=student-works` 요청 + 강사 아님 → 기본 탭으로.

**4) 탭 상세 (§§5.1.1~5.1.6 참조)**

#### 출력
- 헤더(사진·이름·headline·바이오·위치·관심사·외부 링크·팔로워·팔로잉)
- 탭 바(본인/타인에 따라 탭 수 가변)
- 탭 콘텐츠

#### 주요 인터랙션

| 액션 | 동작 |
|---|---|
| 연필 아이콘(본인) | USR-PRF-02 편집 모달 |
| 카메라 아이콘(본인) | USR-PRF-03 사진 모달 |
| 팔로워·팔로잉 숫자 탭 | USR-PRF-04 모달(두 탭 전환) |
| 타인 프로필 팔로우 버튼 | 팔로우 토글(비로그인 → CM-02) |
| 외부 링크 탭 | 해당 URL 새 탭 |
| 탭 전환 | URL 쿼리 갱신, 스크롤 최상단 |

#### 수용기준
- AC-01: Given 본인 업로드 중 `isInstructorUpload === true` 1건 이상 / When 프로필 로드 / Then 헤더에 강사 배지 + `student-works` 탭 노출.
- AC-02: Given 강사 업로드 **전부 삭제** / When 프로필 재로드 / Then 배지·탭 모두 자동 비노출.
- AC-03: Given 탈퇴 작가 프로필 / When 로드 / Then 이름 "작가 미상" + 팔로우·신고 액션 비활성.
- AC-04: Given 타인 프로필에서 `?tab=drafts` URL / When 진입 / Then `exhibition` 탭으로 리다이렉트.
- AC-05: Given 본인 헤더 편집 완료 / When 저장 / Then 헤더에 즉시 반영(리로드 없이).
- AC-06: Given 비로그인 + 타인 프로필 팔로우 버튼 / When 탭 / Then CM-02.

#### 엣지케이스
- EC-01: 존재하지 않는 사용자 ID → "사용자를 찾을 수 없습니다" + 피드로 복귀 CTA.
- EC-02: 대상 사용자가 정지 영구 상태 → 프로필 접근 차단 안내 + 피드로.
- EC-03: 본인 프로필을 다른 탭에서 로그아웃 → 타인 프로필 뷰로 자동 전환.
- EC-04: 관심사·외부 링크가 0건 → 해당 영역 숨김(빈 블록 대신).

#### 의존
- 엔티티: USER_PROFILE, EXHIBITION, DRAFT, FOLLOW, INTERACTION
- 정책: [Policy §4 탈퇴 작가](Policy_v1.md#4-탈퇴-작가-정책) · [Policy §13.4 강사 파생](Policy_v1.md#13-업로드-유형역할-정책)
- 연결 화면: USR-EXH-01, USR-UPL-02, USR-PRF-02~04, USR-AUT-02, CM-02

#### 5.1.1 전시 탭 (`exhibition`)

**데이터**
- 소스: 본인이 올린 전시(`artistId === 본인 && !isInstructorUpload`) + **참여 작가로 연결된 전시**(타인이 올린 그룹 전시의 참여 작가로 본인이 등록된 경우)
- 강사 업로드는 여기서 제외(§5.1.3 수강생 작품 탭에서 별도).
- 타인 프로필에서 볼 때는 승인 완료(`approved`) + 공개(`!isHidden`)만.

**뷰**
- 필터: 전체 / 혼자 / 함께 (각 개수 표시).
- 본인 프로필: "내가 올린 전시만" 체크박스 — 참여 작가로만 연결된 전시 숨김.
- 전시 단위 카드(썸네일 + 전시명 + 그룹명(그룹) 또는 작가명(혼자) + 이미지 개수 배지).

**수정·삭제 권한**
- **본인이 직접 올린 전시만** 점점점 메뉴 노출(`artistId === 본인 || authorId === 본인`).
- 참여 작가로만 연결된 전시는 메뉴 없음(수정·삭제 불가. 업로더를 통해 수정 안내).

**점점점 메뉴**
- **전시 수정** → `/upload?edit=<id>`
- **삭제** (파괴적 스타일)

**전시 수정 동작**
- 기존 정보 전부 프리필.
- 저장 시 `feedReviewStatus`를 `pending`으로 재설정.
- AP 적립 없음.
- 반려 사유(`rejectionReason`)는 새 객체에 포함되지 않아 자연스럽게 제거.
- 비회원 초대 미발송(수정 모드).

**전시 삭제 동작**
- 확인 다이얼로그 제목에 전시명, 설명에 "영구 삭제, 되돌릴 수 없음".
- 보류된 비회원 초대가 있으면 추가 경고("초대가 무효화됩니다").
- 확인 시 전시·관련 인터랙션·알림·신고·Pick/Curation 참조 연쇄 정리.
- 24시간 내 삭제 시 AP -20.

**검수 상태 배지 (본인에게만)**
- `pending` → "확인 중"(muted)
- `rejected` → "게시 불가"(빨간 + 화살표 아이콘, 클릭 시 반려 사유 모달)
- `approved`는 배지 없음

**반려 전시 진입 동작**
- 본인이 rejected 전시 탭 → 상세 모달 대신 **반려 사유 모달** 오픈
  - "게시 불가 안내" 제목 + 사유(저품질/스팸/부적절/저작권) + "수정 후 재업로드하면 재심사를 받을 수 있습니다"
  - [닫기] / [수정하기] 버튼

#### 5.1.2 내 작품 탭 (`works`) — 본인만

**개념**: 전시 안의 **개별 이미지 1장 단위**로 평탄화한 그리드. "내가 그린 그림을 한 장씩 훑어보기 + 작품명 빠르게 수정" 용도.

**데이터**
- 본인 `artistWorks`(내가 올린 전시 + 참여 작가 연결 전시) + 태그된 작품(본인이 참여 작가로 등록된 전시)을 이미지 단위로 평탄화.
- `imageArtists` 있으면 → 본인 인덱스만 표시. 없으면 → 전 이미지.

**뷰**
- 이미지 단위 썸네일 그리드
- 카드 하단: "작품명" 배지 + 작품명 + 연필 아이콘 / "전시명" 배지 + 전시명

**작품명 변경 (인라인)**
- 작품명 텍스트·연필 영역 탭 → 중앙 모달, 현재 이름 프리필
- 카운터 `n/20` 실시간 표시. 20자 초과 입력 차단.
- 빈 문자열·공백만 저장 시 에러.
- 저장 시 해당 이미지의 제목만 수정 — 전시 객체(전시명·커버·검수 상태 등)는 건드리지 않음.
- **승인 완료 전시의 이미지 제목을 바꿔도 재검수 대상 아님** (전시 수정과 다름).

**이미지 썸네일 탭 → 전용 뷰어**
- WorkDetailModal과 별개의 가벼운 뷰어(좌/우 화살표로 내 이미지만 탐색).

**알려진 한계 — 동일 이미지 중복 표시**
- 같은 이미지를 여러 전시에 올리면 전시 수만큼 중복 표시됨. Phase 2에서 그룹화 UI 검토.

#### 5.1.3 수강생 작품 탭 (`student-works`) — 강사만

**노출 조건**
- 본인 업로드 중 `isInstructorUpload === true`가 1건 이상.

**데이터**
- `isInstructorUpload === true && artistId === 본인 && groupName 있음 && 그룹 전시 && 다른 수강생 크레딧 있음`

**뷰**
- 전시 단위 카드(수강생 작품 모음 그룹 전시)
- 카드 하단: 전시명 + 그룹명 + 크레딧된 수강생 이름 목록

**점점점 메뉴**(본인 프로필에서만)
- **전시 수정** → `/upload?edit=<id>` (강사 역할 자동 프리필, 본인 슬롯 없음)
- **삭제** (파괴적)

#### 5.1.4 좋아요·저장 탭

- **좋아요**(`likes`): 내가 좋아요한 타인 전시만(본인 작품 제외).
- **저장**(`saved`): 내가 저장한 타인 전시만.
- 전시 단위 카드(썸네일·전시명·작가명).
- 카드 탭 → USR-EXH-01.
- 해제: 상세 모달에서 토글(별도 해제 버튼 없음).

#### 5.1.5 초안 탭 (`drafts`) — 본인만

- 업로드 대기 중인 수동 저장 초안 목록.
- 카드: 초안 대표 이미지 + 제목 + 마지막 수정 시각.
- 카드 탭 → `/upload?draft=<id>` "이어서 작업".
- 점점점 메뉴: **삭제** 만.
  - 확인 다이얼로그: "이 초안을 삭제할까요?" + "영구 삭제"
  - 확인 시 초안 삭제.

#### 5.1.6 탭 공통 규칙

- `pending`·`rejected` 전시는 소유자 외에게 비노출.
- `isHidden` 전시는 타인 프로필에서 비노출.
- 탈퇴 작가는 전체 인터랙션 차단.
- 점점점 메뉴 터치 타깃 44×44px 준수.

## 8. USR-NTF · 알림

### USR-NTF-01 · 알림 센터

**목적**: 좋아요·팔로우·Pick·검수·이벤트·시스템 등 7가지 타입의 인앱 알림을 시간순으로 제공하고, 사용자가 읽음·삭제·원본 이동을 관리하도록 함.
**트리거**: 헤더 알림 아이콘(Bell + unread 카운트 배지), `/notifications` 직접 진입. 로그인 필수(비로그인 → `/login?redirect=/notifications`).
**우선순위**: **P0**

#### 입력
- 알림 목록(90일 보관 + 200건 상한, FIFO 자동 정리)
- 필터(읽음/안읽음 탭 + 타입 칩)
- 사용자 알림 설정(NOTIFICATION_SETTING)

#### 처리

**1) 알림 타입 7종과 메시지 포맷**

| 타입 | 아이콘 | 배경색 | 메시지 템플릿 | 라우팅 |
|---|---|---|---|---|
| `like` | 하트 | 빨강 50 | "[작가명]님이 '전시명'을 좋아합니다" | 전시 상세(USR-EXH-01) |
| `follow` | 사람 추가 | 파랑 50 | "[작가명]님이 회원님을 팔로우합니다" | 프로필(USR-PRF-01) |
| `pick` | 별 | 주황 50 | "'전시명'이 Artier's Pick으로 선정되었습니다" | 전시 상세 |
| `event` | 캘린더 | 초록 50 | 운영팀 작성 메시지(이벤트 공지) | 이벤트 상세(USR-EVT-02) |
| `system.review_approved` | 체크 | 회색 50 | "'전시명'이 게시되었습니다" | 전시 상세 |
| `system.review_rejected` | 경고 | 회색 50 | "'전시명'이 반려되었습니다 — 사유: [사유]" | 프로필 전시 탭(USR-PRF-05) → 반려 모달(USR-PRF-12) |
| `system.invite_accepted` | 사람 | 회색 50 | "[이름]님이 초대를 수락했습니다" | 프로필(USR-PRF-01) |

**2) 필터**
- **상단 탭**: 읽음 / 안읽음(기본)
- **타입 칩**: 전체 / 좋아요 / 팔로우 / Pick / 이벤트 / 시스템
- 필터 조합 결과는 최신순 정렬.

**3) 일괄 액션**
- "모두 읽음": 확인 다이얼로그 → 모든 `read: true` 갱신 + 헤더 배지 감소.
- "읽은 알림 삭제": 확인 다이얼로그 → `read: true` 항목 영구 제거.

**4) 개별 액션**
- 알림 카드 본체 탭 → 원본으로 이동 + 해당 알림 `read: true` 자동 갱신.
- 우측 X 버튼 탭 → 해당 알림 즉시 삭제(undo 없음).

**5) 자동 정리**
- 90일 초과 알림은 저장 시점에 자동 제거.
- 200건 상한 시 가장 오래된 것부터 FIFO.

**6) 알림 생성 조건**
- `like` 생성: 다른 사용자가 내 전시에 좋아요, 내 알림 설정 "좋아요" ON 시.
- `follow` 생성: 다른 사용자가 나를 팔로우, 내 알림 설정 "팔로우" ON 시.
- `pick` 생성: 내 전시가 Pick 선정될 때.
- `event` 생성: 참여한 이벤트 공지·종료 알림.
- `system.review_approved` / `rejected`: 내 전시 검수 판정 시.
- `system.invite_accepted`: 내가 초대한 비회원이 가입 + 자동 매칭 성공 시.

#### 출력
- 필터 바 · 일괄 액션 버튼
- 알림 카드 리스트 (타입별 아이콘·배경색 · 본문 · 상대 시간 · X 버튼)
- 빈 상태: "알림이 없습니다" + 아이콘

#### 수용기준
- AC-01: Given 비로그인 / When `/notifications` 접근 / Then `/login?redirect=/notifications`로 리다이렉트.
- AC-02: Given 안읽은 알림 15건 / When "모두 읽음" 확인 / Then 모두 `read: true` + 헤더 unread 배지 사라짐.
- AC-03: Given `like` 알림 탭 / When 클릭 / Then USR-EXH-01로 이동 + 해당 알림 `read: true`.
- AC-04: Given `system.review_rejected` 알림 탭 / When 클릭 / Then 프로필 전시 탭 + 반려 사유 모달(USR-PRF-12) 자동 오픈.
- AC-05: Given 92일 전 알림 / When 로드 / Then 비노출(자동 정리됨).
- AC-06: Given 알림 설정 "좋아요" OFF / When 누군가 내 전시에 좋아요 / Then 알림 미생성.
- AC-07: Given 201번째 알림 생성 / When 저장 / Then 가장 오래된 1건 자동 삭제.
- AC-08: Given 필터 "Pick" 칩 선택 / When 탭 / Then pick 타입만 노출 + URL `?type=pick`.

#### 엣지케이스
- EC-01: 알림이 가리키는 전시·작가가 삭제됨 → 카드에 "삭제된 콘텐츠" 표시 + 탭 시 이동 차단 + 에러 토스트.
- EC-02: 알림 저장 실패(스토리지 쿼터) → 가장 오래된 읽은 알림 자동 제거 후 재시도.
- EC-03: 알림 탭과 상세 모달 동시 오픈 경쟁 → 모달 진입 후 알림 read 처리(최종 일관성).
- EC-04: 다른 탭에서 알림 설정 변경 → storage 이벤트로 감지 + 화면 재렌더.

#### 비기능 요건
- 알림 센터 초기 렌더 < 500ms (200건 기준).
- 배지 카운트는 `storage` 이벤트·커스텀 이벤트로 실시간 반영.
- 접근성: 각 카드에 `aria-label`로 타입·시간·대상 포함.

#### 의존
- 엔티티: NOTIFICATION, NOTIFICATION_SETTING
- 정책: [Policy §1 알림 채널](Policy_v1.md#1-알림-채널-정책) · [Policy §20 확정 수치](Policy_v1.md#20-확정-수치-종합표)
- 연결 화면: USR-EXH-01, USR-PRF-01, USR-PRF-05, USR-PRF-12, USR-EVT-02, USR-AUT-02

## 9. USR-STG · 설정

### USR-STG-01 · 설정 홈

**목적**: 계정·알림·글꼴·언어 등 사용자 개인 설정 관리 + 로그아웃·탈퇴 진입.
**트리거**: 헤더 아바타 → 설정, `/settings` 직접 진입.
**우선순위**: **P0**

#### 입력
- 현재 사용자 프로필
- 알림 설정·글꼴 배율

#### 처리
1. 계정 ID·이메일 확인(읽기 전용).
2. 글꼴 크기: 작게(100%) / 보통(110%) / 크게(120%) — 루트 폰트 스케일.
3. 알림 토글 6종: 좋아요 · 팔로우 · 그룹 초대 · 팔로잉 신작 · 주간 테마 · 마케팅.
4. 비밀번호 변경 진입(이메일 사용자만 활성; 소셜 사용자는 비활성 + 안내).
5. 로그아웃.
6. 회원 탈퇴 진입 → USR-STG-03.
7. 하위 해시 앵커 `#notifications` 지원.

#### 수용기준
- AC-01: Given 글꼴 "크게" 선택 / When 저장 / Then 전체 앱 폰트가 120% 스케일.
- AC-02: Given 소셜 사용자 / When 비밀번호 변경 버튼 / Then 비활성 + 툴팁 "소셜 로그인 사용자는 이 기능이 제공되지 않습니다".
- AC-03: Given 알림 "좋아요" 토글 OFF / When 이후 다른 사용자가 내 작품에 좋아요 / Then 알림 미생성.

#### 의존
- 엔티티: USER_PROFILE, NOTIFICATION_SETTING, UI_PREFERENCES
- 정책: [Policy §19 접근성](Policy_v1.md#19-접근성시니어-친화-정책)
- 연결 화면: USR-STG-02, USR-STG-03, USR-INF-07

### USR-STG-03 · 탈퇴 모달

**목적**: 사용자 탈퇴 + 사유 수집.
**트리거**: USR-STG-01 탈퇴 버튼.
**우선순위**: **P0**

#### 입력
- 비밀번호(이메일 사용자) 또는 소셜 재인증 체크(소셜 사용자)
- 탈퇴 사유 5택: 이용 빈도 낮음 / 타 서비스 이용 / 개인정보 우려 / 이용 불편 / 기타
- 기타 선택 시 텍스트 입력 1건

#### 처리
1. 비밀번호·재인증 확인.
2. [Policy §4](Policy_v1.md#4-탈퇴-작가-정책)에 따라:
   - 작가명 "작가 미상"으로 익명화
   - 프로필 접근 차단
   - 작품은 유지, 인터랙션 차단
3. 로그아웃 + USR-AUT-02로 이동.

#### 출력
- 경고 블록("이 작업은 되돌릴 수 없습니다")
- 사유 라디오 + 기타 텍스트 필드
- 파괴적 확정 버튼

#### 수용기준
- AC-01: Given 비밀번호 오답 / When 확정 시도 / Then 에러.
- AC-02: Given "기타" 선택 + 텍스트 빈 값 / When 확정 시도 / Then 에러 "사유를 입력해 주세요".
- AC-03: Given 확정 / When 완료 / Then 로그아웃 + 프로필 익명화 + 로그인 화면으로 이동.

#### 의존
- 엔티티: USER_PROFILE (탈퇴 상태 전환)
- 정책: [Policy §4](Policy_v1.md#4-탈퇴-작가-정책)
- 연결 화면: USR-AUT-02

6. 위탁(카카오 알림톡·SMS 게이트웨이·이메일 등 Phase 2)
7. 이용자의 권리(열람·정정·삭제·처리 정지 요청)
8. 쿠키·트래킹(수집 항목과 거부 방법)
9. 개인정보 파기 절차·방법
10. DPO 연락처
11. 변경 고지 절차

#### 수용기준
- AC-01: Given 진입 / When 로드 / Then DPO 이름·연락처 상단 표시.
- AC-02: Given 쿠키 배너 "자세히" / When 탭 / Then 8번 "쿠키·트래킹" 섹션으로 이동.
- AC-03: Given 만 14세 미만 사용자(차단돼야 하지만 우회 진입) / When 페이지 로드 / Then 본 페이지는 정상 표시(법적 문서이므로 열람 차단하지 않음).

#### 엣지케이스
- EC-01: 개정일이 최근 30일 이내 → 상단 "최근 개정되었습니다" 배지.

#### 노트
- 런칭 전 법무 확정 필수([SystemArch §10 N-10](SystemArchitecture_v1.md), [Policy §21.1 L-2](Policy_v1.md#21-법무-체크포인트)).
- **데이터 내보내기·삭제 요청 기능은 런칭 전 필수**([Policy §21.3](Policy_v1.md#21-법무-체크포인트)) — 요청 접수 채널은 USR-INF-07.

### USR-INF-07 · 문의하기
**목적**: 사용자 문의 수집.
**트리거**: 푸터 링크, `/contact`, 정지 안내·에러 바운더리에서 연결.
**우선순위**: **P0**
#### 입력
- 이름
- 이메일
- 카테고리: 계정 / 업로드 / 신고 / 제안 / 버그 / 기타
- 내용 (최대 1000자)
- 첨부 파일 (5MB/개, 최대 3개 — 이미지/PDF/문서)
#### 처리
- 형식·용량 검증 후 문의 큐에 저장.
- 정지 계정(USR-AUT-07) 진입 시 카테고리 "신고·기타" 프리필.
#### 수용기준
- AC-01: Given 내용 1001자 / When 제출 / Then 에러 "1000자 이하로 입력해 주세요".
- AC-02: Given 첨부 4개 시도 / When 업로드 / Then 에러 "첨부는 최대 3개입니다".
- AC-03: Given 파일 6MB / When 업로드 / Then 에러 "5MB 이하 파일만 첨부할 수 있습니다".
#### 의존
- 엔티티: INQUIRY
- 연결 화면: 운영 측 ADM-* (향후)

## 11. CM · 공통 컴포넌트·다이얼로그

> 모든 CM 화면의 상세 정의는 [IA_ScreenList §4](IA_ScreenList_v1.md#4-공통-팝업다이얼로그-cm). 아래는 사용자 앱(USR) 기준의 수용기준.

### CM-01 · 확인 다이얼로그
- AC-01: Given 파괴적 액션 트리거 / When 다이얼로그 오픈 / Then 확정 버튼 빨간 스타일 + 취소는 기본 스타일.
- AC-02: Given 배경 탭 또는 ESC / When 트리거 / Then 닫힘(취소와 동일).
- AC-03: Given 제목에 변수(예: 전시명) 포함 / When 오픈 / Then 변수가 렌더링됨.

### CM-02 · 로그인 유도 모달
- AC-01: Given 비로그인 + 좋아요 시도 / When 탭 / Then 모달 오픈.
- AC-02: Given 세션당 1회 모달 표시 후 같은 액션 재시도 / When 탭 / Then 모달 대신 토스트.
- AC-03: Given 로그인 버튼 탭 / When 이동 / Then `/login?redirect=<현재경로>`.

### CM-03 · 신고 모달
- 사유 4종 선택(저품질/스팸/부적절/저작권) + 상세 텍스트(선택).
- 동일 `(신고자, 대상)` 조합 재신고 시도 → 에러 "이미 신고한 대상입니다".

### CM-04 · 초대 본인 확인 게이트
- 홈 진입 시 pending 초대 목록이 있으면 순차 노출.
- 3택: 수락 / 거부 / 나중에.
- 수락 → 수동 매칭 승격 + 알림 1건. 거부 → 발신자 경고 +1.

### CM-05 · 쿠키 동의
- 최초 방문 시 하단 배너. "동의" 탭 시 배너 닫힘 + 플래그 저장.

### CM-06 · 오프라인 배너
- 네트워크 오프라인 감지 시 상단 고정 배너. 온라인 복귀 시 자동 해제.

### CM-07 · 에러 바운더리
- 렌더 예외 시 "문제가 발생했어요" + 새로고침 버튼 + 문의(USR-INF-07) 링크.

### CM-08 · 404 / 500 / 점검
- 각 상황에 맞는 안내 + 홈 복귀 CTA.

### CM-09 · QA 바로가기 (DEV·검수 전용)
- 플래그 활성 시 플로팅 버튼 노출. 런칭 빌드 비활성.

### CM-10 · 데모 툴 (`/demo`, `/demo/reference`)
- PM 데모·검수 전용. **배포 전 제거** 또는 플래그 차단.

## 12. 사용자 플로우 요약

### 12.1 신규 가입 플로우

1. USR-AUT-02 → 소셜 or 이메일 로그인 선택
2. USR-AUT-03 (이메일) 또는 USR-AUT-05 (소셜) → 약관·닉네임
3. USR-AUT-06 온보딩 → 실명·전화·이메일·프로필 이미지
4. USR-BRW-01 → 서비스 시작

### 12.2 SMS 초대 → 가입 플로우

1. 외부에서 초대 링크(`?from=invite`) → USR-EXH-03 랜딩
2. 가입 CTA → USR-AUT-03 → 이름·전화 프리필
3. 온보딩 완료 → 자동 매칭 시도
4. 이름 일치 → 자동 연결 + 알림 / 이름 불일치 → CM-04

### 12.3 작품 업로드 플로우

1. USR-UPL-01 유형·역할 선택
2. USR-UPL-02 입력(이미지·작가·메타)
3. 발행 → `feedReviewStatus: pending`
4. USR-UPL-03 완료 확인 → USR-PRF-01 or USR-BRW-01
5. 검수 승인 시 피드 노출 + 팔로워 알림 + 보류된 비회원 초대 발송

### 12.4 둘러보기 → 인터랙션 플로우

1. USR-BRW-01 피드
2. 카드 탭 → USR-EXH-01 상세
3. 좋아요·저장·팔로우·공유·신고 — 비로그인 시 CM-02
4. 신고 → CM-03

### 12.5 신고 → 처리 플로우 (사용자 측)

1. USR-EXH-01 or USR-PRF-01 신고 버튼 → CM-03
2. 신고 접수 → 운영팀 큐(ADM-RPT-01)에 표시
3. 처리 결과에 따라 대상 작가·신고자 알림(§12.1/§12.4)

### 12.6 회원 탈퇴 플로우

1. USR-STG-01 → USR-STG-03
2. 비밀번호·재인증 확인 → 탈퇴 사유 선택 → 확정
3. 작가명 "작가 미상" 익명화 + 작품 유지 + 인터랙션 차단
4. USR-AUT-02로 이동
