# SuperGallery_Phase 1 PRD (작성 중)

## 1. 개요 (Overview)

### 1-1. 프로젝트 배경

- 그림을 그리는 많은 창작자들이 자신의 작품을 선보이고 소통할 수 있는 전문적인 공간이 부족함.
- 특히, 경력이 단절되었거나 비전공자인 실력 있는 작가들이 재도전할 수 있는 기회가 제한적임.
- Artier는 누구나 자신의 작품을 디지털 갤러리에 전시하고, 이를 통해 새로운 기회를 발견하는 것을 돕는 플랫폼으로 시작하고자 함.

### 1-2. Phase 1 목표

- **핵심 가설 검증**: ’창작자들은 자신의 작품을 손쉽게 전시하고, 이를 통해 만족감을 얻으며, 다른 사용자들과 교류하고 싶어한다’는 가설을 검증함.
- **초기 사용자 확보**: 서비스를 자발적으로 사용하는 초기 창작자 그룹(100명)을 확보하여 플랫폼 활성화의 기반을 마련함.
- **안정적인 MVP 런칭**: 5월 첫째 주까지 핵심 기능을 포함한 서비스를 안정적으로 출시하고 운영함.

## 3. 서비스 구조 (Information Architecture) - 1차 MVP 기준

```
Artier (MVP)
├── 둘러보기 (홈)
│   ├── 메인 배너 (이벤트/공지)
│   └── 작품 피드 (랜덤)
│
├── 작품 상세
│   ├── 작품 이미지 (딥줌 뷰어)
│   ├── 작품 정보 (제목)
│   ├── 작가 정보 (프로필, 다른 작품 보기)
│   ├── 인터랙션 (좋아요, 저장)
│   └── 공유하기 (고유 URL)
│
├── 작품 업로드
│   ├── 작품 업로드 (드로잉툴(그림,타임랩스 불러오기), 이미지(png, jpeg 불러오기), 텍스트 추가)
│   ├── 전체 전시 꾸미기 (배경색상 설정, 콘텐츠 간격 설정)
│   ├── 개별 작품 정보 입력 (개별 작품 제목(선택), 작업자(선택))
│   └── 정보 입력 (썸네일, 전시 제목, 카테고리, 태그, 그룹명, 공동작업자 태깅)
│
├── 이벤트
│   ├── 진행 중인 이벤트 목록
│   └── 이벤트 상세 페이지
│
└── 마이페이지
    ├── 내 프로필 관리
    ├── 내가 올린 작품 관리
    ├── 좋아요, 저장한 작품
    └── 초안 보기
    
```

  • (한국) 카카오톡, 휴대폰 번호, 이메일 로그인/간편가입
  • (한국 외) 이메일, 구글, 애플 로그인/간편가입

| **USER-002** | **작가 프로필 페이지** | - 작가의 프로필 이미지, 작가명, 자기소개, 관심 분야, 주요 작업 도구 등을 표시하는 페이지가 있어야 함.

| **USER-004** | **작가 팔로우/언팔로우** | - 다른 사용자의 프로필 및 작품 상세 페이지에서 '팔로우' 버튼을 누를 수 있어야 함.
- 팔로우 시 버튼은 '팔로잉' 상태로 변경되며, 다시 누르면 언팔로우 처리되어야 함.
- 팔로우 기능 자체를 우선 구현하며, 실시간 알림 전송은 Phase 2에서 구현함. |

### Epic 2: 작품 업로드 및 전시

| ID | 기능명 | 상세 요구사항 (음슴체) |
| --- | --- | --- |
| **ART-001** | **작품 업로드** | - 웹에서 여러 장의 이미지(JPG, PNG, WEBP 등, 최대 10MB)를 드래그 앤 드롭 또는 파일 선택으로 업로드할 수 있어야 함.
- 업로드 시, 작품 제목, 카테고리(미술, 패션, 공예, 제품 디자인), 창작툴을 입력할 수 있어야 함.

- 사용자는 작품 업로드 시 배경색, 작품 간 간격 등 간단한 전시 효과를 설정할 수 있어야 함. |
| **ART-003** | **저작권 보호 뷰어** | - 작품 상세 페이지에서 이미지 클릭 시, 고해상도 이미지 뷰어가 모달(Modal) 형태로 제공되어야 함.
- 작품에는 이미지 우클릭 방지 기능이 기본으로 적용되어야 함. |
| **ART-004** | **작품 관리** | - ‘내 프로필 > 작품 관리’ 탭에서 내가 올린 작품 목록을 확인하고, 수정/삭제할 수 있어야 함. |
| **ART-005** | **공유하기** | - 작품 상세 페이지에서 ‘공유하기’ 버튼을 누르면 외부 채널 직접 공유 기능 오픈 |
| **ART-006** | **사진 차단 기능** | - 작품 업로드 시 스마트폰이나 카메라로 찍은 사진 차단 되어야 함
- EXIF 데이터 분석 및 차단: 카메라 메타데이터 포함된 경우 업로드 자동 차단 |

## 5. 화면 설계 (Wireframe & Prototype)

### 5-1. 메인 피드 (둘러보기)

- **상단 (GNB)**: 화면 상단에는 서비스 로고, ‘둘러보기’, ‘이벤트’ 등 주요 메뉴가 위치함. 우측에는 알림 아이콘, ‘작품 업로드’ 버튼, 그리고 사용자 프로필 아이콘, 언어 변경(영/한)이 순서대로 배치됨.
- **중앙 (콘텐츠 영역)**:
    - **히어로 배너**: GNB 바로 아래에는 주목도 높은 이벤트나 기획전을 알리는 대형 배너가 슬라이드 형태로 자동 전환됨.
    - **카테고리 필터**: 카테고리 필터는 Phase 1에서는 제공하지 않음
    - **작품 그리드**: 메인 콘텐츠 영역으로, 다양한 크기의 작품 이미지들이 벽돌처럼 엇갈리게 배치된 Masonry 레이아웃을 사용함. 
    각 작품 카드 위에는 작품명이 표시되고,  하단에는 좋아요 수와 작가명이 표시.
    - 작가명 섹션: 단일 작가, 공동 작업 인지에 따라 표기 방식 분기됨
        - 단일 작가: 업로드 작가의 닉네임 표기
        - 공동 작업: 작품 업로드 시 그룹명을 입력 하였다면 팀명이 표시되고, 미입력 시 공동 작업자 라는 명칭으로 표출. 해당 명칭에 호버 시 툴팁 형태로 작업자 리스트 오픈
    - Phase 1에는 피드 랜덤 표출하나, 이전에 봤었던 전시를 제외하고 랜덤 제공함

### 5-2. 작품 상세

- 메인 피드에서 특정 작품을 클릭하면, 화면 전체를 덮는 라이트박스(모달) 형태로 상세 정보가 나타남.
- **라이트박스 상단 영역(라이트박스 외부):** 모달 위 작품 제목과 작가명(그룹명) 오픈, 우상단에는 팔로우 버튼
- **라이트박스 내부 영역**: 선택된 작품의 고화질 이미지가 크게 표시됨. 여러 장의 이미지로 구성된 작품일 경우, 상하로 이동하며 그림 보기 진행. 
그림 보기 아래에 작가 프로필 아이콘과 팔로우를 다시 한번 노출하며, 가장 하단에는 해당 작가의 다른 작품들을 작은 썸네일 형태로 보여주어, 사용자의 추가적인 탐색을 자연스럽게 유도함.
- **우측 영역(라이트박스 외부)**:
    - 상단부터 작가 프로필 액션 버튼, 그 아래로 ‘좋아요’, ‘저장’, ‘공유하기’ 사용자가 작품과 상호작용할 수 있는 주요 액션 버튼 배치
- **하단 영역(라이트박스 외부):**
    - 이전, 다음 버튼이 있으며 클릭 시 아티스트 간 이동

### 5-3. 작품 업로드

- 작가가 자신의 작품을 등록하는 페이지로, 크게 세 단계로 구성됨.
- **1단계 (파일 선택)**: 화면 중앙의 점선 영역에 이미지 파일을 드래그 앤 드롭하거나, ‘작품 추가’ 버튼을 눌러 로컬 파일을 선택함. ’드로잉 툴에서 가져오기’를 통해 SGF 연동도 가능함.
- **2단계 (전시 설정)**: 우측 패널에서 업로드된 작품들이 전시될 배경색과 작품 간의 간격을 슬라이더로 조절하여 ’1-Click 갤러리’의 디자인을 커스터마이징함.
- **3단계 (정보 입력)**: ‘세부 정보 설정’ 버튼을 누르면 작품의 제목, 카테고리, 태그, 그룹명 등 메타데이터를 입력하는 모달 창이 열림. 모든 정보 입력 및 ‘본인 창작물 여부’ 확인 체크 후 ‘계속’ 버튼을 눌러 업로드를 완료함. 
계속 버튼 아래 초안으로 저장, 미리보기 확인 버튼도 제공하며, 초안으로 저장 시 업로드 되지 않고 프로필>초안 탭에서 작품 확인 가능

### 5-4. 작가 프로필

- **상단 (프로필 정보)**: 화면 좌측에 작가의 프로필 이미지, 자기소개, 관심 분야, 팔로워/팔로잉 수, 사용하는 작업 도구 등 상세 정보가 표시됨. ‘프로필 편집’ 버튼을 통해 수정 가능함.
- **하단 (콘텐츠 탭)**: ‘작품’, ‘작품 관리’, ‘좋아요’, ‘저장’, ‘초안’ 등 탭 메뉴를 통해 작가 본인의 활동 내역을 확인하고 관리할 수 있음.
    - 작품: 해당 작가가 포함된, 제작한 모든 전시 형태가 그리드 형태로 나열됨.
    - 작품 보기: 개별 작품(1개)이 그리드 형태로 나열됨
    - 좋아요: 좋아요 누른 전시 모아보기
    - 저장: 저장 누른 전시 모아보기
    - 초안: 작품 업로드 시 [초안으로 저장] 버튼 클릭한 미완성 작품에 대해 보기 가능. 작품 클릭 시 업로드 편집 화면으로 이동

| **지원 기기** | - PC 및 모바일(스마트폰) 반응형 웹으로 동시 지원. 단일 코드베이스로 개발 공수를 최소화함.
- 태블릿은 반응형 웹으로 자동 대응 |
| **성능** | - 이미지 로딩 최적화: 메인 페이지 및 작품 상세 페이지의 이미지는 3초 이내에 로드되어야 함.
- 서버 응답 시간: 주요 API 요청에 대한 서버 평균 응답 시간은 500ms 이하를 목표로 함. |
| **보안** | - 사용자 개인정보(이메일, 비밀번호 등)는 암호화하여 안전하게 저장해야 함.
- 기본적인 웹 취약점(XSS, CSRF 등)에 대한 방어 조치가 적용되어야 함. |
| **데이터** | - GA4를 연동하여 주요 사용자 행동(페이지뷰, 가입, 작품 업로드, 이벤트 참여 등)을 추적하고, 핵심 지표를 측정할 수 있는 대시보드를 구성해야 함. |
| **운영** | - PM이 직접 업로드된 콘텐츠를 검수하고 가이드라인 위반 시 대응할 수 있는 최소한의 관리자 기능이 필요함.  |
